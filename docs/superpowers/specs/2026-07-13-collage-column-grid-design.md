# 히어로 콜라주 — 풀스크린 포스터 → 컬럼 그리드

- 날짜: 2026-07-13
- 대상: `src/hero/HeroReveal.tsx`, `src/hero/HeroReveal.module.css`, `src/constants/church.ts`, `.claude/rules/DESIGN.md`
- 선행 스펙: `2026-07-13-hero-poster-crossfade-design.md` (영상→포스터 크로스페이드, `<source media>`)

## 1. 배경

메인 히어로 `HeroReveal`은 단일 스크럽 타임라인으로 십자가 리빌 → 카피 → 미디어 전환 → 사진 4장
진입을 연출한다.

기존 배치는 **"중앙 카드 주변을 타일이 둘러싼다"** 는 전제 위에 있었다. 타일이 화면 가장자리에서
날아 들어와 중앙 카드 모서리를 물고 겹친다(레이어드 오버랩). 이 전제에는 구조적 긴장이 있다 —
중앙과 타일이 같은 공간을 두고 다투므로 **하나를 키우면 다른 하나가 작아진다.** 사진을 키워달라는
요구가 반복적으로 이 벽에 부딪혔다.

또 타일 슬롯이 모두 고정 비율(4:3·3:2·16:10)이라 사진을 `object-fit: cover`로 잘라 넣었다.
`collage-1`(1160×992, 비율 1.17)은 나머지 셋(1.78 / 1.50 / 1.78)보다 현저히 덜 넓어서, 같은
슬롯에 넣으면 **위아래가 약 27% 잘렸다**(설교자 머리 위·강대상 아래가 날아간다).

## 2. 결정

두 가지를 바꾼다.

**(1) 히어로 미디어는 카드로 남지 않는다.** 축소 카드는 두 번 손해였다. `clip-path`는 리사이즈가
아니라 **창**이라 포스터 면적의 약 13%만 보여줬고(십자가·꽃·하늘은 창 밖), 그러면서 컨테이너 폭의
**46%(525/1152)** 를 먹어 사진을 작게 만들었다. 대신 **풀스크린을 유지한 채** 영상을 포스터로
굳히고(포스터를 처음으로 온전히 보여준다), 그다음 포스터가 흰 캔버스로 페이드 아웃하며 **비켜준다.**
그 폭 전부를 사진이 가져간다.

**(2) 크롭을 레이아웃으로 푼다.** 슬롯에 사진을 맞추는 대신 **사진 비율에 슬롯을 맞춘다.**
좌·우 컬럼 폭을 서로 다르게 잡아 두 컬럼 높이가 정확히 일치하게 만든다. 결과적으로 **사진 4장 전부
원본 비율, 크롭 0.**

`collage-1`의 "세로로 긺"은 높이를 **주지 않는다**. 덜 넓은 사진이라 같은 폭에서 키가 커지므로,
컬럼 폭을 좁히는 **압력**으로 작용한다. `collage-1`은 좁은 왼쪽 컬럼에 들어가 넷 중 가장 작은 타일이
된다. 이것이 크롭 0의 대가다.

## 3. 레이아웃 대수 (핵심)

수치를 하드코딩하지 않는다. **사진 비율만 선언하고 나머지는 JS가 푼다** — 사진을 갈아끼워도 상하
정렬이 저절로 맞는다.

콜라주 폭 `W`, 간격 `g`(= `--spacing-lg`, 24px), 컬럼당 셀 `n`개일 때:

```
sL = Σ(1/좌측 사진 비율)      // 폭 1px당 늘어나는 좌 컬럼 높이
sR = Σ(1/우측 사진 비율)
vGap = (n-1)·g                // 컬럼 내부 세로 간격 합

좌 컬럼 폭 L = (W - g)·sR / (sL + sR)
우 컬럼 폭 R = (W - g)·sL / (sL + sR)
콜라주 높이 H = (W - g)·(sL·sR / (sL + sR)) + vGap
```

유도: 컬럼 높이 `H = 폭 · Σ(1/비율) + vGap` 이고 `L + R + g = W` 이다. 두 컬럼 높이를 같게 두면
`L·sL = R·sR` 이 나오고, 여기에 폭 제약을 대입하면 위 식이 된다.

**폭 `W`는 두 곳에서 캡된다:**

```
W = min( 컨테이너 콘텐츠 폭,        // 1200 - 2·24 = 1152 (DESIGN.md — 콘텐츠 1200px 캡)
         뷰포트 폭 - 2·패딩,
         (헤더 아래 남는 높이 - vGap) / k + g )   where k = sL·sR / (sL + sR)
```

세 번째 항이 높이 캡이다. 폭과 높이가 `k`로 묶여 있으므로 **높이를 제한하려면 폭을 줄이는 수밖에
없다.** 이게 없으면 짧은 뷰포트(예: 1440×700)에서 상단 타일이 고정 헤더(80px)에 잘린다.

### 확정 수치 (현재 자산, 1440×900 · W=1152 · g=24 · n=2)

| 사진 | 크기 | 비율 |
|---|---|---|
| `collage-1` | 1160×992 | 1.169 |
| `collage-2` | 4032×2268 | 1.778 |
| `collage-3` | 5184×3456 | 1.500 |
| `collage-4` | 4000×2252 | 1.776 |
| `hero-poster` | 1920×1456 | 1.319 |

좌 컬럼 = `collage-1` + `collage-3` / 우 컬럼 = `collage-2` + `collage-4`

| | 폭 | 셀 높이 | 합 |
|---|---|---|---|
| 좌 컬럼 | **480px** | 410 + 24 + 320 | **754** |
| 우 컬럼 | **648px** | 365 + 24 + 365 | **754** |

480 + 648 + 24 = **1152** ✓ · 두 높이 모두 **754px** ✓ · 타일 4장 크롭 **0** ✓

중앙 카드를 뺀 효과(1440×900 기준):

| 사진 | 카드 있을 때 | 카드 뺀 뒤 |
|---|---|---|
| `collage-1` | 246×210 | **480×410** |
| `collage-3` | 246×164 | **480×320** |
| `collage-2` | 333×187 | **648×365** |
| `collage-4` | 333×187 | **648×365** |
| 콜라주 높이 | 398px | **754px** |

선형 약 2배, 면적 약 3.8배. 754px은 헤더 아래 가용 높이(900 - 80 - 48 = 772px)를 거의 채우므로
"사진은 작은데 여백만 넓다"는 문제도 함께 사라진다.

## 4. 모션

| 구간 | 동작 |
|---|---|
| 0.00–0.34 | 십자가 열쇠구멍 확대 (easeIn) |
| 0.36–0.46 | 카피 인 |
| 0.46–0.60 | **영상 → 포스터 크로스페이드 (풀스크린 유지, 축소 없음)** |
| 0.56–0.64 | 카피 아웃 |
| 0.64 | 헤더 solid 전환 임계 (흰 캔버스가 드러나기 직전) |
| 0.66–0.80 | **포스터 페이드 아웃 → 흰 캔버스** |
| 0.70–0.96 | 사진 4장 드리프트 인 (heroOut과 겹쳐 끊김 없이) |

포스터 페이드는 **선형**이다. `easeOut`을 걸면 앞구간에서 확 사라지고 끝에서 늘어져(진행 50%에
이미 opacity 0.125) 뚝 끊겨 보인다. 크로스디졸브는 등속이 맞다.

타일 스태거(컬럼 안에서 위 셀이 먼저):

| 타일 | 컬럼 | 진입 | 구간 |
|---|---|---|---|
| `collage-1` | 좌 | 아래에서 ↑ (translateY 40vh → 0) | 0.70–0.88 |
| `collage-2` | 우 | 위에서 ↓ (translateY -40vh → 0) | 0.72–0.90 |
| `collage-3` | 좌 | 아래에서 ↑ | 0.76–0.94 |
| `collage-4` | 우 | 위에서 ↓ | 0.78–0.96 |

스크럽이라 스크롤 위치의 순수 함수다 — 되올리면 역재생된다. 조립이 끝난 뒤(p=1) 더 내려도
흩어지지 않고, sticky가 풀리며 완성된 채로 페이지와 함께 올라간다.

`transform`·`opacity`만 쓴다(reflow 금지). `clip-path`는 더 이상 쓰지 않는다 — 축소가 없으므로.

## 5. 구현

### 5.1 타입 — `src/hero/types.ts`

레이아웃 대수가 사진 비율을 필요로 하므로 선언한다. 런타임 `naturalWidth` 측정은 비동기라
첫 페인트에 기하가 없고, 하드코딩한 폭 값은 사진 교체 시 정렬이 조용히 깨진다.

`HeroMedia`에 `aspect`를 붙이지 **않는다** — `DeptHero`가 공유하는 타입이라 비율을 알 필요 없는
소비자까지 오염된다. 콜라주 타일 전용 타입을 둔다:

```ts
// 콜라주 타일 — 비율이 레이아웃의 입력이라 필수다(HeroMedia와 달리 영상 변형이 없다).
export interface CollageTile {
  src: string;
  alt?: string;
  aspect: number; // 가로/세로
}
```

### 5.2 상수 — `src/constants/church.ts`

- `COLLAGE_TILES: CollageTile[]` — 각 항목에 `aspect` 기입
- `HERO_POSTER_ASPECT` — 포스터 원본 비율. **정적 스택(§5.6)에서만** 쓴다(스크럽 레이아웃의
  입력이 아니다 — 중앙 카드가 없으므로).

### 5.3 프롭 배선

`HeroRevealProps.tiles`는 `CollageTile[]`, `posterAspect: number`. `HeroHeaderSync` → `HeroReveal`
경로로 전달하고 `page.tsx`가 상수를 주입한다.

### 5.4 `HeroReveal.tsx`

- `measure()`가 §3 공식으로 `W · H · L · R`과 콜라주 박스 원점을 계산한다.
  세로 중앙정렬 기준은 **헤더 아래 영역**(`--spacing-nav`) — 이 시점의 헤더는 이미 solid다.
- 계산 결과를 **타일 4개의 인라인 `left/top/width/height`** 로 직접 적용한다. 컬럼 폭이 사진 비율에서
  나오므로 CSS만으로는 표현할 수 없다.
- `.center`는 `position: absolute; inset: 0` 풀스크린 고정. 스크럽은 그 `opacity`만 만진다.
- `TILES` 상수는 슬롯 좌표가 아니라 **컬럼 소속 + 진입 방향 + 구간**만 들고 있는다.
- `slots = TILES.slice(0, tiles.length)` — `TILES`를 그대로 순회하면 타일이 모자랄 때 터진다.
  한쪽 컬럼이 비면(`sL` 또는 `sR`이 0) 대수가 성립하지 않으므로 콜라주 기하만 건너뛴다
  (십자가·카피·포스터는 그대로 동작).
- cleanup에서 인라인 기하를 전부 지운다(브레이크포인트 전환 시 모바일 CSS를 덮지 않게).

### 5.5 `HeroReveal.module.css`

기하는 전부 JS 인라인으로 간다. CSS는 외형(hairline·radius·overflow·will-change)과 초기 상태,
그리고 정적 스택만 정의한다.

### 5.6 모바일 · reduced-motion — 정적 스택 공유

`@media (max-width: 639px), (prefers-reduced-motion: reduce)` — **한 블록을 공유한다.**

세로 스택: 카피 → 포스터 카드(`aspect-ratio: posterAspect` 인라인, 크롭 0) → 사진 4장(좌/우 교차
정렬). `.sticky`는 `position: static` + `max-width: var(--container-max)`.

reduced-motion을 "데스크톱 스크럽의 최종 프레임"(사진 4장만)으로 두지 **않는다.** 그러면 reduced
사용자는 포스터를 **아예 못 본다**(영상은 `<source media>`가 이미 배제한다). 같은 콘텐츠를 전부
보여주는 모바일 스택을 공유하는 게 맞다. JS는 reduced에서 즉시 반환한다 — 스크럽 없는 화면에
절대좌표 격자를 만들 이유가 없다.

모바일은 IntersectionObserver로 1회 슬라이드 인, reduced는 그마저 없이 즉시 완성 상태.

`.center`의 인라인 `aspect-ratio`는 스크럽 모드에서 무시된다(`absolute; inset: 0`이라 폭·높이가
이미 확정) — 정적 스택에서만 발효한다.

## 7. 수용하는 것

**좌·우 컬럼 폭이 다르다.** 480 vs 648. 콜라주를 컨테이너에 꽉 채우면서 크롭도 0으로 하려면 피할 수
없다. 비대칭 컬럼이 만드는 에디토리얼한 리듬의 일부로 받아들인다.

**히어로 미디어가 최종 화면에 남지 않는다.** 스크롤을 다 내리면 흰 캔버스 위 사진 4장만 남는다.
이는 손실이 아니라 교환이다 — 포스터는 풀스크린 구간(0.46–0.66)에서 **전체가** 보이고, 그 대가로
사진이 2배 커진다. 축소 카드는 포스터를 13%만 보여주면서 사진까지 작게 만들었다.

**`collage-1`이 가장 작다.** 넷 중 유일하게 덜 넓은 사진(1.17)이라 좁은 컬럼에 들어간다.
크롭 0의 직접적인 대가다.

검토했으나 채택하지 않은 대안: 중앙 카드 유지 + 사진 확대(폭이 `W ≈ 2.86·H + 11`로 잠겨 불가),
포스터 줌아웃(카드 크기에 맞춰 `transform: scale` — 별개의 연출 결정, 여전히 카드가 폭을 먹는다),
컨테이너 캡 완화("하나의 컨테이너" 원칙 위반), 사진 추가("collage 1~4만" 제약 위반).

스타일로 만들지 않는다는 구현 노트 4항).
