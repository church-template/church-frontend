## 본문

## 📝 현재 문제점

- 추가·수정·삭제·저장 등 같은 의미의 버튼이 화면마다 형식(색·강조·라벨·아이콘)이 제각각이다.
- 삭제 버튼이 화면에 따라 4가지 형태(빨강 확인 버튼·회색·텍스트·맨버튼)로 혼재한다.
- 같은 목록 행의 수정/삭제 액션이 도메인에 따라 회색과 텍스트형으로 갈린다.
- 디자인 시스템 Button을 우회하는 맨버튼/맨링크가 곳곳에 있어 각자 다른 인라인 스타일을 쓴다.
- 아이콘 정책이 분열돼 있다(한 화면만 아이콘+텍스트, 나머지는 텍스트 전용 또는 아이콘 전용).
- 라벨이 흔들린다(예: 역할 추가 / 새 X, 저장 / 변경).
- 다이얼로그 하단 버튼의 순서와 취소 버튼 유무가 화면마다 다르다.

## 🛠️ 해결 방안 / 제안 기능

- 의미를 기준으로 버튼 형식(색·강조·라벨·아이콘·위치)을 고정하는 단일 규칙표를 만든다. CTA 색상은 화면당 하나로 절제한다.
- UX 법칙을 규칙으로 번역해 적용한다: 오컴의 면도날(규칙 최소화), 파레토(핵심 5개 액션 우선), 포스텔의 법칙(같은 의미는 항상 같은 형식), 피크-엔드(파괴적 확인 순간만 강조), 능동적 사용자(자명한 라벨·아이콘), 파킨슨의 법칙(규칙 명문화).
- 의미가 다른 액션은 구분한다: 연결 해제(사진 제거·역할 회수)는 원본·역할이 보존되므로 삭제와 라벨·아이콘을 구분해 유지한다.
- 어드민·콘텐츠·갤러리·마이페이지·인증 등 전 영역에 일괄 적용한다.

## ⚙️ 작업 내용

- 기반: 공용 버튼의 아이콘 단독 형태 확장, 액션 라벨·아이콘을 상수로 중앙화.
- 파괴적 확인 다이얼로그에 취소 버튼을 추가하고 하단 버튼 순서를 통일.
- 어드민 관리(역할·직분·태그·부서·회원·미디어) 버튼 규칙 통일.
- 콘텐츠(주보·설교·공지·일정) 버튼 규칙 통일.
- 갤러리·마이페이지·인증 영역 버튼 규칙 통일.
- 전역 검증(빌드·테스트·시각 점검).
