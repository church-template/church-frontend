## 본문

## 📝 현재 문제점

- 배포 전 12차원 종합 점검(적대적 검증 포함)에서 확정된 should_fix 개선 항목들이 미반영 상태였다.
- TanStack Query 클라이언트에 기본 옵션이 없어 탭 복귀 시 회원 영역 쿼리가 일괄 재요청되는 부담이 있었다.
- 태그 선택 컴포넌트가 로딩/오류 상태를 구분하지 못해 "태그 없음"과 "불러오기 실패"가 동일하게 보였다.
- 일부 히어로 텍스트 색상이 디자인 토큰 대신 인라인 색상값으로 지정되어 토큰 규칙에 어긋났다.
- 어드민 목록 무효화 쿼리키가 컴포넌트마다 하드코딩되어 중앙 규약과 분리되어 있었다.
- 핵심 에러 처리기의 방어 분기 일부가 테스트로 검증되지 않았다.
- 운영 환경변수의 필요성과 주입 시점이 문서로 명확히 안내되지 않았다.

## 🛠️ 해결 방안 / 제안 기능

- 점검에서 확정된 개선 6건을 반영하여 배포 전 코드 품질과 견고성을 끌어올린다.
- 동작 변경 없이 기존 컨벤션을 따르는 범위 안에서 최소 수정으로 처리한다.
- 변경 후 전체 테스트, 타입 체크, 린트로 회귀가 없음을 확인한다.

## ⚙️ 작업 내용

- 회원 영역 쿼리 클라이언트에 기본 옵션을 추가하여 불필요한 재요청을 방지한다.
- 태그 선택 컴포넌트에 로딩 상태와 오류 상태 표시를 분리해 제공한다.
- 히어로 텍스트 색상을 디자인 토큰 참조로 변경한다.
- 어드민 목록 무효화 전용 쿼리키 헬퍼를 추가하고 하드코딩된 키를 일괄 치환한다.
- 핵심 에러 처리기의 미검증 방어 분기를 테스트로 보강한다.
- 운영 환경변수 안내 문구를 보강한다.
