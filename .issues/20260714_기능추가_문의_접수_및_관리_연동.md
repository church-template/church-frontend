## 본문

## 📝 현재 문제점

- 백엔드에 문의 API(문의 등록 · 목록 · 상세 · 완료 처리 · 삭제)가 추가되었으나 프론트에는 대응 화면이 전혀 없다.
- 방문자가 홈페이지에서 교회에 문의를 남길 방법이 없다. 현재 연락 수단은 전화번호와 이메일 주소 표기뿐이다.
- 관리자가 접수된 문의를 확인하고 처리 여부를 기록할 화면이 없다.

## 🛠️ 해결 방안 / 제안 기능

- 공개 문의 접수 폼: 연락처 및 위치 페이지(`/about/location`) 하단에 문의 폼 섹션을 추가한다. 비로그인 방문자도 제출할 수 있다. 이름 · 연락처 · 이메일(선택) · 문의 내용 · 개인정보 수집 동의를 입력받고, 제출에 성공하면 폼 자리에 접수 완료 안내(접수 번호 포함)를 표시한다.
- 문의 관리 화면: 마이페이지 관리 허브에 문의 관리 카드를 추가하고, 전용 화면(`/mypage/manage/inquiries`)에서 문의 관리 권한 보유자만 접근할 수 있게 한다. 목록은 미처리 · 완료 필터와 페이지네이션을 제공하고, 행을 클릭하면 상세 팝업에서 문의 내용을 읽고 완료 처리 · 완료 취소 · 삭제를 할 수 있다.
- 문의 내용은 목록에 포함되지 않으므로, 내용 확인과 처리를 같은 자리(상세 팝업)에서 하도록 한다.
- 설계 스펙: `docs/superpowers/specs/2026-07-14-inquiry-frontend-design.md`

## ⚙️ 작업 내용

- 문의 API 연동 계층 추가 (공개 등록 · 관리자 목록/상세/완료/삭제)
- 공개 문의 폼 섹션 추가 및 연락처 페이지 배치
- 문의 관리 화면(목록 · 필터 · 페이지네이션 · 상세 팝업) 추가
- 문의 관리 권한 라벨 및 관리 허브 카드 등록
- 문의 접수 횟수 제한(과다 제출) 안내 문구 처리
- 디자인 시스템 문서에 신규 컴포넌트 항목 등록
- 단위 테스트 작성 (API 계층 · 공개 폼 · 관리 화면)
