# 🚀 [기능개선][공개페이지][UX] 공지·주보·일정·설교 페이지 이동 시 로딩 스켈레톤 추가

## 📝 현재 문제점

- 공개 페이지(공지·주보·일정·설교)는 링크 클릭 후 서버 응답이 올 때까지 이전 화면이 그대로 멈춰 있어, 클릭이 접수됐는지 알 수 있는 신호가 전혀 없음
- 시스템 피드백이 0.4초를 넘기면 사용자 이탈·혼란이 생기는데(도허티 임계), 특히 주 사용자층인 고령 교인은 반응이 없으면 같은 링크를 다시 누르게 됨
- 폼 제출(버튼 스피너)과 회원 영역 조회(pending 표시)는 피드백이 있으나, 공개 페이지 이동만 비어 있음

## 🛠️ 해결 방안 / 제안 기능

- 각 공개 세그먼트에 로딩 스켈레톤을 추가해 클릭 즉시 해당 콘텐츠 영역에 자리표시가 뜨도록 함 (내비게이션·뒤로가기는 계속 사용 가능)
- 스켈레톤은 실제 페이지와 동일한 반응형 구조(모바일 1열/태블릿 2열/데스크톱 3열, 행 목록)를 미러링해 실제 콘텐츠로 교체될 때 레이아웃이 흔들리지 않도록 함
- 화면 전체를 막는 전역 로딩 모달은 채택하지 않음 (짧은 이동에도 조작을 차단해 오히려 경험 저하)
- 상세 스펙: docs/superpowers/specs/2026-07-17-navigation-loading-feedback-design.md

## ⚙️ 작업 내용

- 공용 스켈레톤 3종 추가 (목록 행 / 카드 그리드 / 상세 본문)
- 공지·주보·일정·설교의 목록 및 상세 세그먼트에 로딩 자리표시 배치
- 단위 테스트 작성 및 네트워크 스로틀·모바일 폭 수동 검증
