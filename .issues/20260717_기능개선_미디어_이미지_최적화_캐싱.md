# 🚀 [기능개선][공개페이지][성능] 백엔드 미디어 이미지 최적화·CDN 캐싱 적용

## 📝 현재 문제점

- 백엔드에서 내려오는 미디어 이미지(설교 썸네일 등)가 원본 크기 그대로 브라우저에 전송됨 — 리사이즈·포맷 변환·CDN 캐싱이 전혀 적용되지 않음
- 원본이 큰 경우 모바일에서 로딩이 느리고 데이터 소모가 커서, 목록 화면(카드 그리드)의 체감 성능이 떨어짐
- 정적 이미지(로고·부서 히어로 등)는 이미 최적화 파이프라인을 타고 있어, 백엔드 미디어 이미지만 구멍으로 남아 있음

## 🛠️ 해결 방안 / 제안 기능

- 프레임워크의 이미지 최적화 파이프라인을 백엔드 미디어 이미지에도 적용해, 기기 화면 폭에 맞는 크기와 최신 포맷으로 변환된 이미지가 CDN에 캐싱되어 내려가도록 함
- 적용 후 목록·상세 화면에서 화질 저하나 레이아웃 변화 없이 전송량만 줄어드는 것을 확인

## ⚙️ 작업 내용

- 외부 미디어 출처 허용 설정 추가
- 미디어 이미지를 사용하는 카드·상세 컴포넌트의 이미지 렌더링 교체
- 교체 후 화질·레이아웃·로딩 속도 확인
