## 제목

🚀 [기능개선][상수][콘텐츠] 프로젝트 템플릿 상수를 실제 교회(은샘교회) 값으로 교체

## 본문

## 📝 현재 문제점

- `src/constants/` 의 콘텐츠 상수가 템플릿 기본값(플레이스홀더) 상태로 남아 있다.
- 예: `CHURCH_DOMAIN = "example.org"`, `CHURCH_ADDRESS = "서울특별시 ○○구 ○○로 00"`, `CHURCH_PHONE = "02-000-0000"`, 오시는 길 교통편 `"지하철 ○○역 0번 출구"` 등.
- 이대로 배포하면 사용자에게 가짜/예시 정보가 그대로 노출된다.
- 실제 교회 정보 참고본(`docs/constants.md` — 은샘교회 기준)이 존재하므로 이를 기준으로 실제 값을 적용해야 한다.

## 🛠️ 해결 방안 / 제안 기능

- `docs/constants.md` 의 실제 교회 정보를 단일 진실로 삼아 `src/constants/` 의 플레이스홀더 값을 실제 값으로 교체한다.
- 콘텐츠 하드코딩 금지 규칙(가이드 12장)에 따라 모든 노출 텍스트는 상수 경로로만 주입한다.
- 히어로/콜라주/부서 히어로 등 미디어 에셋도 실제 이미지·영상으로 교체(또는 교체 경로 정리)한다.

## ⚙️ 작업 내용

- `church.ts` — 교회명, 도메인, 주소, 연락처, 이메일, 지도 임베드/검색 URL, 좌표
- `content.ts` — 오시는 길 교통편, 새가족 안내 등 노출 문구
- `departments.ts` — 사역 부서 정보 및 부서 히어로 이미지 경로
- `terms.ts` — 이용약관·개인정보 처리방침 본문(교회별 실제 문구)
- 히어로/콜라주 미디어 에셋(`/public`) 실제 파일로 교체
- 교체 후 푸터·오시는 길·부서 페이지 노출값 확인
