## 제목

🎨 [디자인][교육부서][학생부] 학생부 페이지(/departments/student) UI/UX 개선

## 본문

## 🖌️ 요청 내용

- `/departments/student` 학생부 상세 페이지의 UI/UX를 개선한다.
- 현재는 공용 부서 상세 템플릿(DeptHero + 인도자 + 설명 마크다운 + 하위부서)을 그대로 사용하는데, 학생부는 인도자·하위부서가 없어 실제로는 히어로와 짧은 설명 두 문단만 노출되어 화면이 비어 보인다.
- 중·고등학생(다음 세대) 부서 성격에 맞게 예배 시간·모임 안내, 활동 소개, 활동 사진 등 정보가 시각적으로 잘 전달되도록 섹션 구성을 보강한다.
- 디자인 시스템 토큰과 페이지 리듬(흰 섹션 ↔ 회색 밴드 ↔ 다크/미디어 밴드)에 맞춘다.

## 🎯 기대 결과

- 히어로 + 텍스트만 있던 빈 화면이, 부서 분위기와 정보를 담은 짜임새 있는 공개 페이지로 개선된다.
- 데스크톱/태블릿/모바일 반응형 레이아웃이 정상 동작한다.
- 기존 재디자인 페이지(교회사진, 예배시간, 소개·비전 등)와 시각적 톤이 통일된다.

## 🧩 설계 방향 (합의됨)

- **새 시각 요소를 발명하지 않는다.** 기존 홈/예배/교회사진 섹션 패턴을 재사용한다.
- **공통 옵션 필드 방식.** `Department` 타입에 섹션별 옵션 필드를 더하고, 페이지는 데이터가 있는 섹션만 조건부 렌더한다. 학생부만 채우고 청년부·예배부 등은 나중에 채우면 동일 레이아웃이 자동 노출된다(재사용성 확보).
- **활동 사진은 갤러리 섹션.** 카테고리 탭 없이 그리드 + 클릭 확대(라이트박스). 콘텐츠는 상수에서 `public/` 정적 에셋으로 주입.

## 🗂️ 데이터 모델 (`src/constants/departments.ts`)

`Department` 인터페이스에 아래 필드를 **모두 옵션**으로 추가한다(기존 부서 무영향).

| 필드 | 형태 | 용도 |
|---|---|---|
| `intro?` | `{ heading, lead }` | 인트로 헤딩 + 리드 문단 |
| `features?` | `{ icon(직렬화 키), title, desc }[]` | 3-up 기능 카드 |
| `info?` | `{ label, value }[]` | 알림 사항(담당자·모임 시간·연락처·모임 장소) |
| `activities?` | `string[]` | 주요 활동 목록 |
| `programs?` | `{ name, desc }[]` | 특별 프로그램 |
| `gallery?` | `{ src, alt }[]` | 활동 사진 |
| `invite?` | `{ heading, body }` | 닫는 초대 문구 |

- `icon`은 직렬화 가능한 **문자열 키**로 두고 컴포넌트에서 lucide 아이콘으로 매핑한다(`MinistryCards`의 `ICONS` 맵과 동일 방식, 상수는 직렬화 가능한 값만 보유).
- 사진 경로는 `public/dept/student/*` 관례의 placeholder로 시작하고 배포 시 실제 사진으로 교체한다(히어로 미디어와 동일 관례).

## 🧱 새 섹션 컴포넌트 (`src/components/departments/`)

각 컴포넌트는 프로젝트 테스트 관례에 맞춰 테스트를 동반한다(vitest, `globals:false` 명시 import, jest-dom 미사용).

| 컴포넌트 | 재사용 패턴 | 밴드(배경) |
|---|---|---|
| `DeptFeatures` | `MinistryCards`(아이콘 32 currentColor + 제목 + 설명, tone 교차) — 상단에 `intro` 헤딩+리드 포함(`WorshipRegular` 패턴) | 흰 |
| `DeptInfo` | `WorshipRegular`의 `Card surface="soft"` 2×2, label(muted)+value(ink) | 회색 |
| `DeptActivities` | notice-row식 헤어라인 목록(lucide 아이콘 currentColor) | 흰 |
| `DeptPrograms` | `Card` + tone 교차 2×2(이름 + 설명) | 회색 |
| `DeptGallery` | `PhotoLightbox` 재사용(탭 없이 그리드 + 확대 모달, 키보드·좌우 이동) | 미디어 |
| `DeptInvite` | surface-soft 초대 문구(전화·시간 중복 제거, 메시지만) | 회색 |

## 🧬 페이지 조립 (`src/app/departments/[slug]/page.tsx`)

DeptHero → 기존 설명(인도자/마크다운) → 아래 섹션을 **삼항 조건부**(`{cond ? <X/> : null}`)로 순차 렌더 → 기존 `CtaBand` → `SiteFooter`.

```
DeptHero
(기존) DeptLeader? + description?
DeptFeatures?   ← 흰
DeptInfo?       ← 회색
DeptActivities? ← 흰
DeptPrograms?   ← 회색
DeptGallery?    ← 미디어
DeptInvite?     ← 회색
(기존) SubDepartments?
CtaBand         ← 다크 프리푸터
SiteFooter
```

흰 ↔ 회색 ↔ 흰 ↔ 회색 ↔ 미디어 ↔ 회색 ↔ 다크의 밴드 리듬을 유지한다.

## 📝 학생부 콘텐츠(상수 주입안)

- **인트로**: "중·고등학생들이 하나님의 사랑 안에서 건강하게 성장할 수 있도록 돕는 사역 / 또래 친구들과 함께 신앙을 배우고 나누며 꿈과 비전을 발견"
- **기능 카드 3**: 청소년 신앙 교육(성경 중심 체계적 신앙 교육) · 또래 교제(건전한 친구 관계·공동체 의식) · 창의적 활동(재미있는 활동을 통한 전인적 성장)
- **알림 사항**: 담당자=학생부 담당 선생님 / 모임 시간=매주 토요일 오전 11시(학생·청년예배) / 연락처=041-337-2298 / 모임 장소=은샘침례교회 학생부실
- **주요 활동**: 토요일 학생 예배 및 성경공부 / 여름·겨울 수련회 참가 / 찬양과 율동 활동 / 배드민턴 등 재미있는 활동
- **특별 프로그램**: 여름 수련회 · 겨울 수련회 · 찬양 경연대회 · 배드민턴 프로그램(각 설명 포함)
- **닫는 초대**: "학생부에서 함께해요" + 새로운 친구·활동을 통해 신앙을 배워가자는 초대 문구
- **활동 사진**: 추후 몇 장 추가 예정(placeholder로 시작).

### 콘텐츠 정리 규칙

- **🎉 등 UI 이모지 전부 제거**(하드 룰).
- 담당자·모임 시간이 원문에서 히어로·알림·닫는문구에 **3중 중복** → **알림 사항 1회**로 통합, 닫는 문구는 초대 메시지만 남긴다.
- 주요 활동(정기)과 특별 프로그램(계절 이벤트)이 수련회·배드민턴에서 겹치나, 성격이 달라 편집상 구분을 유지한다.

## ✅ 준수 사항 (하드 룰)

- 사용자 노출 텍스트·이미지는 하드코딩 금지 → 상수(`departments.ts`) 주입, 교회별 교체 용이.
- 텍스트 스타일은 `typo.*` 상수만 사용, hex·px 인라인 금지(토큰 참조), arbitrary value 금지.
- 아이콘은 `lucide-react`만(색 `currentColor`, 크기 `size`).
- JSX 조건부는 삼항(`{cond ? <X/> : null}`).
- UI 이모지 금지.
