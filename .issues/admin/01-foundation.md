## 제목

⚙️ [기능추가][어드민][기반] 어드민 공용 인프라 — 권한 게이트·관리 화면 토대

## 본문

## 📝 현재 문제점

- 관리자 기능을 붙이려면 모든 도메인이 공통으로 쓸 토대가 필요하다. 권한에 따라 액션을 보여주는 장치, 관리용 목록 표, 입력 폼, 파일 업로더, 삭제 확인창, 동시 수정 충돌 안내가 전부 미구현이다. 각 도메인이 따로 만들면 중복과 불일치가 생긴다.
- 관리자가 들어갈 입구(마이페이지 관리 허브)와 운영 전용 화면 경로도 없다.

## 🛠️ 해결 방안 / 제안 기능

- 어드민 전 도메인이 공유하는 공통 토대를 먼저 만든다. 이 이슈 자체에는 도메인 기능이 없고, 02 이후 도메인 이슈가 이 토대를 사용한다.
- 진입 모델(공개 화면 인라인 액션, 마이페이지 관리 허브, 운영 전용 화면)과 권한 기준(내 정보 실시간 조회)을 확립한다.

## ⚙️ 작업 내용

- 권한 게이트 컴포넌트와 훅 — 권한 보유 시에만 액션 노출, 로딩 중 깜빡임 방지
- 어드민 쓰기 공통 처리 — 성공 시 관리 목록·상세 즉시 갱신, 공개 화면 반영 지연 안내
- 공용 목록 표 — 정렬·페이지네이션·행 액션
- 공용 입력 폼 — 마크다운 에디터(작성/미리보기 탭), 태그 다중 선택, 날짜·시간 선택, 파일 업로더
- 삭제 확인창, 동시 수정 충돌 안내, 오류 메시지 매핑 확장
- 마이페이지 관리 허브와 운영 도메인 전용 화면 경로 골격
- 디자인 시스템 문서에 신규 공용 컴포넌트 등록
