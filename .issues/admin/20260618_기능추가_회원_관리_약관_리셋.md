## 제목

⚙️ [기능추가][어드민][회원·약관] 회원 관리 및 약관 일괄 리셋

## 본문

## 📝 현재 문제점

- 운영자가 교인 목록을 보고 가입자를 승인하거나 회원 정보를 다룰 화면이 없다. 거버넌스에서 가장 민감한 영역이다.
- 가입자 승인(역할 부여), 역할 회수, 비밀번호 초기화, 약관 재동의 유도를 화면에서 할 수 없다.

## 🛠️ 해결 방안 / 제안 기능

- 관리 허브 하위 전용 화면에서 회원 목록·상세, 회원 정보 수정, 역할 부여·회수, 비밀번호 초기화를 제공한다.
- 역할 부여가 곧 교인 승인이다. 가입자에게 회원 역할을 부여해 승인하고, 회수로 되돌린다.
- 약관·개인정보 동의를 대상별로 일괄 리셋해 다음 로그인 시 재동의를 유도한다. 되돌릴 수 없는 작업이라 강한 확인창을 둔다.
- 상위 권한·자기 자신에 대한 변경은 미리 막는다. 화면 가드는 보조 수단이며 최종 방어는 서버가 한다.

## ⚙️ 작업 내용

- 회원 목록 — 페이지네이션 제공. 백엔드 검색 파라미터가 없어 검색은 페이지네이션·정렬 기준으로 제공하고, 필요 시 현재 페이지 내 필터로 한정한다.
- 회원 상세 조회 — 역할·권한·동의 상태·승인 여부 표시
- 회원 정보 수정 — 이름·전화·이메일
- 역할 부여·회수 — 회원 역할 부여가 곧 교인 승인, 부여 가능 역할 목록은 역할·권한 관리 화면의 목록을 재사용
- 비밀번호 초기화 — 임시 비밀번호를 1회만 표시하고 화면을 벗어나면 잔류하지 않게 처리
- 약관 일괄 리셋 — 대상 선택(약관/개인정보), 강한 확인창, 재동의 유도
- 위계 가드 — 내 최대 우선순위를 넘는 역할 부여·회수, 자기 자신 대상 변경은 버튼 비활성
- 권한 게이트 — 회원 조회·정보수정·비번초기화·약관리셋은 회원 관리 권한, 역할 부여·회수는 역할·권한 관리 권한으로 분리 적용
- 의존 — 역할·권한 관리(선행 이슈)의 역할 목록 토대를 재사용한다
