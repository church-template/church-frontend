## 제목

⚙️ [기능추가][어드민][역할·권한] 역할·권한 관리 화면

## 본문

## 📝 현재 문제점

- 운영자가 역할과 권한을 화면에서 다룰 방법이 없다. 역할 묶음과 역할별 권한 구성은 백엔드 API만 열려 있고 프론트 연동이 0%다.
- 회원에게 역할을 부여하려면 부여 가능한 역할 목록이 화면에 먼저 있어야 하는데, 그 토대가 없다. (회원 관리의 선행 조건)
- 거버넌스(#07)에서 가장 먼저 굳혀야 하는 영역이다. 비파괴적이라 회원 관리보다 위험이 낮다.

## 🛠️ 해결 방안 / 제안 기능

- 관리 허브 하위 전용 화면에서 역할 목록 조회, 역할 추가·수정·삭제, 역할별 권한 일괄 설정을 제공한다.
- 권한 카탈로그를 불러와 역할마다 보유 권한을 켜고 끄는 형태(권한 매트릭스)로 설정한다.
- 상위 위계 역할과 시스템 역할은 편집·삭제 버튼을 미리 막아 실수와 권한 상승을 방지한다. 단, 화면 가드는 보조 수단이며 최종 방어는 서버가 한다.

## ⚙️ 작업 내용

- 역할 목록 조회 — 우선순위 위계와 시스템 역할 여부를 함께 표시
- 역할 추가·수정·삭제
- 역할별 권한 일괄 설정 — 권한 카탈로그 기반 체크 매트릭스로 보유 권한 교체
- 위계 가드 — 내 최대 우선순위를 넘는 역할, 시스템 역할은 편집·삭제·권한변경 비활성
- 사용 중인 역할 삭제 차단 안내
- 권한 게이트 적용 및 관리 허브 카드 자동 노출
- 공개 소비자가 없으므로 관리 화면만 즉시 갱신하고 공개 페이지 무효화는 수행하지 않는다
