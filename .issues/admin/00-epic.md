## 본문

## 📝 현재 문제점

- 1차 배치(T1~T16)는 공개 페이지와 회원 자가 기능만 연동했고 관리자 영역은 의도적으로 제외했다.
- 그 결과 백엔드 관리자 API(쓰기 약 45개, 14개 운영 도메인)는 열려 있으나 프론트 연동이 0%다. 권한을 가진 운영자가 설교·공지·일정·부서·갤러리·주보·미디어·태그·직분·회원·역할·권한·약관을 화면에서 직접 다룰 방법이 없다.

## 🛠️ 해결 방안 / 제안 기능

- 관리자 기능 전체를 연동한다. 별도 관리자 사이트를 만들지 않고 세 갈래로 분산한다.
  - 공개 페이지가 있는 도메인은 권한 보유 시 그 목록·상세에 등록·수정·삭제 버튼을 인라인으로 노출한다.
  - 마이페이지에 권한 보유 도메인을 한눈에 모으는 관리 허브를 둔다.
  - 공개 화면이 없는 운영 도메인은 관리 허브 하위 전용 화면으로 둔다.
- 권한 표시 기준은 로그인 토큰이 아니라 내 정보 실시간 조회를 단일 기준으로 한다.

## ⚙️ 작업 내용

- 하위 이슈로 분할한다(권장 진행 순서):
  - 01 어드민 기반 인프라 (권한 게이트·공용 컴포넌트·관리 허브 토대)
  - 02 콘텐츠 관리 (설교·공지)
  - 03 일정 관리
  - 05 미디어 기반 (갤러리·주보·미디어 라이브러리)
  - 04 부서 관리
  - 06 분류·운영 소도메인 (태그·직분)
  - 07 회원·권한 거버넌스
- 의존: 01(기반)이 모든 하위 이슈의 선행이며, 02~07은 01 이후 병렬 진행이 가능하다.
- 공통 규칙: 어드민 쓰기는 성공 시 관리 화면을 즉시 갱신하되 공개 페이지 반영은 지연될 수 있음을 안내한다. 동시 수정 충돌·오류 메시지·권한 게이트·디자인 토큰 공유 원칙을 모든 하위 이슈가 따른다.
