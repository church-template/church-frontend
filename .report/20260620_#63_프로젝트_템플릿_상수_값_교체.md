# 프로젝트 템플릿 상수 값 교체 (#63)

### 📌 작업 개요

샘플/플레이스홀더로 채워져 있던 프론트 콘텐츠 상수를 실제 교회(은샘교회)의 확정 정보로 교체했다.
교회 고유값(이름·도메인·주소·연락처·히어로 카피), 정적 페이지 콘텐츠(소개·연혁·목회자 인사말·교회 사진·예배시간),
사역 부서, 사이트 정보구조(IA)를 모두 실제 값으로 채우고, 새 콘텐츠를 노출할 정적 페이지 3종을 추가했다.
콘텐츠는 모두 상수에서 주입하며 컴포넌트 하드코딩은 두지 않았다(가이드 12장).

### 🎯 구현 목표

- 교회 고유값 상수(`church.ts`)를 실제 교회 정보로 교체 + 정식 명칭 상수 추가
- 정적 콘텐츠 상수(`content.ts`)에 소개·비전·연혁·목회자 인사말·예배시간·교회 사진 데이터 채움
- 사역 부서(`departments.ts`)를 실제 5개 부서 flat 구조로 재구성
- 사이트 IA(`navigation.ts`)의 "교회안내" 메뉴를 실제 페이지 구성으로 재배열
- 신규 페이지(소개·목회자 인사말·교회 사진) 추가 및 푸터 레이아웃 정리
- 관련 테스트를 새 상수/구조에 맞춰 동기화

### ✅ 구현 내용

#### 교회 고유값 상수 교체
- **파일**: `src/constants/church.ts`
- **변경 내용**:
  - `CHURCH_NAME` = "은샘교회"(약칭), 신규 `CHURCH_NAME_FULL` = "기독교 한국침례회 은샘교회"(정식 명칭) 추가
  - `CHURCH_DOMAIN` = "eunsaem.com"(실제 도메인 값만 정확히 보관, DNS 연결은 별개)
  - 주소·전화·이메일을 실제 값으로 교체(`CHURCH_ADDRESS`·`CHURCH_PHONE`·`CHURCH_EMAIL`)
  - `HERO_CAPTION`을 실제 교회 카피("함께함이 축복이 되는 교회" 등)로 교체
  - `MAP_EMBED_SRC`를 빈 문자열로 설정(네이버 지도 임베드 제거) → 오시는 길 페이지가 외부 지도 링크로 폴백
- **이유**: 약칭/정식 명칭을 분리해 헤더·푸터 로고는 약칭, 저작권·약관 등 공식 표기는 정식 명칭을 쓰도록 함

#### 정적 페이지 콘텐츠 상수 채움
- **파일**: `src/constants/content.ts`
- **변경 내용**:

  - `HISTORY`: 연혁 7개 항목(2011~2025)을 `year`·`text`·`desc`에 더해 상세 UI용 `details`·`significance`까지 포함해 구성

  - `VISION`·`LOCATION`·`WORSHIP_SERVICES`(예배시간 4종)를 실제 값으로 정리
- **이유**: 사용자 노출 텍스트를 단일 상수 파일에서 주입(배포 시 이 파일 교체만으로 다른 교회에 재사용)

### 🔧 주요 변경사항 상세

#### 지도 임베드 제거와 폴백
`MAP_EMBED_SRC`를 빈 문자열로 두면 오시는 길 페이지(`/about/location`)가 iframe 대신
"지도에서 보기" 외부 카카오맵 검색 링크(`mapSearchUrl`)로 폴백한다. 임베드가 설정된 배포
환경의 iframe 분기는 `location/page.embed.test.tsx`에서 모듈 모킹으로 별도 커버한다.

#### 연혁 상세 데이터 보존
`HISTORY.items`에 `details`(상세 항목 배열)·`significance`(의의) 필드를 함께 담았으나,
현재 연혁 페이지(`/about/history`)는 `year`·`text`만 목록으로 렌더한다. 상세 필드는 추후
상세 UI를 위해 데이터로만 보존한다.

#### 로고 상징색
`ABOUT.symbolism`의 색(파랑/초록/빨강/주황)은 신학적 상징(콘텐츠)이며 UI 브랜드 팔레트와
무관하다. 소개 페이지는 현재 텍스트 카드로만 노출하고 색 연출은 보류했다.

**특이사항**:
- 사역 부서·네비게이션 slug 일치를 `navigation.test.ts`가 드리프트 감시로 검증한다.
- 신규 페이지·푸터 모두 콘텐츠를 상수에서 주입해 컴포넌트 내 문자열 인라인이 없다.

### 📌 참고사항

- **연혁 상세 UI 미구현**: `details`·`significance`는 데이터로만 보존(상세 화면 추후).
- **상징색 색 연출 보류**: 현재 텍스트 노출만, 색을 활용한 시각 연출은 후속.
- **`/about/vision` 페이지 보존**: 비전 페이지는 라우트로 남아 있으나 "교회안내" 메뉴에서는
  제외(소개 페이지에 비전 카피가 통합됨). 라우트 자체는 삭제하지 않음.
- **도메인 DNS는 별개**: `CHURCH_DOMAIN`은 값만 정확히 보관하며 실제 도메인 연결과는 무관.
  현재 코드에서 canonical/og 등 URL 생성에는 미사용(추후 SEO 메타 도입 시 사용).
- **연락처 마스킹**: 본 보고서에서는 교회 대표 전화·이메일을 마스킹 처리했다(원본 상수에는
  실제 값 입력됨). 예: 이메일 `user@example.com`, 전화 `010-1234-5678` 형태로 표기.
