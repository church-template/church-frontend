# 성경통독 챌린지 프론트엔드 연동 (#88)

### 📌 작업 개요

백엔드에 추가된 성경통독 챌린지 API 9종을 프론트 UI와 연결. 교인이 챌린지를 조회·참여하고 매일 읽기를 기록하는 회원 영역과, 관리자가 챌린지를 개설·수정·삭제하는 어드민 화면까지 도메인 전 흐름을 구현.

고령 사용자층을 고려해 "오늘 할 일" 다크 밴드 + 초대형 타이포(C-2 디자인), 일상어 문구, 벽걸이 달력식 기록 UI로 설계. 전 API가 회원 전용(`CHALLENGE_PARTICIPATE` / 어드민 `CHALLENGE_MANAGE`)이라 갤러리 패턴(게이트 + TanStack Query 클라이언트 컴포넌트)으로 구현, ISR 없음.

- 설계 스펙: `docs/superpowers/specs/2026-07-07-bible-challenge-frontend-design.md`
- 구현 계획: `docs/superpowers/plans/2026-07-07-bible-challenge-frontend.md`

### 🎯 구현 목표

- 공개/회원: 목록·상세·참여·읽기 기록/취소·내 진행 대시보드·달력 히트맵·내 참여 이력
- 관리자: 챌린지 개설(POST)·수정(PATCH)·삭제(DELETE)
- 진입점: GNB 예배·설교 그룹 + 마이페이지 이력 섹션

### ✅ 구현 내용

#### 1. 성경 구조 상수 + API 클라이언트

- **파일**: `src/constants/bible.ts`, `src/lib/api/challenges.ts`, `src/lib/api/challenges.admin.ts`, `src/lib/api/types.ts`
- **변경 내용**: 개신교 정경 66권 1189장 상수(권 이름·장 수·누적합) + 헬퍼(`locate`·`chapterCount`·`dailyGoalOf`·`challengeEndDate`·`formatRange`). 회원 GET/POST/DELETE 함수 8종, 어드민 쓰기 3종.
- **이유**: 백엔드 `BibleStructure`와 동일 숫자를 프론트에도 두어 범위 표기·어드민 미리보기를 서버 호출 없이 계산. 상수 스냅샷(구약 929·신약 260·전체 1189장)을 테스트로 드리프트 감시.

#### 2. 네비게이션·권한 라벨·디자인 토큰

- **파일**: `src/constants/navigation.ts`, `src/components/shell/MegaMenu.tsx`, `src/constants/permissions.ts`, `src/app/globals.css`, `.claude/rules/DESIGN.md`
- **변경 내용**: 예배·설교 그룹에 "성경통독"(`/challenges`, `bookOpenCheck` 아이콘) 링크. 권한 한글 라벨 2건. 다크 밴드 액센트 토큰 `--color-primary-on-dark: #4d82ff`. DESIGN.md에 컴포넌트 3항목 등록.
- **이유**: `#0052ff`는 다크 배경 위 대비가 부족해 밴드 전용 액센트 토큰을 분리(단일 액센트 원칙의 문서화된 예외).

#### 3. 회원 게이트 + 쿼리·뮤테이션 훅

- **파일**: `src/components/challenges/ChallengeGate.tsx`, `src/components/challenges/queries.ts`
- **변경 내용**: 갤러리 게이트 동형(하이드레이션→비로그인 안내→권한 안내→통과). 쿼리 훅 5종 + 뮤테이션 3종.
- **이유**: 권한 없으면 children을 마운트하지 않아 API 호출 0회. join/read/취소가 모두 `MyProgressResponse`를 반환하므로 `setQueryData`로 즉시 반영 + logs·detail·participations만 무효화(낙관적 업데이트 불필요).

#### 4. 상세 대시보드 (C-2 다크 몰입)

- **파일**: `src/components/challenges/TodayBand.tsx`, `ReadingCalendar.tsx`, `ReadDialog.tsx`, `ChallengeDetail.tsx`, `src/app/(site)/challenges/[id]/page.tsx`
- **변경 내용**: 오늘의 통독 다크 밴드(초대형 타이포 5상태: 기록 전/후·UPCOMING·ENDED·완독), 벽걸이 달력(읽은 날 ✓+장 수, 셀 탭=기록/취소 입구), 장 수 입력 다이얼로그.
- **이유**: 고령 사용자 동선 — "다 읽었어요" 버튼 한 번이 핵심. 스트릭→"N일 연속으로 읽고 있어요", 페이스→"목표보다 N일 빨라요" 등 일상어만 사용.

- **이유**: 매일 오는 사용자는 클릭 한 번으로 대시보드 도달.

#### 7. 어드민 CRUD

- **파일**: `src/components/admin/challenges/ChallengeFormDialog.tsx`, `ChallengeManager.tsx`, `schema.ts`, `src/app/(site)/mypage/manage/challenges/page.tsx`, `src/lib/admin/manageDomains.ts`
- **변경 내용**: 개설·수정 폼(범위 프리셋 전체/구약/신약, "총 N장·하루 N장·종료일" 실시간 미리보기, 낙관락 version 시드), DataTable 관리 화면, 관리 허브 카드 등록.

### 🔧 주요 변경사항 상세 (리뷰 반영)

다단계 코드 리뷰에서 다음을 근본 수정:

- **어드민 PATCH dirty 전송**: 백엔드가 구간·기간 변경을 필드 존재(presence)로 판정하므로, 참여자 있는 챌린지의 제목 수정조차 400이 나던 문제를 RHF `dirtyFields` 기반 body 구성으로 해결.
- **자정 경계 갱신**: 전역 `refetchOnWindowFocus: false`라 동작하지 않던 부분을 `useMyProgress`·`useMyLogs` 쿼리 단위 오버라이드로 실동작화(탭 며칠 열어두는 고령 사용자 대비).
- **목록 피처 페이지네이션 취약점**: 참여 중 챌린지의 시작 권을 목록 페이지에서 찾다가 다른 페이지면 창세기로 오표시되던 것을, 상세 조회에서 소싱하도록 수정.

**특이사항**:
- `max-w-md` 등 t-shirt 폭 클래스가 프로젝트 spacing 토큰과 충돌 → `max-w-[var(--container-modal)]` 사용
- 네이티브 number input `min`/`step`이 jsdom에서 submit을 삼켜 zod 단독 검증으로 통일
- 상세 이탈·후속 항목은 스펙 문서 §9(구현 노트)에 기록
