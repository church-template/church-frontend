# [#2] 프로젝트 셋업 보강

### 📌 작업 개요
교회 홈페이지 프론트엔드(Next.js 16) 프로젝트 골격 점검 및 누락분 보강. create-next-app 스캐폴드가 이미 존재하여 신규 생성 대신 **검증 + 환경/상수/규약 추가**로 마무리.

### 🎯 구현 목표
- 기존 스캐폴드(App Router·TS·Tailwind v4·`@/*` alias·Turbopack) 검증
- 환경값과 교회 고유값의 분리 정착 (env 최소화, 매직스트링 제거)
- 데이터 패칭 경계 규약 문서화

### ✅ 구현 내용

#### env 최소화 — 환경값만 분리
- **파일**: `.env.example`, `.gitignore`
- **변경 내용**: `NEXT_PUBLIC_API_BASE`(백엔드 base URL) 한 항목만 env로 둠. `.gitignore`의 `.env*` 무시에 `!.env.example` negation 추가하여 템플릿만 추적 가능하게 함
- **이유**: 환경(dev/prod)마다 바뀌는 값만 env. 교회 고유값까지 env에 두던 초기안은 과한 설정이라 제거

#### 교회 상수 모듈 — 매직스트링 제거
- **파일**: `src/constants/church.ts`, `src/hero/types.ts`
- **변경 내용**: 교회명·도메인·히어로 미디어·카피를 상수로 모음. 히어로 미디어 타입(`HeroMedia` 판별 유니온)을 별도 모듈로 분리해 메인/부서 히어로가 공유하도록 함
- **이유**: "환경"이 아니라 "교회"마다 다른 값이라 env가 아닌 상수가 적합. 컴포넌트가 import하여 문자열 인라인 하드코딩을 방지

#### 개발 규약 문서화
- **파일**: `README.md`
- **변경 내용**: 데이터 패칭 경계(공개=서버 컴포넌트+ISR / 회원=TanStack Query+authFetch), 설정 분리, 토큰 참조 원칙을 "개발 규약" 섹션으로 추가
- **이유**: 후속 태스크가 공유할 아키텍처 기준을 코드 진입 전에 명시

### 🔧 주요 변경사항 상세

#### src/constants/church.ts
교회명(은샘교회)·도메인·히어로 배경(정적 에셋 경로)·카피를 상수로 노출. 카피는 줄 단위 배열로 두어 `\n` 이스케이프 불필요.

**특이사항**:
- 히어로 영상은 정적 에셋 방식 채택. 백엔드 미디어 서빙이 Range(부분 요청)를 지원하지 않아 스트리밍/seek에 부적합하기 때문
- `HeroMedia` 타입은 `src/hero/types.ts`에 단일 정의 → 메인 히어로·부서 히어로가 공유
