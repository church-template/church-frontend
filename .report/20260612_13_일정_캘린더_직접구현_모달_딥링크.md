# 일정 캘린더 (#13)

### 📌 작업 개요

읽기 전용 일정 캘린더(`/events`)를 외부 캘린더 라이브러리 없이 **직접 구현**하고, 일정 상세를 **모달 + 딥링크(`/events/{id}`)**로 연결. 데스크톱은 월 그리드, 모바일(<1024px)은 날짜그룹 목록으로 같은 데이터를 다르게 표시. 공개 페이지이므로 서버 컴포넌트 + ISR로 패칭하고, KST 날짜 계산을 런타임 TZ에 흔들리지 않게 직접 처리.

**참조**: 가이드 15.3·10장·13.2, OpenAPI `/api/events`·`/api/events/{id}` / 설계·계획 문서: `docs/superpowers/specs/2026-06-12-event-calendar-design.md`, `docs/superpowers/plans/2026-06-12-event-calendar.md`

---

### 🎯 구현 목표

- 목록 `/events`: `GET /api/events?year&month&size=200`(쌍 필수)·`tagId` — 월 그리드/모바일 목록, 태그 필터, 월 네비게이션
- 상세: 칩 클릭 → 모달(클라 fetch) + 딥링크 `/events/{id}`(`GET /api/events/{id}`, description 마크다운)
- 패칭 경계: 서버 컴포넌트 `fetch` + ISR(60s). TanStack 미사용(회원 영역 전용 규칙)
- 일정 CRUD(어드민)는 범위 밖

- **변경 내용**: `getEvents`/`getEvent`/`buildEventQuery`/`EVENTS_PAGE_SIZE`(200) 추가. `EventDetailResponse` 타입 정의

#### KST 캘린더 코어
- **파일**: `src/lib/calendar.ts`(신규)
- **변경 내용**: 민간 날짜 격자(`monthMatrix`)·이벤트 버킷팅(`bucketEvents`)·모델 빌더(`buildCalendarModel`)·`resolveMonth`·`formatAllDayRange` 등 순수 함수
- **이유**: 서버 컴포넌트(프로덕션 UTC)에서 셀을 계산하므로, 런타임 로컬 TZ에 의존하는 라이브러리 대신 `Date.UTC` + Intl(Asia/Seoul)만으로 KST 벽시계 날짜를 산출

#### 표시 컴포넌트
- **파일**: `src/components/events/EventChip.tsx`·`EventDayPopover.tsx`·`EventDetailView.tsx`·`EventDetailModal.tsx`·`EventCalendar.tsx`(전부 신규)
- **변경 내용**: 칩(allDay 시간 생략)·"+n" 더보기 Popover·상세 본문(모달/딥링크 공유)·Dialog 모달(description 클라 fetch)·그리드+모바일 목록+월 네비+모달 오케스트레이션
- **이유**: 동작 컴포넌트(Dialog·Popover)는 기존 `components/ui/` 래퍼 재사용, 상세 표시는 `EventDetailView` 한 곳으로 단일화해 모달·딥링크 중복 제거

#### 페이지
- **파일**: `src/app/(site)/events/page.tsx`·`src/app/(site)/events/[id]/page.tsx`(신규)
- **변경 내용**: 목록 페이지는 `searchParams`(year·month·tagId) 파싱 → 일정·태그 병렬 fetch → KST 모델 산출 → `TagFilter`+`EventCalendar`. 딥링크는 id 검증 → `getEvent` → `EventDetailView`
- **이유**: 설교·공지 목록 패턴 미러링(서버 컴포넌트 + URL 구동 필터), 월 이동도 `<Link>`로 URL 교체해 북마크·공유 가능

#### 기존 자산 재사용 수정
- **파일**: `src/components/cards/EventCard.tsx`·`src/lib/date.ts`(수정)
- **변경 내용**: `EventCard`의 `date`를 optional로(없으면 날짜 배지 생략) → 모바일 목록에 재사용. `date.ts`에 `formatClockTime`(KST "HH:mm") 추가
- **이유**: 모바일 목록은 날짜를 그룹 헤더가 담당하므로 카드 배지 중복 제거. 신규 축약 카드를 만들지 않고 기존 컴포넌트 재사용

### 🔧 주요 변경사항 상세

#### KST 셀 계산 — 런타임 TZ에 안전
서버에서 월 격자와 이벤트→셀 매핑을 1회 계산해 직렬화 모델로 클라에 전달한다. 셀 판정은 Date의 로컬-TZ 메서드 대신 `Date.UTC`(요일·격자 산술)와 Intl Asia/Seoul(인스턴트→KST 민간 날짜)만 사용해, 프로덕션/CI(UTC)에서도 동일 결과를 보장하고 하이드레이션 불일치를 없앤다.

**특이사항**:
- `end_at` 배타 경계: 종료 1ms 전의 민간 날짜(`lastC`)를 마지막 점유일로 계산 → 종료가 자정이면 그날은 자동 제외(예: `end=2026-07-01T00:00` → 6월 30일까지)
- 점 이벤트(`endAt=null`)는 시작일 셀만, 기간 이벤트는 시작~종료 모든 셀(달 경계 OVERLAP 포함)에 칩 반복
- 퇴화 입력(`end≤start`)·Invalid 입력(offset 접미사 등)은 방어 처리(시작일만/버킷 스킵)

#### 상세 모달 — 클라 fetch
칩 클릭 시 카드 데이터로 제목을 즉시 띄우고, description은 `GET /api/events/{id}`를 클라에서 직접 fetch해 마크다운 렌더. 공개 단건이라 인증이 필요 없어 TanStack 없이 처리. 딥링크 페이지와 동일한 `EventDetailView`로 표시를 단일화.

**특이사항**:
- 모달 클라 fetch는 `NEXT_PUBLIC_API_BASE`가 클라 번들에 인라인돼야 도달(미설정 시 동일 오리진 404) — 운영 env 필수
- effect 내 동기 `setState`(린트 `react-hooks/set-state-in-effect`) 회피: 이전 이벤트 데이터가 남아도 id 비교로 표시 여부를 파생, 불일치면 Skeleton 유지

#### date-fns 미도입 결정
스택 표는 "date-fns로 셀 계산"을 명시하나, date-fns 캘린더 함수는 런타임 로컬 TZ로 동작해 서버 버킷팅에서 KST 경계가 어긋난다. 더 정확하고 단순한 직접 구현을 택하고 의존성을 추가하지 않음(`docs/.../event-calendar-design.md` §9에 사유 기록).

### 📌 참고사항

- **백엔드 확인 필요(1건)**: all-day 다일 이벤트의 `end_at` 인코딩(이틀 행사 → `end`가 `06-16T00:00`인지). 현재는 "배타(익일 자정)" 가정으로 구현했고, 실 데이터로 어긋나면 `bucketEvents`·`formatAllDayRange`의 `lastC` 산출 한 곳만 조정하면 됨(설계 §13 A1)
- 일정 등록·수정·삭제(어드민 `/api/admin/events`)는 이번 범위 밖
- 기간 이벤트 가로 연결 바, 태그 다중선택, 월 200건 초과 페이지네이션은 후속 과제
