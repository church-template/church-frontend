# T10 설교 페이지 (목록·검색·상세) 구현 보고서

### 📌 작업 개요

공개 설교 페이지를 구현했다. 목록(`/sermons`, 검색·태그필터·페이지네이션)과 상세(`/sermons/[id]`, 영상·메타·마크다운 본문)를 Next.js 서버 컴포넌트로 작성했다. 필터 UX는 "미니멀 + 클릭 구동"으로, 검색창·태그필·활성칩만 상시 노출하고 설교자·시리즈·태그는 상세 페이지에서 클릭해 목록 필터로 연결한다.

브레인스토밍 → 설계 스펙 → 구현 계획 → TDD 구현 → 타입·보안 2중 리뷰 → 트리아지·수정 → 레이어별 커밋 순으로 진행했다.

### 🎯 구현 목표

- 목록: `GET /api/sermons`(검색 `q`·태그·설교자·시리즈·기간 필터, 페이지네이션) — URL 쿼리와 동기화
- 상세: `GET /api/sermons/{id}`(조회수 +1 → `no-store`) — 마크다운 본문·영상·오디오·작성자
- 데이터 경계: 공개 도메인은 서버 컴포넌트 `fetch`(TanStack Query 미사용)
- 기존 "after" 프로덕션 컴포넌트(SermonCard·Input·Badge·Pagination·TagFilter·EmptyState·MarkdownContent) 재사용

### ✅ 구현 내용

#### 1) 데이터 계층

- **파일**: `src/lib/api/sermons.ts`, `src/lib/api/tags.ts`, `src/lib/api/types.ts`(편집), `src/lib/youtube.ts`, `src/lib/url.ts`
- **변경 내용**:
  - `getSermons(params)` — 목록 fetch, `revalidate: 60`(목록 캐시 가능). `buildSermonQuery`로 `q·preacher·series·from·to·tagId·page·size·sort` 직렬화(공유 `buildListQuery`는 이 필드들을 안 다뤄 전용 빌더 신설).
  - `getSermon(id)` — 상세 fetch, `cache: 'no-store'`. 404는 `null` 반환(호출부에서 `notFound()`), 그 외 에러는 throw.
  - `getTags()` — 공개 태그 목록(`revalidate: 300`), TagFilter용.
  - `types.ts`에 `SermonDetailResponse` 추가(본문·videoUrl·audioUrl·version·작성자).
  - `parseYouTubeId(url)` — `watch?v=`·`youtu.be`·`embed/`에서 11자 id 추출, 그 외 null.
  - `isSafeHttpUrl(url)` — `http(s)`만 허용하는 스킴 화이트리스트(보안 하드닝).

#### 2) 설교 전용 컴포넌트

- **파일**: `src/components/sermons/SermonVideo.tsx`(client), `SermonAudio.tsx`(server), `SermonSearch.tsx`(client), `ActiveFilters.tsx`(client)
- **변경 내용**:
  - `SermonVideo` — 유튜브면 썸네일 facade(가벼움)로 띄우고 클릭 시에만 `iframe` 로드. 그 외 안전한 URL은 "영상 보기" 링크 폴백.
  - `SermonAudio` — 음원 파일 확장자(`.mp3/.m4a/.aac/.ogg/.wav`)면 `<audio>` 플레이어, 아니면 "오디오 듣기" 링크. 네이티브 audio라 서버 컴포넌트.
  - `SermonSearch` — searchPill 입력, 제출 시 `q`만 set/delete·`page` 리셋(기존 쿼리 보존). 외부 URL 변경을 입력값에 반영(아래 특이사항).
  - `ActiveFilters` — `q·설교자·시리즈·기간` 활성 칩 + ✕ 제거(해당 param·page만 드롭). `buildChips` 순수 함수로 분리해 테스트 용이. tagId는 TagFilter가 이미 표시하므로 제외.

#### 3) 목록·상세 페이지

- **파일**: `src/app/(site)/sermons/page.tsx`, `src/app/(site)/sermons/[id]/page.tsx`
- **변경 내용**:
  - 목록 — `searchParams`(이 Next 버전은 Promise → `await`)를 정규화(`toNum`·`toStr`·`toDate`)해 `getSermons`·`getTags` 병렬 fetch. 빈 결과는 EmptyState, 아니면 3-up 카드 그리드 + Pagination(`totalPages>1`).
  - 상세 — `params` await, id 양의 정수 검증 후 `getSermon`. 없으면 `notFound()`. 영상우선 레이아웃(영상 → 제목 → 클릭메타 → 태그 → 오디오 → 마크다운). 설교자·시리즈·태그를 `/sermons?...` 링크로 연결.
  - `(site)` 라우트 그룹에 두어 SiteShell(라이트 헤더·CTA밴드·푸터) 자동 적용. 네비의 `설교 → /sermons` 링크는 기존 존재.

### 🔧 주요 변경사항 상세

#### 동적 렌더링으로 CI 빌드 안전성 확보

목록은 `searchParams` 접근, 상세는 `no-store`라 두 라우트 모두 **동적(ƒ)** 으로 렌더된다. 빌드 결과에서 `/sermons`·`/sermons/[id]`가 `ƒ (Dynamic)`으로 표시되어, 백엔드가 없는 CI 빌드에서 prerender fetch가 발생하지 않는다. 별도 `connection()`·`force-dynamic` 불필요.

#### 검색 입력 동기화 (key 기반 remount)

`SermonSearch`는 `useState`로 입력값을 관리하는데, 활성칩 ✕로 URL의 `q`가 제거돼도 같은 라우트라 컴포넌트가 remount되지 않아 입력값이 남는 데싱크가 있었다. 외부 래퍼가 `currentQ`를 읽어 `<SearchFormInner key={currentQ} initialQ={currentQ} />`로 넘기도록 해, URL의 `q`가 바뀌면 key 변경 → 내부 컴포넌트 remount → `useState` 재초기화로 입력값이 동기화된다. `useEffect`+setState 안티패턴(린트 `react-hooks/set-state-in-effect` 위반)을 피한 해법이다.

#### 검색 안내 문구

검색 placeholder를 **"제목·설교자·시리즈·성경구절"**로 적용했다. OpenAPI 설명("제목/내용")은 코드와 불일치(오기)이며, 백엔드 `q`는 본문(content)을 매칭하지 않는다.

#### 보안 하드닝 (리뷰 반영)

- **외부 URL 스킴 검증**: `videoUrl`/`audioUrl`에 `javascript:`·`data:` 스킴이 저장될 경우의 XSS를 막기 위해 `isSafeHttpUrl`로 `http(s)`만 통과시킨다. 불안전 URL은 렌더하지 않는다.
- **id 양의 정수 가드**: 상세 `id`가 `0`·음수면 백엔드 호출 없이 `notFound()`.
- **날짜 포맷 검증**: `from`/`to`는 `yyyy-MM-dd` 형식만 백엔드로 전달.
- **tabnabbing 방지**: 외부 링크에 `rel="noopener noreferrer"`.

**특이사항**:
- 외부 입력 텍스트·이미지·영상은 API에서 주입(하드코딩 없음). 색·간격은 디자인 토큰만 사용(hex·px 인라인 없음). 아이콘은 lucide-react(뒤로가기 `ChevronLeft` 포함).
- 카드 목록은 상세로 가는 단일 링크라 카드 안에 필터 링크를 중첩하지 않는다(`<a>` 중첩 회피). 클릭 필터는 상세 페이지에 둔다.
