# #52 마이페이지·회원 관리 UI 개선 + 승인 단일 소스 전환

## 📌 작업 개요

마이페이지·어드민 회원 관리 화면에서 발견된 두 가지 문제를 함께 수정했다.

1. **승인 판정 버그**: `ProfileCard`가 `me.roles.includes("MEMBER")`로 교인 승인 여부를 클라이언트에서 직접 계산했다. 어드민·슈퍼어드민 계정은 `MEMBER` 역할 없이도 승인 상태이므로 `manage` 화면의 회원 목록(백엔드 `approved: true`)과 마이페이지 배지("비승인")가 불일치했다. 백엔드 `me.approved` 필드를 단일 소스로 삼아 수정했다.
2. **관리 허브 가독성**: `ManageHub`가 도메인 카드를 단일 목록으로 나열해, 권한이 많을수록 허브가 구별 없이 길어졌다. DESIGN.md 명세(굵기 위계 + 헤어라인 divider)에 따라 테마별 카테고리 섹션으로 재구성했다.

추가로 어드민 `MemberDetailDialog`의 섹션 구분선, `ResetPasswordSection` 버튼 줄바꿈, `MypageContent` 로그아웃 버튼 위치를 정비했다.

---

## ✅ 구현 내용

### 1. `src/lib/auth/types.ts` — `MeResponse` `approved` 필드 추가

`MeResponse` 인터페이스에 `approved: boolean` 필드를 추가했다. 백엔드 OpenAPI `isApproved` 필드의 단일 소스다. 주석으로 "roles 직접 판정 금지 — 어드민은 MEMBER 없이도 승인" 규칙을 명시해 재구현을 방지했다.

### 2. `src/components/mypage/ProfileCard.tsx` — 승인 배지 + 역할 칩

- 승인 판정을 `me.approved`로 대체(`roles.includes("MEMBER")` 제거).
- 배지 텍스트 `"교인"` → `"승인"`(`"승인 대기"`). variant는 `approved ? "primary" : "default"`.
- 승인 배지 옆에 `me.roles` 칩을 나열해 RBAC 역할을 시각적으로 노출했다.

### 3. `src/lib/admin/manageDomains.ts` — 카테고리 모델 추가

`ManageDomain`에 `category: ManageCategory` 필드를 추가하고, 11개 도메인을 `content·media·org·governance` 네 카테고리로 분류했다. `MANAGE_CATEGORIES` 배열이 렌더 순서와 라벨을 단일 소스로 정의한다. 보유 카드 0개인 카테고리는 허브에서 제목째 숨긴다.

```
content    : 설교·공지·일정·태그
media      : 갤러리·주보·미디어
org        : 부서·직분
governance : 회원·역할·권한
```

### 4. `src/components/mypage/ManageHub.tsx` — 카테고리 섹션 렌더

- `MANAGE_CATEGORIES` 순서대로 그룹을 구성, 보유 권한 도메인이 있는 카테고리만 렌더한다.
- 카테고리 제목 `typo.titleSm`(600) ↔ 카드 라벨 `typo.bodyMd`(400) **굵기 대비**로 정보 변경 지점을 드러낸다.
- 첫 그룹을 제외한 카테고리 경계에 `border-t border-hairline`(1px 헤어라인 divider) + `mt-lg pt-lg` 여백을 적용했다.
- 관리 섹션 제목(`h2`) 바로 아래 `border-t-2 border-ink`(2px ink 앵커 divider)를 추가해 제목↔본문 위계를 강조했다. 그림자 단계 추가 없이 헤어라인+무게 위계만으로 처리한다(DESIGN.md 준수).

### 5. `src/components/admin/members/MemberDetailDialog.tsx` — 섹션 divider

- 헤더 메타(승인·직분·가입일) 바로 아래 `border-t-2 border-ink` 앵커 divider를 삽입해 `ManageHub`와 동일한 위계 패턴을 적용했다.
- 프로필 폼 / 역할 / 약관 / 비밀번호 섹션 사이에 `border-t border-hairline`(1px) divider를 추가해 정보가 한 덩어리로 쏟아지지 않도록 했다.

### 6. `src/components/admin/members/ResetPasswordSection.tsx` — 버튼 레이아웃

확인 단계(`confirming`)에서 질문 텍스트와 버튼을 `flex-col gap-sm`으로 분리했다. 기존 한 줄 배치에서는 버튼 텍스트가 세로로 줄바꿈되었다. 초기화·취소 버튼 모두 `shrink-0 whitespace-nowrap`으로 고정 너비를 보장했다.

### 7. `src/components/mypage/MypageContent.tsx` — 로그아웃 버튼

- 하단 행을 `flex items-center justify-between gap-md`로 변경해 탈퇴(`WithdrawDialog`)는 좌측, 로그아웃은 **우측 정렬**했다.
- 로그아웃 버튼 variant를 `"secondary"` → `"destructive"`(빨강)로 변경했다. 세션을 종료하는 비가역 조작이므로 파괴적 시각 처리가 적합하다.

## 🔧 주요 변경사항 상세

### 승인 판정 버그 — 근본 원인

기존 코드:
```ts
const approved = me.roles.includes("MEMBER");
```
이 계산식은 일반 교인 계정에서만 정확하다. 어드민·슈퍼어드민은 `MEMBER` 역할 없이 서버가 `approved: true`를 반환하므로 마이페이지 배지가 "교인"(비승인) 표시로 잘못 나왔다.

수정 후:
```ts
const approved = me.approved; // 백엔드 단일 소스
```

### 카테고리 렌더 로직

```ts
const groups = MANAGE_CATEGORIES
  .map((cat) => ({
    cat,
    domains: MANAGE_DOMAINS.filter(
      (d) => d.category === cat.key && hasPermission(d.permission, me)
    ),
  }))
  .filter((g) => g.domains.length > 0);
```
`MANAGE_CATEGORIES` 배열 순서가 렌더 순서를 결정한다. 필터 결과가 빈 카테고리는 제목째 제외된다.

### divider 위계 패턴

| 위치 | 스타일 | 의미 |
|---|---|---|
| 관리 섹션 제목 하단 | `border-t-2 border-ink` | 제목↔본문 앵커(굵은 기준선) |
| 카테고리 사이 | `border-t border-hairline` | 정보 변경 지점(1px 헤어라인) |
| 다이얼로그 섹션 사이 | `border-t border-hairline` | 섹션 구분(1px 헤어라인) |

## 📌 참고사항

- **승인 판정은 `me.approved` 단일 소스**. 프론트에서 `roles` 배열로 재구현하면 어드민 계정에서 불일치가 재발한다. `types.ts` 주석과 `MeResponse.approved` 필드가 단일 진실을 보장한다.
- `MANAGE_DOMAINS` 신규 도메인 추가 시 반드시 `category` 필드를 지정해야 한다. 누락 시 타입 에러로 컴파일이 차단된다(`ManageCategory` 유니온 강제).
- 로그아웃 `"destructive"` variant는 `button-destructive`(DESIGN.md)를 사용한다. 모달 확정 버튼 외 용도이지만 세션 종료라는 비가역 조작에 한해 예외로 적용했다.
