# 🚀 [기능개선][설교][권한] 설교 목록·상세 회원전용 전환 (SERMON_VIEW)

### 📌 작업 개요

설교 목록·상세를 갤러리와 동일한 회원전용(교인 승인 `MEMBER` 이상)으로 전환.
백엔드는 `SERMON_VIEW` 권한 신설·적용 완료 상태였고(V15 마이그레이션·SecurityConfig), 이번 작업은 프론트 전환분.
갤러리 전용이던 게이트를 공용 `MemberGate`로 일반화하고, 설교 데이터 패칭을 공개 영역(RSC fetch+ISR)에서 회원 영역(TanStack Query + authFetch)으로 이동.

**보고서 파일**: `.report/20260720_#103_설교_회원전용_전환.md`

### 🎯 구현 목표

- 비로그인 사용자 → 설교 진입 시 로그인 안내(`?next=` 복귀 링크 포함)
- 승인 전 회원(`USER`) → 교인 승인 안내
- `MEMBER` 이상 → 기존과 동일한 목록·검색·필터·상세·수정 경험 유지
- 게이트 미통과 시 설교 API 호출 0회 (안내만 표시)
- 메인 페이지 "최신 설교 3"은 티저(제목·설교자·날짜만)로 공개 유지

### ✅ 구현 내용

#### 1. 회원전용 게이트 공용화 — MemberGate
- **파일**: `src/components/common/MemberGate.tsx` (신규), `src/components/gallery/GalleryGate.tsx` (삭제)
- **변경 내용**: GalleryGate 로직을 `permission`·`domainLabel`·`skeleton` props를 받는 공용 컴포넌트로 추출. 갤러리 호출부 4곳(목록·앨범 등록·상세·수정)을 `MemberGate`로 교체
- **이유**: 동일 게이트 로직의 세 번째 복제를 막고(갤러리·설교), 이후 도메인 전환 시 재사용

#### 2. 설교 API 클라이언트 전환
- **파일**: `src/lib/api/sermons.ts`
- **변경 내용**: 서버 fetch(`getSermons`·`getSermon`, ISR revalidate 60/no-store) 제거 → `fetchSermons`·`fetchSermon`(authFetch + parseJson) 추가. 쿼리 빌더·파라미터 타입은 유지
- **이유**: `/api/sermons`가 인증 필요로 바뀌었고, STATELESS 구조상 토큰이 클라이언트에만 있어 서버 컴포넌트 fetch 불가

#### 3. 쿼리 훅 신설
- **파일**: `src/components/sermons/queries.ts` (신규)
- **변경 내용**: `useSermons`(목록, keepPreviousData) · `useSermon`(상세) · `useSermonTags`(공개 태그 재사용). 갤러리 `queries.ts`와 동형
- **이유**: 게이트 통과 후에만 마운트되는 회원 영역 패칭 관례(retry:false, 401 갱신은 authFetch가 담당)

#### 4. 목록 페이지 전환
- **파일**: `src/app/(site)/sermons/page.tsx`, `src/components/sermons/SermonList.tsx` (신규), `src/app/(site)/sermons/loading.tsx` (삭제)
- **변경 내용**: 페이지는 제목+등록버튼+게이트만 남긴 정적 셸로, 목록·검색·태그필터·페이지네이션은 `SermonList`(클라이언트)가 URL 파라미터를 파싱해 조회. 기존 파라미터 검증(숫자·yyyy-MM-dd 방어) 로직 그대로 이동
- **이유**: 검색/필터 UI 컴포넌트(SermonSearch·TagFilter·ActiveFilters·Pagination)는 URL만 조작하므로 무수정 재사용 가능

#### 5. 상세 페이지 전환
- **파일**: `src/app/(site)/sermons/[id]/page.tsx`, `src/components/sermons/SermonDetail.tsx` (신규), `src/app/(site)/sermons/[id]/loading.tsx` (삭제)
- **변경 내용**: RSC는 id 검증(비숫자·0·음수 → notFound)만 담당, 영상·오디오·메타 링크·마크다운 본문 렌더는 클라이언트로 이동. 404 포함 에러는 안내+다시 시도 버튼
- **이유**: 상세 GET에 조회수+1 부수효과가 있어 클라이언트 직접 호출 구조가 자연스러움

#### 6. 수정 페이지 시드 전환
- **파일**: `src/app/(site)/sermons/[id]/edit/page.tsx`, `src/components/sermons/SermonEditLoader.tsx` (신규)
- **변경 내용**: RSC `getSermon` 시드 → 클라이언트 `useSermon` 시드(keyed 마운트로 defaultValues 고정). `MemberGate(SERMON_VIEW)` + `RequirePermission(SERMON_WRITE)` 이중 게이트
- **이유**: 시드 GET 자체가 SERMON_VIEW를 요구하므로 회원 영역 시드 관례(AlbumEditLoader 동형) 적용

#### 7. 쓰기 무효화 방식 전환
- **파일**: `src/components/sermons/SermonForm.tsx`, `src/components/sermons/SermonAdminActions.tsx`, `src/lib/admin/revalidate.ts`
- **변경 내용**: 등록·수정·삭제 성공 시 `revalidateSermons()`(ISR 태그 무효화) → `queryClient.invalidateQueries`(`["sermons"]`·`["sermon", id]`). 409 재편집도 `router.refresh()` → 상세 쿼리 무효화. `revalidateSermons` 함수 제거
- **이유**: 공개 ISR 캐시가 사라졌으므로 무효화 대상이 회원 쿼리 캐시로 바뀜

#### 8. 문서 갱신
- **파일**: `docs/church-frontend-guide.md`, `docs/api-docs.json`
- **변경 내용**: 권한 매핑표·경로 인가 3분법에 `SERMON_VIEW` 행 추가, 차단 UX 문단에 설교 병기, 도메인 표 "조회 공개" → "조회 SERMON_VIEW(회원전용)", 교인 승인 흐름에 SERMON_VIEW 획득 반영. api-docs.json은 백엔드 OpenAPI 최신본으로 교체

### 🔧 주요 변경사항 상세

#### MemberGate 분기 순서 (기존 동작 보존)
하이드레이션 전 스켈레톤 → 토큰 없음(로그인 안내) → me 로딩(스켈레톤) → 에러(다시 시도) → 권한 없음(승인 안내) → 통과.
`!accessToken`을 `isPending`보다 먼저 평가하는 기존 방어(비로그인 시 무한 스켈레톤 방지) 유지.

**특이사항**:
- 스켈레톤 testid가 `gallery-skeleton` → `member-gate-skeleton`으로 변경 (테스트에서 참조 시 주의)
- 상세 페이지는 카드 그리드 대신 `DetailSkeleton`을 게이트 스켈레톤으로 주입

#### 페이지 테스트 재구성
구 RSC 페이지 테스트 3개(목록·상세·수정)가 서버 fetch 전제라 셸 테스트로 재작성.
기존 커버리지는 소실 없이 이전 — URL 파라미터 파싱·방어는 `SermonList.test`, 클릭 메타 링크·조회수·영상/오디오 분기는 `SermonDetail.test`, id 검증(notFound)은 각 셸 테스트가 담당.

#### 라우팅 결과 변화
`/sermons` 목록이 동적 렌더(searchParams) → 정적 셸 + 클라 패칭으로 바뀌어 빌드 시 백엔드 의존 제거. 상세·수정은 동적 유지(id 검증).
