# 어드민 02 콘텐츠(설교·공지) 등록·수정·삭제

### 📌 작업 개요

어드민 트랙(2차 배치)의 wave1 도메인. 권한 보유 운영자가 **공개 설교·공지 페이지의 인라인 액션 + 전용 편집 라우트**로 설교·공지를 등록·수정·삭제하도록 연동했다. 별도 `/admin` 백오피스가 아니라 공개 RSC 페이지 위에 client island를 얹는 진입 모델을 따른다.

02는 어드민 트랙 공유 자산(`MarkdownEditor`·`TagMultiSelect`·인라인 액션 패턴)의 **단일 생산자**로, 이를 최초 구현해 후속 도메인(03 일정·05 미디어)이 재사용한다.

**이연(02 범위 외)**: 본문 이미지·PDF 업로드 UI(→05 MediaUploader), 신규 태그 생성(→06 태그 관리).

### 🎯 구현 목표

- 공개 설교·공지 목록/상세에 권한 보유 시 등록·수정·삭제 인라인 액션 노출
- 본문이 길어 작성·수정은 전용 편집 화면(작성/미리보기 탭 에디터 + 태그 다중선택)

- 동시 수정 충돌(낙관락) 안내·입력 검증·공개 반영 지연 안내

### 🔧 주요 변경사항 상세

#### 어드민 쓰기 함수의 `*.admin.ts` 분리 (RSC 번들 경계)
편집 라우트(서버 컴포넌트)가 프리필을 위해 공개 GET 모듈(`sermons.ts`)을 import하는데, 같은 모듈에 어드민 쓰기 함수를 두면 `apiMutate`→`authFetch`→`authStore`(`useSyncExternalStore`, 클라이언트 전용) 체인이 서버 번들에 끌려가 Turbopack 빌드가 실패했다. 어드민 쓰기 함수·타입을 `sermons.admin.ts`/`notices.admin.ts`로 분리하고 공개 모듈은 `export type`만 재노출해 해결했다.

**특이사항**: 공개 GET을 RSC가 쓰는 도메인(03 일정·05 갤러리/주보)도 동일 분리를 따라야 한다. 조율 문서·전략에 규칙으로 반영했다.

#### 미리보기 변환 게이트
Radix `TabsContent`는 비활성 시 언마운트되지 않고 hidden으로 남는다. 미리보기 `MarkdownContent`를 `tab === "preview"`일 때만 마운트해 작성 중(키 입력마다) 동기 마크다운 변환이 도는 것을 막았다.

#### 목록 카드 중첩 링크 회피
공개 목록 카드/행은 전체가 단일 링크라 카드 안에 액션 버튼을 넣으면 중첩 `<a>` 위반이 된다. 목록에는 등록 버튼만 두고 개별 수정/삭제는 상세 페이지에서 처리한다.

#### 테스트 하네스 보정
Radix Tabs는 jsdom에서 `fireEvent.click`으로 탭이 전환되지 않아 `fireEvent.mouseDown`을 사용했다. 새 client island(useMe·QueryClient 의존) 주입으로 기존 공개 페이지 테스트 4개가 깨져, 각 테스트에 island 스텁 mock을 추가해 회귀를 복구했다(단언 약화 없음).
