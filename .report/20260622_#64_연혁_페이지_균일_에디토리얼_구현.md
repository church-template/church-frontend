### 📌 작업 개요

연혁 페이지(`/about/history`)의 연출 방식을 **균일 에디토리얼**로 재정립. 이전에 시도한 풀스크린 중앙 축소 히어로(`HistoryHero`, `clip-path` 스크럽 연출)를 폐기하고, "연혁" 제목 블록을 시대 카드와 동일한 Reveal 흐름의 한 블록으로 단일화. 데스크톱 2단 레이아웃(좌 sticky 사진 / 우 카드 스택)과 모바일 단일 컬럼은 그대로 유지하면서, 폐기 모듈 5묶음 및 고아 파일을 정리했다.

---

### 🎯 구현 목표

- 풀스크린 히어로 분기(`HistoryHero`) 제거 → 제목 섹션을 단일 Reveal 블록으로 승격
- 모든 시대 균일 처리(첫 시대 특별 연출 없음)
- 폐기 파일(`HistoryHero`, `historyScrub`, `HistoryYearRail`, `tone`, `HistoryIntro` 및 각 테스트) 제거
- `pnpm exec tsc --noEmit` / `pnpm lint` / `pnpm exec vitest run` 전 통과 유지

### 🔧 주요 변경사항 상세

#### 폐기 모듈 정리

아래 파일들이 제거됐다. 모두 히어로 분기 제거 후 아무도 import하지 않는 고아 상태가 됐거나, 처음부터 테스트 외 참조가 없던 파일이다.

| 제거 파일 | 사유 |
|---|---|
| `HistoryHero.tsx` / `HistoryHero.module.css` / `HistoryHero.test.tsx` | 풀스크린 중앙 축소 히어로 — 분기 제거 후 미사용 |
| `historyScrub.ts` / `historyScrub.test.ts` | 스크럽 연출 전용 유틸 — 테스트 외 import 0 |
| `HistoryYearRail.tsx` / `HistoryYearRail.test.tsx` | 세로 연도 레일 — 테스트 외 import 0 |
| `tone.ts` / `tone.test.ts` | 시대별 배경 톤 유틸 — 테스트 외 import 0 |
| `HistoryIntro.tsx` / `HistoryIntro.test.tsx` | `displayMega` 교회명 인트로 — 테스트 외 import 0 |

`src/hero/scrub.ts`(공유 유틸)는 MediaCollage 등이 여전히 사용하므로 유지.

#### 의미구조 정정

기존 히어로 없음 분기(`else`)에서 `<p>`가 연혁 제목을, `<h1>`이 인트로를 담당하던 역전 구조를 수정. 현재는 `<h1>연혁</h1>`(페이지 대표 heading) + `<p>{intro}</p>`(리드 카피) 순서로 올바른 의미구조 적용.

#### 좌측 aside 단순화

스크롤 추적 active 시대의 사진(`HistoryMedia`)만 렌더. 연도 라벨·시대 제목은 의미 전달이 목적이므로 aria-hidden인 aside가 아니라 우측 카드에서만 노출. `.asideInner` / `.asideYear` / `.asideTitle` CSS 규칙이 정의돼 있으나 실제 사용되지 않음(잔류 규칙이나 레이아웃 무해).

### 📌 참고사항

- 각 시대 사진(`/public/history/{id}.jpg`)은 placeholder 상태. 교체 시 `src/constants/content.ts`의 `media.src` 경로만 수정하면 된다.
- `HistoryStory.module.css`의 `.asideYear` / `.asideTitle` 규칙은 현재 마크업에서 사용되지 않는다. aside에 연도·라벨을 다시 노출할 필요가 생기면 재활성 가능(규칙 삭제하지 않음).
- `HistoryBand`(메인 페이지)는 동일한 `HISTORY` 상수를 공유. 데이터 변경 시 메인 섹션도 자동 반영.
- `useHistoryScrollEngine`의 jsdom 환경 한계(`getBoundingClientRect` → 0 반환)로 active 전환 자체는 단위 테스트로 검증 불가. Playwright 수동 시각 확인으로 보완.
