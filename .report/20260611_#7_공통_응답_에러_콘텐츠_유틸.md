# T06 공통 응답·에러·콘텐츠 유틸 구현 보고서

이슈: #7
날짜: 2026-06-11

## 1. 작업 개요

T05 인증 인프라(`authFetch`·토큰 refresh·Zustand) 위에, 전 도메인이 공유하는 공통 코어를 구축했다. 구성 요소는 다섯 갈래다.

- **응답 봉투**: 비-2xx 응답을 RFC 7807 기반 `ApiError`로 변환(7필드 완전체).
- **errorCode → UI 라우터**: `errorCode` 단일 기준으로 분기하고, 컨텍스트 의존 효과(라우터·RHF·모달)는 소비측이 콜백으로 주입, 미제공 시 토스트로 폴백.
- **페이지네이션·태그필터**: Spring `PagedModel` 직렬화 타입과 0-base·tagId 단수 표준 목록 쿼리 빌더, 그리고 이를 소비하는 `Pagination`·`TagFilter` 클라이언트 컴포넌트.
- **마크다운**: `marked` 변환 + `isomorphic-dompurify` 새니타이즈, `media:{id}` → 공개 서빙 URL 치환.
- **날짜 파서**: offset 없는 서버 `LocalDateTime`에 `+09:00`(KST)을 부착해 런타임 TZ와 무관하게 파싱.

소비처는 UI 도메인 페이지(T14+: 설교·공지·일정·부서·갤러리 등)와 공개 콘텐츠(서버 컴포넌트 + ISR)다.

## 2. 구현 내용 (파일별)

### `src/lib/auth/apiError.ts` — 응답 봉투

- `ApiError`를 RFC 7807 7필드 완전체로 확장: `status`·`errorCode`·`detail`에 더해 `title`·`instance`·`errors`(`FieldError[]`)·`references`(`MediaReference[]`)를 보강. 조건부 키(`errors`는 `INVALID_INPUT_VALUE`, `references`는 `MEDIA_IN_USE` 시에만)는 백엔드 `@JsonInclude(NON_NULL)` 직렬화에 맞춰 옵셔널.
- `Error.message`는 `detail ?? title ?? "HTTP {status}"` 순으로 폴백.
- `parseJson<T>`이 비-2xx 시 본문을 방어적으로 파싱(`res.clone().json().catch(() => ({}))`)해 7필드를 모두 채워 throw. 생성자 인자 순서를 유지해 기존 3-인자 호출과 하위 호환된다.

### `src/lib/auth/handleApiError.ts` — errorCode → UI 라우터

- `ApiErrorHandlers` 인터페이스로 소비측이 컨텍스트 효과를 콜백 주입: `onFieldErrors`(RHF `setError`)·`onRedirectToLogin`·`onReedit`(낙관적 락 충돌 재편집)·`onMediaReferences`·`onDuplicate`·`onAuthFailed`.
- 분기는 `errorCode`로만 수행(status·title로 분기하지 않음). 각 케이스는 핸들러 주입 시 콜백, 미제공 시 `notify` 토스트로 폴백한다.
- **`INVALID_TOKEN` 침묵 금지**: 이 지점 도달은 `authFetch`가 refresh 실패 후 `forceLogout`을 완료한 상태(세션 만료)이므로, 핸들러 없으면 "세션이 만료되었습니다…" 토스트를 띄운다.
- `AUTHENTICATION_FAILED`은 가입 여부 비노출을 위해 고정 메시지("전화번호 또는 비밀번호가 올바르지 않습니다.")만 출력.
- 회원·어드민(클라이언트) 흐름 전용이며 `notify`가 client seam이다. 공개 서버 페이지의 비-2xx는 `notFound()`/에러 바운더리가 처리하고 이 함수를 호출하지 않는다.

### `src/lib/page.ts` — 페이지네이션 타입·쿼리 빌더

- `Page<T>`(`content` + `page: PageMeta`)와 `PageMeta`(`size`·`number`(0-base)·`totalElements`·`totalPages`) 정의. Spring `PagedModel`(VIA_DTO) 직렬화 형태에 정확히 대응(`page` 하위 키 4개).
- `buildListQuery(q)`가 표준 목록 쿼리 문자열을 생성: `page` 0-base, `tagId` 단수, `undefined` 필드는 생략, 결과 없으면 빈 문자열 반환.

### `src/lib/markdown.ts` — 마크다운 파이프라인

- 3단계: ① `media:{id}` → `/api/media/{id}` 치환 ② `marked.parse(..., { async: false })`로 MD → HTML ③ `DOMPurify.sanitize`로 마크다운 산출 태그 allowlist만 통과.
- `MEDIA_REF` 정규식 `/media:(\d+)(?!\d)/g`은 경계 안전: `media:420`이 `media:42` + 잔여 `0`으로 쪼개지지 않도록 negative lookahead로 오탐 방지(서버 추적 규약과 동일).
- `ALLOWED_TAGS`는 헤딩·강조·리스트·링크·이미지·코드·표 등 마크다운이 실제로 생성하는 태그로 한정, `ALLOWED_ATTR`은 `href`·`title`·`src`·`alt`. 저자 작성 raw HTML(`<div>`·`<iframe>` 등)은 태그가 제거되고 내용은 보존, `<script>`·이벤트 핸들러는 통째로 제거된다.

### `src/lib/date.ts` — 서버 날짜 파서

- `parseServerDate(s)`이 offset 없는 `LocalDateTime`에 `+09:00`을 명시 부착 후 파싱. datetime 문자열(`T` 포함)은 그대로, date-only(`YYYY-MM-DD`)는 `T00:00:00+09:00`을 붙여 KST 자정으로 해석.
- 브라우저 로컬 TZ 파싱으로 인한 SSR·비-KST 환경 어긋남을 차단(KST 가정 고정).

### `src/components/common/` — 공통 표시 컴포넌트

- **`EmptyState.tsx`**(순수/서버): "등록된 ○○가 없습니다" 패턴 빈 목록 표준 표시. `typo.bodyMd`·`text-muted` 토큰 사용.
- **`Skeleton.tsx`**(순수/서버): 로딩 자리표시. `animate-pulse` + `surface-strong` + `rounded-md`.
- **`MarkdownContent.tsx`**(순수/서버): `renderMarkdown`이 새니타이즈한 HTML을 `dangerouslySetInnerHTML`로 주입(allowlist가 안전성을 보장). `prose-church` 클래스 적용, ISR 시 서버에서 변환.
- **`Pagination.tsx`**(클라이언트): `pageItems`로 7개 초과 시 첫·끝 + 현재 주변 + 말줄임 토큰 계산(0-base). `usePathname`·`useSearchParams`로 기존 쿼리(tagId·sort) 보존하며 `page`만 교체. `aria-current`·`aria-disabled`·`aria-label` 접근성. 내부 `PaginationControls`를 `Suspense`로 래핑해 export.
- **`TagFilter.tsx`**(클라이언트): tagId 교체/제거 + page 리셋, 다른 쿼리 보존. "전체" 칩 + 태그 칩, `aria-pressed`·`role="group"`. 내부 `TagFilterPills`를 `Suspense`로 래핑해 export.

### `src/app/globals.css` — `.prose-church`

- `@tailwindcss/typography`(스택 밖) 미사용. DESIGN.md `body-md`(행간 1.7) 기준으로 토큰만 참조해 prose 스타일 직접 정의: 헤딩(굵기 500, `display-md`/`title-lg`/`title-md`), 링크(`primary`/`primary-active`), 리스트·blockquote·img(라운드 `lg`)·code/pre 등. hex·px 인라인 없이 CSS 변수 참조.

- Next 16 testing 문서 기준 수동 설정. `environment: "jsdom"`, `vite-tsconfig-paths`·`@vitejs/plugin-react` 플러그인.

## 3. 주요 설계 결정

- **DOMPurify SSR(isomorphic-dompurify)**: 공개 콘텐츠 상세는 서버 컴포넌트 + ISR로 렌더되므로, 브라우저 전용 DOMPurify 대신 SSR에서도 동작하는 isomorphic 빌드를 채택.
- **핸들러 주입 + 토스트 폴백**: `handleApiError`가 컨텍스트 의존 효과(라우터/RHF/모달)를 직접 끌어쓰지 않고 콜백으로 받아, 라이브러리 코드의 컨텍스트 결합을 제거. 미제공 시 토스트로 폴백해 "침묵하는 실패"를 막는다. 특히 `INVALID_TOKEN`은 폴백 토스트를 강제한다.
- **raw HTML allowlist**: `marked` 산출 태그만 화이트리스트로 통과시켜 저자 raw HTML(`<div>` 등) 태그를 제거하고 `<script>`는 통째 제거. 가이드 5.3의 `USE_PROFILES` 방식 대신 명시적 `ALLOWED_TAGS`/`ALLOWED_ATTR`을 쓴 의도적 차이로, 가이드 5.1 "raw HTML 비허용" + 이슈 §7 "raw HTML 제거" 요건을 더 엄격히 충족.
- **Pagination/TagFilter 내부 Suspense**: `useSearchParams`를 쓰는 클라이언트 컴포넌트가 공개 ISR prerender에서 "Missing Suspense boundary"로 빌드 실패하지 않도록, 공개 export를 `Suspense`로 감싼 래퍼로 제공.
