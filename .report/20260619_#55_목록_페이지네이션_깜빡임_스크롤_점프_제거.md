# 목록 페이지 전환 시 전체 리렌더(깜빡임)·스크롤 점프 제거

### 📌 작업 개요
회원 영역 클라이언트 목록(회원 관리·미디어 라이브러리·갤러리 앨범)에서 페이지를 넘길 때, 목록이 스켈레톤으로 전부 비워졌다 다시 채워지며 깜빡이고 화면이 상단으로 스크롤 점프하던 문제를 제거. 보이는 목록 부분만 매끄럽게 교체되도록 개선.

### 🔍 문제 분석
페이지 전환 체감이 "전체 리로드"처럼 보인 원인은 세 가지였다.

- 목록 쿼리(TanStack Query)에 `placeholderData`가 없어, 페이지를 바꾸면 `queryKey`가 변경되며 `isPending`이 다시 `true`가 됨 → 표/그리드가 스켈레톤으로 전부 사라졌다가 새 데이터로 다시 그려짐. (깜빡임의 1순위 원인)
- 페이지 버튼이 `<Link>` 네비게이션이라 클릭 시 화면이 상단으로 스크롤 점프.

공개 ISR 페이지(설교·공지·주보)는 서버 컴포넌트 + ISR 캐시 구조라 현재 방식이 올바르므로 이번 개선 대상에서 제외.

### 🎯 구현 목표
- 페이지 전환 중 이전 행을 유지하다 새 데이터 도착 시 행만 교체(스켈레톤 깜빡임 제거)
- 페이지 버튼 클릭 시 스크롤 점프 제거
- 로딩 중 목록에 미세 디밍 + `aria-busy`로 진행 상태 표시
- URL 구동 페이지네이션(딥링크·뒤로가기)·공개 ISR 페이지 동작은 그대로 보존

### ✅ 구현 내용

#### 공유 Pagination에 `scroll` prop 추가
- **파일**: `src/components/common/Pagination.tsx`
- **변경 내용**: `Pagination`/`PaginationControls`/`Arrow`에 선택적 `scroll?: boolean` 추가(기본값 `true`). 숫자 링크와 이전/다음 화살표 `<Link>` 양쪽에 전달.
- **이유**: 대상 목록만 `scroll={false}`로 스크롤 점프를 끄고, 공개 페이지는 prop 미지정 시 기본값 `true`로 기존 스크롤-투-탑 동작을 그대로 유지하기 위함. 공유 컴포넌트를 전역 변경하지 않고 호출부별로 분기.

#### 회원 관리 목록 개선
- **파일**: `src/components/admin/members/MemberManager.tsx`
- **변경 내용**: members 쿼리에 `placeholderData: keepPreviousData` 적용. `DataTable`을 디밍 wrapper로 감싸 `isPlaceholderData`일 때 `opacity-60 transition-opacity` + `aria-busy` 부여. `<Pagination>`에 `scroll={false}` 전달.
- **이유**: 페이지 전환 중 이전 회원 행을 유지하면서, 로딩 중임을 시각·접근성 양면으로 알림.

#### 미디어 라이브러리 목록 개선
- **파일**: `src/components/admin/media/MediaLibrary.tsx`
- **변경 내용**: 위와 동일 패턴(keepPreviousData + 디밍 wrapper + `scroll={false}`).

#### 갤러리 앨범 목록 개선
- **파일**: `src/components/gallery/queries.ts`, `src/components/gallery/AlbumList.tsx`
- **변경 내용**: `useAlbums` 쿼리에만 `placeholderData: keepPreviousData` 추가. `AlbumList`의 앨범 그리드 div에 `isPlaceholderData` 디밍 + `aria-busy` 병합, `<Pagination>`에 `scroll={false}` 전달.
- **이유**: 상세(`useAlbum`)·태그(`useGalleryTags`) 쿼리는 페이지네이션 대상이 아니므로 목록 쿼리에만 선별 적용.

### 🔧 주요 변경사항 상세

#### keepPreviousData 적용 범위
목록 쿼리 3개에만 개별 적용하고, 전역 `QueryClient` 기본옵션에는 넣지 않음. 전역에 걸면 상세 조회 등 페이지네이션과 무관한 쿼리까지 영향을 받기 때문.

**동작**:
- 첫 로드: 데이터 없음 → `isPending=true` → 스켈레톤 표시(정상)
- 페이지 전환: `isPending=false` 유지, `isPlaceholderData=true` → 이전 행 유지 + 미세 디밍 → 새 데이터 도착 시 행 교체

#### Pagination 기본값 보존 설계
`scroll` 기본값을 `true`로 두어 공개 ISR 3페이지(`<Pagination page={...} />` 무인자 호출)는 변경 없이 기존 스크롤 동작을 유지. 회귀 방지를 위해 "기본값은 scroll 유지(true)" 테스트를 추가해 못박음.

**특이사항**:
- 디밍 className은 `cn()` 내부 `&&`로 조합(프로젝트 규칙상 이 위치의 `&&`는 허용 예외). `opacity-60`·`transition-opacity` 표준 Tailwind 유틸만 사용, arbitrary value·인라인 hex/px 없음.
- 갤러리 앨범은 페이지 전환 *중 fetch 에러* 시 이전 행을 유지하지 않고 에러 화면으로 전환됨(회원/미디어는 toast로 처리하며 표 유지). 이번 범위(깜빡임·스크롤만)에서 에러 처리 일관화는 비대상. 정상 전환 경로에서는 셋 다 동일하게 깜빡임이 사라짐.

### 🧪 테스트 및 검증
- Pagination `scroll` prop 단위 테스트 3건 추가(숫자 링크·다음 화살표 전달, 기본값 true 보존). next/link mock이 `scroll`을 `data-scroll`로 노출하도록 보강해 검증.
- 전체 테스트 189 파일 / 763건 PASS(회귀 없음).
- `pnpm lint` 0 errors, `npx tsc --noEmit` 0 errors.
- 공개 ISR 3페이지(설교·공지·주보)는 무인자 호출로 기본 `scroll=true` → 스크롤 동작 보존 확인.

### 📌 참고사항
- 범위에서 의도적으로 제외: 페이지 이동 시 서버(RSC) 왕복 제거(history API 얕은 라우팅), 목록 island 분리(렌더 격리). 추가 최적화가 필요하면 별도 진행 가능.
- `MypageContent.*`의 무관한 기존 수정은 이번 작업에서 건드리지 않음.
