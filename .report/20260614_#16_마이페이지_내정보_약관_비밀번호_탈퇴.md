# 마이페이지 (내 정보·약관·비밀번호·회원 탈퇴) #16

## 📌 작업 개요

로그인 회원이 본인 정보를 조회·수정하고, 약관 동의 상태를 확인하며, 비밀번호 변경과 회원 탈퇴를 수행하는 `/mypage`를 구현. 회원 영역이므로 전부 클라이언트 + TanStack Query + `authFetch` 기반이며, 권한·약관 등 라이브 값은 토큰이 아니라 `GET /api/members/me`를 신뢰한다(가이드 1.5).

기능별 6커밋으로 구성: ①내 정보 API·타입 ②destructive 버튼·토큰 ③폼 스키마·권한 라벨 ④프로필 카드·편집 폼 ⑤비번·약관·탈퇴 ⑥페이지 조립·UI.

## 🎯 구현 목표

- 프로필 조회/수정(이름·전화·이메일), 비밀번호 변경
- 약관 동의 상태 표시 + 재동의 링크(본인 행동, 권한 불필요)
- 회원 탈퇴(비밀번호 재인증 모달, 확인 버튼 빨강 강조)
- `permissions` 기반 표시(권한 한글 칩·교인 승인 배지)

## 🔧 주요 변경사항 상세

**에러 처리** — 전 폼이 `handleApiError(e, handlers)`로 errorCode 기준 분기. `INVALID_INPUT_VALUE`는 `onFieldErrors`로 폼별 화이트리스트 매핑(`setError(field, { message: reason })`), 전화 중복은 `onDuplicate`, 탈퇴 비번 오류는 `onAuthFailed`로 모달 인라인.

**탈퇴 세션 정리** — `authApi.withdraw`가 성공 시 내부에서 `authStore.clear()` 수행, 호출측은 `removeQueries(['me'])` + 홈 이동만. 서버가 전체 세션을 무효화(access 블랙리스트)하므로 `logout`은 호출하지 않음(호출 시 401).

**특이사항**:
- 약관 재동의 제출(`PATCH /api/members/me/agreements`)은 본인 인증만 필요(권한 X). 관리자 일괄 리셋(`POST /api/admin/agreements/reset`, MEMBER_MANAGE)과는 별개 — 마이페이지는 리셋을 건드리지 않고 상태 표시 + 링크만.
- 회원 ID(raw UUID) 표시는 가독성·UX상 노이즈라 **제외**(사용자 결정). 이슈 완료조건의 "uuid 표시"에서 의도적으로 빠짐.

## 📌 참고사항
- **비밀번호 변경 세션 영향 미확정**: PATCH `/me` password 변경 시 기존 토큰 유효성을 OpenAPI가 명시하지 않음. "재로그인 불필요"를 단정하지 않고 성공 후 `invalidateQueries(['me'])`로 401 시 정상 만료 흐름을 타게 처리. 실제 백엔드 동작 확인 후 갱신 필요.
- **탈퇴 비번 오류 errorCode**: `AUTHENTICATION_FAILED` 가정. 실제 응답 확인 후 매핑 보정 가능(구조는 불변).
- **미커밋 항목**: `docs/api-docs.json`(`/api/members/me` PATCH·DELETE OpenAPI 추가분), 설계·계획 문서(`docs/superpowers/{specs,plans}/2026-06-13-mypage*.md`)는 커밋 보류 상태.
