# 주보 목록·PDF 열람 (#14)

### 📌 작업 개요

주보 목록(`/bulletins`)을 공개 서버 컴포넌트로 구현하고, 항목 클릭 시 브라우저 기본 PDF 뷰어로 새 탭 직행. 주보는 본문이 없고 `mediaId` FK만 가지므로, 상세 페이지 없이 행 클릭 = PDF 열람으로 설계(D1·D2). 공지·설교·일정과 동일한 패턴(서버 컴포넌트 + ISR + 공용 페이지네이션)을 따르되 구조를 최소로 유지.

**참조:** 가이드 10장(주보)·6장(미디어)·7장(작성자), OpenAPI `/api/bulletins`·`/api/media/{id}` / 설계 문서: `docs/superpowers/specs/2026-06-12-bulletins-design.md`, 계획: `docs/superpowers/plans/2026-06-12-bulletins.md`

---

### 🎯 구현 목표

- 목록 `/bulletins`: `GET /api/bulletins?page`(필터 없음 — 가이드 10장) → `BulletinRow` 행 목록 + `Pagination`
- PDF 열람: 행 클릭 → `GET /api/media/{mediaId}` 새 탭(`<a target="_blank">`) — 브라우저 기본 뷰어
- 패칭 경계: 서버 컴포넌트 `fetch` + ISR(`revalidate: 60`). TanStack 미사용(회원 영역 전용 규칙)
- 주보 등록·수정·삭제(어드민) 및 PDF 임베드 뷰어는 범위 밖

### ✅ 구현 내용

#### 주보 API 클라이언트·타입
- **파일**: `src/lib/api/types.ts`(수정), `src/lib/api/bulletins.ts`(신규)
- **변경 내용**: `BulletinCardResponse` 타입 추가(id·title·serviceDate·mediaId·createdAt·author). `getBulletins` fetch 함수 + `BulletinListParams`(ListQuery에서 `tagId` 제거 — 타입으로 차단)
- **이유**: 필터 없는 주보 파라미터는 공용 `buildListQuery`(DRY)를 재사용하되 `tagId`만 타입 단계에서 차단. 조회수 부수효과 없으므로 `revalidate: 60` 캐싱 가능(`no-store` 불필요)

#### BulletinRow 행 컴포넌트
- **파일**: `src/components/cards/BulletinRow.tsx`(신규)
- **변경 내용**: notice-row 시각 문법(hairline 구분·hover 시 제목 primary 전이·행 전체 클릭 영역)을 따르는 행 컴포넌트. 루트는 `<a target="_blank" rel="noopener noreferrer">` — 외부(백엔드 오리진) PDF라 `next/link` 미사용
- **이유**: 외부 앵커 동작과 author 표기 차이로 NoticeRow 일반화 대신 신설(D4 — 단일 사용처 선제 일반화는 YAGNI). `author` null/빈 값은 삼항으로 줄 생략, 스크린리더용 sr-only 새 탭 안내 포함

#### 주보 목록 페이지
- **파일**: `src/app/(site)/bulletins/page.tsx`(신규)
- **변경 내용**: `page` 파라미터만 파싱(`toNum` NaN 방어 — 공지와 동일). 단일 `getBulletins` fetch, 빈 목록 시 `EmptyState`, `totalPages > 1`일 때만 `Pagination`
- **이유**: 태그·검색 필터 없으므로 공지보다 단순한 구조. 서버 정렬(`serviceDate,desc`)을 신뢰해 프론트 재정렬 없음

#### DESIGN.md bulletin-row 항목 추가
- **파일**: `.claude/rules/DESIGN.md`(수정)
- **변경 내용**: `components:` 절에 `bulletin-row` 정의 추가(제목 `typo.title-sm` + 예배일 `typo.datetime` + 작성자 `typo.body-sm`, 행 전체 새 탭 PDF 링크, hairline 구분). 타이포그래피 위계 표에 `title-sm` 행 추가
- **이유**: 문서에 없는 컴포넌트를 임의 스타일로 만들지 않는 프로젝트 규칙 — 구현 전에 DESIGN.md에 먼저 등재

### 🔧 주요 변경사항 상세

#### 핵심 설계 결정 4가지

| # | 결정 | 근거 |
|---|---|---|
| D1 | 새 탭 직행(`<a target="_blank">`) | 가장 단순. Android `<iframe>` PDF 미지원 회피. 백엔드가 `Content-Disposition: inline` 보장 |
| D2 | 상세 페이지 없음 | 행 클릭 = PDF 열람. 상세 API 추가 정보(`updatedAt`·`version`)는 공개 화면 표시 불필요(YAGNI) |
| D3 | 목록 = 행 목록(notice-row 변형) | 썸네일·본문 없이 title·serviceDate·author 세 필드뿐 — 문서형 콘텐츠에 행이 자연스럽고 기존 디자인 범위 내 |
| D4 | `BulletinRow` 신설(NoticeRow 일반화 안 함) | 외부 앵커 동작·author 표기로 NoticeRow와 동작이 다름. 수술적 범위 유지, 기존 공지 코드 불변 |

#### 공용 buildListQuery 재사용(DRY)
설교(`sermons.ts`)는 전용 필터(q·preacher 등)가 있어 자체 쿼리 빌더를 갖지만, 주보는 page·size·sort만 사용해 `ListQuery`의 부분집합이다. 별도 빌더 없이 `buildListQuery`를 그대로 재사용하고, `tagId`만 타입 단계에서 `Omit`으로 차단해 잘못된 파라미터 경로를 컴파일 타임에 막는다.

#### author 마스킹 처리
백엔드가 내리는 `"(탈퇴한 사용자)"`·`"(알 수 없음)"` 등 마스킹 값을 프론트가 재판단 없이 그대로 표기(가이드 7장). `author`가 `null`이거나 없으면 삼항으로 줄 자체를 생략 — `{cond && <X/>}` 패턴(falsy `0`·`""` 렌더) 금지 규칙 준수.

**특이사항:**
- `serviceDate`는 date-only 문자열(`yyyy-MM-dd`). 기존 `parseServerDate`가 `T00:00:00+09:00` 부착(KST 가정) → `formatDate`로 표기. 별도 처리 불필요
- PDF 새 탭 URL은 서버 컴포넌트에서 `apiUrl(/api/media/{mediaId})`로 절대/상대 URL 조합 — `NEXT_PUBLIC_API_BASE`가 빌드 시 앵커에 인라인되므로 서버 렌더에서도 동일 동작

### 📌 참고사항

- **수동 검수 보류(1건)**: 백엔드 테스트 데이터의 `mediaId`가 실파일 미등록(404) 상태. 실 PDF 파일 등록 후 새 탭 inline 열람 수동 검수 진행 필요
- `markdown.ts`의 `media:{id}` 치환이 상대경로(`/api/media/{id}`, `apiUrl` 미사용) — 별도 오리진 배포 시 본문 이미지가 깨질 수 있는 잠재 이슈. 이번 범위 밖이므로 기록만 남김
- PDF 대용량 시 Range 미지원(백엔드 E) — 일반 주보 PDF 열람엔 무방, 프론트 대응 없음
- 주보 등록·수정·삭제(어드민 `/api/admin/bulletins`), PDF 임베드 뷰어는 후속 과제
