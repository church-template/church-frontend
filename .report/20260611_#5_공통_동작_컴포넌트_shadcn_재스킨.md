# T04 공통 동작 컴포넌트(shadcn 재스킨) 구현 보고서

### 📌 작업 개요

동작 중심 컴포넌트 7종(`Dialog`·`Sheet`·`Popover`·`DropdownMenu`·`Select`·`Tabs`·`Toast`)을 Radix/sonner 소스로 **수동 벤더링**한 뒤 DESIGN.md 토큰으로 재스킨했다. shadcn `init`·시맨틱 변수 레이어·`tw-animate-css`는 도입하지 않고, 토큰을 직결로 적용해 접근성 동작은 보존했다. Toast는 루트 layout 전역 컨테이너 + `notify` seam으로 단일화했다.

검증은 `/showcase` 데모 + MCP 브라우저 접근성 실측 + `tsc`·`lint`·`build` 통과로 완료했다. 커밋은 기능별 4개로 분리했다(#5).

---

### 🎯 구현 목표

- 동작 컴포넌트만 shadcn/Radix·sonner로 도입(시각 컴포넌트 Button·Card·Badge·Input은 T03 직접 구현분 유지)
- 기본 룩(zinc 팔레트·기본 radius) 잔재 0 — DESIGN.md 토큰으로 재스킨
- Radix/Radix-기반 접근성(포커스 트랩·스크롤 잠금·포커스 복귀·`aria-live` 낭독) 보존
- portal 컴포넌트가 고정 네비 위에 뜨도록 z-index 레이어 정책 수립
- Toast 출력 채널 단일화(T06 errorCode→UI 연동 대비)

### ✅ 구현 내용

#### 도입 방식 — 수동 벤더링 + 토큰 직결 재스킨
- **파일**: `src/components/ui/{dialog,sheet,popover,dropdown-menu,select,tabs,sonner}.tsx`
- **변경 내용**: ui.shadcn.com 레지스트리 소스를 복사해 `src/components/ui/`에 lowercase로 배치 후 className을 우리 토큰 유틸(`bg-surface-card`·`rounded-xl`·`border-hairline`·`text-ink`·`z-overlay`/`z-popover`·`animate-*`)로 재작성
- **이유**: shadcn `init`은 이 환경(Tailwind v4)에서 `globals.css`에 시맨틱 변수를 자동 주입하고 `tw-animate-css`를 설치해 금지 조건과 충돌한다. 토큰 직결 재스킨은 shadcn 생성 클래스를 어차피 버리므로 CLI 이점(Radix 조합·접근성 배선)은 소스 복사로 동일하게 확보

#### 디자인 토큰 — z-index·모달폭·애니메이션
- **파일**: `src/app/globals.css`, `.claude/rules/DESIGN.md`
- **변경 내용**: `@theme`에 z-index(`--z-nav/popover/overlay/toast` = 10/40/50/60)·모달폭(`--container-modal`)·애니메이션(`--animate-*`) 토큰 추가. 파일 끝에 `@utility z-*` 유틸 + `@keyframes`(overlay/content/popover/slide) + `prefers-reduced-motion` 규칙 추가. DESIGN.md에 "레이어링(z-index)" 항목 신설
- **이유**: portal로 뜨는 동작 컴포넌트가 고정 네비(z 10) 아래 깔리지 않게 z 스케일을 토큰으로 단일 정의. 애니메이션은 외부 라이브러리 없이 자체 키프레임으로 처리

#### Toast 전역 컨테이너 + notify seam
- **파일**: `src/components/ui/sonner.tsx`, `src/lib/notify.ts`, `src/app/layout.tsx`
- **변경 내용**: 루트 layout에 전역 `<Toaster />` 1개 마운트(4초 자동소멸, success/error). `notify.success/error(msg)` seam을 client 전용 모듈로 분리
- **이유**: Toast는 T06 errorCode→UI 매핑의 단일 출력 채널. seam만 호출하게 해 출력 경로를 일원화

#### showcase 데모 섹션
- **파일**: `src/app/showcase/_behavior/BehaviorShowcase.tsx`, `src/app/showcase/page.tsx`
- **변경 내용**: 7종 컴포넌트 트리거 + 데모를 묶은 client 섹션을 `/showcase`에 추가
- **이유**: T03 검증 패턴 연장 — 브라우저에서 동작·재스킨·접근성을 실측하기 위함

### 🔧 주요 변경사항 상세

#### 재스킨 공통 규칙
표면 `bg-surface-card`(다크 위 `bg-surface-dark-elevated`), 백드롭 검은 알파, 라운드는 카드형 콘텐츠 `rounded-xl`(24px) → 내부 아이템 `rounded-md`/`rounded-sm`(중첩 라디우스), 포커스는 `focus-visible:ring-2 ring-primary`(T03 idiom), 텍스트 위계는 `typo.*` 상수. hex·px·zinc·shadcn 기본 변수 잔재 없음.

#### Dialog (Modal)
백드롭 + 센터 카드. `DialogTitle`/`DialogDescription` 보존(`aria-labelledby`/`aria-describedby` 연결). 닫기 버튼은 lucide `X`. 포커스 트랩·트리거 포커스 복귀·ESC·오버레이 닫기·body 스크롤 잠금은 Radix가 제공.

#### Toast (sonner)
**특이사항**: sonner는 표면/보더/라운드를 자체 CSS 변수(`--normal-bg`·`--normal-border`·`--normal-text`·`--border-radius`)로 그린다. 유틸 클래스(`rounded-lg`·`border-hairline`)는 specificity로 덮이므로, 재스킨은 클래스가 아니라 `<Toaster style>`에서 이 변수들을 우리 토큰으로 오버라이드했다. semantic 색(success/error)은 `richColors`를 끄고 아이콘에만 적용(DESIGN: 배경 채움 금지).

#### Tabs / Sheet (리뷰 반영)
- `TabsContent`는 Radix가 `tabIndex=0`을 주입해 포커스를 받으므로 `focus-visible` 링을 추가(WCAG 2.4.7).
- `SheetContent`에 `border border-hairline` 추가(Dialog와 경계 일관성).

### 🧪 테스트 및 검증

MCP 브라우저로 `/showcase` 실측:

- **Modal**: `role=dialog`, `aria-labelledby`→Title 연결, 포커스 트랩(포커스 모달 내부 유지), body 스크롤 잠금, ESC→트리거 포커스 복귀 확인
- **Toast**: 토스트 텍스트가 `aria-live` 영역에 주입됨 확인. 재스킨(16px·hairline 보더·success 아이콘) 확인
- **레이어**: z 토큰 해석값 10/40/50/60, Dialog content z=50·Popover content z=40 실측
- **잔재 스캔**: Dialog 24px/흰색/hairline, Toast 16px/흰색/hairline — zinc·기본 radius 0
- **allowlist**: `ui/`에 도입 7종 lowercase만(T03 PascalCase 무관)
- `pnpm exec tsc --noEmit` · `pnpm lint` · `pnpm build` 전부 통과

#### 검증 중 발견·수정한 버그 2건
1. **sonner 재스킨 누수**: 유틸 클래스가 sonner CSS 변수에 덮여 radius 8px·보더 #ededed 잔존 → `style` 변수 오버라이드로 정정(16px·hairline).
2. **모달 위치 어긋남**: Tailwind v4는 `-translate-x-1/2 -translate-y-1/2`를 `transform`이 아닌 **`translate` 속성**으로 컴파일한다. `content-in` 키프레임이 `transform`에 translate를 또 줘서 합산(-100%)돼, 애니메이션 동안 모달이 트리거 근처에서 시작했다가 가운데로 튀었다 → 키프레임을 scale만 애니메이트하도록 수정(중앙정렬은 `translate` 속성이 담당). 애니메이션 전 구간 정중앙 유지 실측 확인.
