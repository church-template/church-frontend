# T03 공통 시각 컴포넌트 구현 보고서

### 📌 작업 개요

DESIGN.md 디자인 시스템 명세에 따라 공통 시각 컴포넌트를 신규 구현했다.
기본 UI 원자(`Button` · `Badge` · `Input` · `Card`) 4종과 도메인 카드(`SermonCard` · `NoticeRow` · `ScheduleCard` · `EventCard` · `FeatureCard`) 5종이 대상이며, 색·타이포·간격 모두 토큰 유틸만 참조하고 hex·px 인라인을 배제했다.
검증은 쇼케이스 페이지(`/showcase`) Before/After 비교 + `tsc` · `lint` · `build` 통과로 완료했다.

---

### 🎯 구현 목표

- 디자인 토큰(CSS 변수 → Tailwind 유틸)만으로 모든 스타일 표현
- 서버 컴포넌트에서도 안전하게 사용 가능한 shared 컴포넌트 구조 확보
- hover · press · focus-visible · disabled · loading · 에러 보더 등 인터랙션 폴리시 완성
- Before/After 비교 쇼케이스 페이지로 시각적 회귀 방지

---

### ✅ 구현 내용

#### `src/lib/utils.ts` — cn 헬퍼 도입

- **내용**: `clsx` + `tailwind-merge` 조합의 `cn()` 함수 추가
- **이유**: variant 클래스와 소비자가 전달하는 `className` override가 충돌 없이 병합되어야 한다. shadcn 표준 경로(`@/lib/utils`)에 위치시켜 T04 동작 컴포넌트(shadcn)가 무수정으로 import 가능하도록 했다.

#### `src/components/ui/Button.tsx` — shared 컴포넌트

- **내용**: `ButtonVariant` 5종(`primary` · `secondary` · `outlineOnDark` · `tertiary` · `pillCta`) + `buttonVariants()` 함수 + `Button` 컴포넌트
- **주요 결정**:
  - **`"use client"` 없음**: 훅·상태 없이 prop만 전달하므로 서버 컴포넌트에서도 `buttonVariants()`를 안전하게 호출할 수 있다. 링크형 CTA(`<Link className={buttonVariants("pillCta")}>`)가 서버 컴포넌트 안에서 동작하기 위한 조건이다.
  - **`loading` prop**: CSS 스피너(현재 텍스트 색 상속)를 내장해 이중 제출을 방지. lucide-react 등 아이콘 라이브러리 미도입 상태에서도 동작한다.
  - **`rounded-lg`(16px) 고정**: 중첩 라디우스 원칙 — 배지 내부(8px)의 ×2. 직각 모서리 금지(DESIGN Shapes).

#### `src/components/ui/Badge.tsx`

- **내용**: `default` · `primary` 2종 variant. `rounded-sm`(8px) 고정.
- **주요 결정**: 배지까지 `rounded-pill`로 처리하면 전체 UI에 템플릿 느낌이 강해지므로 DESIGN Shapes 개정에 따라 8px로 낮췄다. 버튼(16px)과 배지(8px)의 위계를 중첩 라디우스 원칙으로 통일한다.

#### `src/components/ui/Input.tsx`

- **내용**: `text` · `searchPill` 2종 variant. `error` prop으로 에러 보더 + 메시지 렌더.
- **폴리시 디테일**:
  - `text` variant: 기본 1px hairline → hover 시 `muted-soft` 톤(포커스 예고) → 포커스 시 1px border + 1px ring = 시각상 2px primary(리플로우 없음).
  - `searchPill` variant: 검색 필만 `rounded-pill` 유지 — 인터랙션 패턴이 달라 예외 처리.
  - 에러 상태: DESIGN 명세는 "semantic 색은 폼 메시지 텍스트 전용"이지만 보더에도 적용하는 것이 UX 상 명확하다고 판단해 의도적으로 확장했다. 배경 채움은 여전히 금지.
  - `useId()`로 접근성 aria 연결(`aria-invalid` · `aria-describedby`) 처리.

#### `src/components/ui/Card.tsx`

- **내용**: `surface`(`card` · `soft`) · `bordered` · `interactive` 3개 prop.
- **내용**: `interactive` 활성 시 hover에서 `-translate-y-0.5`(2px 리프트) + `shadow-soft`(`0 4px 12px rgba(0,0,0,0.04)`) 표시. 그림자 단계는 이 하나뿐(DESIGN Elevation).

#### 도메인 카드 5종

| 컴포넌트 | 특이사항 |
|---|---|
| `SermonCard` | `href` 유무로 인터랙티브/비인터랙티브 분기. `group-hover:scale-[1.03]` 썸네일 줌은 href 있을 때만. `<img>` 사용 — T10에서 `next/image` + `remotePatterns` 교체 예정. |
| `NoticeRow` | 행 전체가 `<Link>`. hover 시 제목만 primary 전이(조용한 인터랙티브 힌트). |
| `ScheduleCard` | `surface="soft"` Card 위에 예배명·시간·장소만 표시. 비인터랙티브. |
| `EventCard` | `Badge variant="primary"` 날짜 배지 + 제목 + 요약. `href` 유무로 분기(SermonCard와 동일 패턴). |
| `FeatureCard` | 선택적 icon slot(`ReactNode`) + 제목 + 설명. 비인터랙티브. |

### 🔧 주요 결정 상세

#### `--shadow-soft` 토큰 등록
`globals.css`에 `--shadow-soft: 0 4px 12px rgba(0, 0, 0, 0.04)` 추가. DESIGN Elevation의 유일한 그림자 단계. Tailwind `shadow-soft` 유틸로 참조.

#### `outlineOnDark` variant 용도 명확화
다크 밴드(`bg-surface-dark`) 위에서만 사용하는 보조 CTA. 흰 텍스트 + 흰 보더, hover 시 `on-dark/10` 알파 틴트, focus ring도 `ring-on-dark`로 대비를 보장한다.

#### `buttonVariants()` 함수 노출
클래스 문자열만 반환하는 함수를 별도로 export해 링크형 CTA(`<Link className={buttonVariants("pillCta")}>`)를 서버 컴포넌트에서 사용할 수 있게 했다.

### 📌 참고사항

- **`lucide-react` 미도입**: loading 스피너를 CSS(`border-t-transparent` 회전)로 처리해 아이콘 라이브러리 의존 없이 구현했다.
- **`searchPill`만 `rounded-pill` 유지**: 검색 입력은 DESIGN에서 검색 필을 명시적으로 pill로 지정했으므로 예외 처리.
- **`remotePatterns` 미처리**: `SermonCard`의 `<img>` → `next/image` 교체 및 외부 URL `remotePatterns` 설정은 T10에서 진행한다. 현재는 eslint-disable 주석으로 명시.
- **`_before/` 스냅샷**: 쇼케이스 비교용 폴리시 적용 전 버전. 프로덕션 번들에는 포함되지 않으며 showcase 라우트에서만 참조한다.
