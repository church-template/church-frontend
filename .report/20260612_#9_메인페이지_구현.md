# #9 메인 페이지 구현 보고서

브랜치: `20260610_#9_메인_페이지_13장_14A_CrossHero`

---

## 1. 개요

교회 홈페이지의 **메인 페이지(가이드 13장 / 14A CrossHero)** 를 완성했다. 십자가 열쇠구멍 히어로(CrossHero)와 풀스크린 미디어가 중앙 카드로 축소되는 미디어 콜라주를 연출의 축으로 삼고, 그 아래로 연혁·사역·예배시간·설교·공지·일정 섹션을 잇는다. 공개 콘텐츠는 서버 컴포넌트에서 `/api/main`을 한 번에 fetch(ISR revalidate 60)하며, 사용자 노출 텍스트는 전부 상수(`src/constants/church.ts`, `content.ts`) 또는 API로 주입된다(하드코딩 0).

이 과정에서 T07부터 잠복하던 **전역 타이포 붕괴 버그(`cn()` 병합)** 를 잡아냈고, 주 사용자층(고령 교인) 가독성 기준으로 타이포·UI 스케일을 확대했으며, 우리은행 GNB 방식의 **풀폭 메가메뉴 헤더**와 IO 기반 등장 연출을 갖는 **연혁·사역 섹션**을 추가했다.

기커밋 + 미커밋 합산 #9 범위는 15개 커밋 규모이며, 테스트 191개가 모두 통과한다.

## 2. 주요 변경

### 2.1 메인 페이지 골격 · 공개 데이터 레이어 (기커밋)

- `src/app/page.tsx` — 섹션 구성. `await connection()`으로 prerender를 요청 시점까지 미뤄 CI(백엔드 없음) 빌드를 통과시키되, fetch 데이터 캐시(revalidate 60)는 그대로 살린다. `HeroHeaderSync`로 투명 헤더와 히어로를 합성하고, 그 아래 MediaCollage → HistoryBand → MinistryCards → Worship → Sermon → Notice → Event 순으로 배치한다.
- `src/lib/api/main.ts` — 메인 통합 조회(`getMain`). OpenAPI 단일 진실 전제로 zod 검증은 생략하고 배열 누락만 방어(`?? []`). 실패 시 루트 `error.tsx`로 위임.
- `src/hero/CrossHero.tsx` (+ `.module.css`) — 십자가 열쇠구멍 히어로(14A). 풀스크린 전환 후 카피 등장. 비율은 14A.3 확정값 하드코딩(SVG 좌표는 검증 로직 예외).
- `src/hero/MediaCollage.tsx` (+ `.module.css`, `scrub.ts`) — 풀스크린 미디어가 `clip-path`로 중앙 카드로 축소되며 주변 타일이 슬라이드 인하는 스크럽 섹션. transform/clip-path/opacity만 사용(reflow 회피), reduced-motion이면 완성 콜라주 정적 표시.

### 2.2 날짜 유틸 — Intl 기반 (기커밋)

`src/lib/date.ts`. 서버의 offset 없는 LocalDateTime에 `+09:00`을 명시 부착해 파싱(`parseServerDate`)하고, `Intl.DateTimeFormat`을 `Asia/Seoul` 고정으로 써서 서버 런타임 TZ와 무관하게 KST 표기를 보장한다. EventCard 시간줄(`formatEventTime`)은 allDay·당일·여러 날 케이스를 분기 처리한다. **date-fns는 도입하지 않았다**(아래 3.1).

### 2.3 `cn()` 커스텀 타이포 병합 버그 수정 (미커밋분, 커밋 `c81c677`)

- 파일: `src/lib/utils.ts` (+ 회귀 테스트 `utils.test.ts`)
- **무엇/왜**: tailwind-merge 기본 설정이 미지의 `text-*` 클래스를 **색상 그룹**으로 분류한다. 그래서 `cn(typo.displayLg, "text-ink")` 같은 조합에서 크기 클래스(`text-display-lg`)가 색상과 충돌한 것으로 오인돼 제거됐고, **T07 이후 전 섹션 헤딩이 16px로 붕괴**해 있었다.
- **어떻게**: `extendTailwindMerge`로 16개 커스텀 타이포 토큰(`display-mega`~`nav-link`)을 `font-size` 그룹에 명시 등록. 새 `--text-*` 토큰 추가 시 이 목록에도 등록하도록 주석으로 못 박았고, `utils.test.ts`가 회귀를 감시한다.

### 2.4 고령 사용자 타이포·UI 확대 (미커밋분, 커밋 `ca0ef10`)

- 파일: `src/app/globals.css`, `src/components/ui/Button.tsx`, `.claude/rules/DESIGN.md`
- 본문 16→**20px**, 보조 14→**18px**, datetime 14→**18px**, nav-link 15→**21px**, title-md 18→**22px**, button 16→**20px** 등 `--text-*` 토큰을 일괄 상향. 헤더 높이 64→**80px**, Button 높이 h11→**h12(48px)**. "15px 미만 텍스트 금지" 원칙을 DESIGN.md와 동기화.

### 2.5 연혁·사역 섹션 (미커밋분)

- 파일: `src/components/main/HistoryBand.tsx`, `MinistryCards.tsx`, `Reveal.tsx`(+ `.module.css`), 콘텐츠는 `src/constants/content.ts`의 `HISTORY`·`MINISTRIES` 확장.
- **연혁(HistoryBand)**: 연도 배지 + 헤드라인 + 설명의 풀폭 라운드 밴드를 세로로 이어 붙인다. 상수 배열(`HISTORY.items`) 편집만으로 항목을 추가할 수 있다.
- **사역(MinistryCards)**: lucide 아이콘(32, currentColor) + 제목 + 설명의 3-up 카드. 아이콘 키는 상수에 직렬화 가능한 문자열로 두고 컴포넌트가 lucide로 매핑한다.
- 두 섹션 모두 참조의 브랜드 3색을 그대로 쓰지 않고 `surface-dark`·`primary-soft`·`surface-soft` **토큰 교차**로 재해석(단일 액센트 원칙). 인접 중복을 피하려 시작 톤을 서로 다르게 둔다.
- **Reveal**: `IntersectionObserver`로 뷰포트 진입 시 **1회** fade+slide-up. 카드별 스태거(`delay = i*120`). reduced-motion이면 IO를 등록하지 않고 CSS가 즉시 표시.

### 2.6 메가메뉴 헤더 개편 (미커밋분)

- 파일: `src/components/shell/SiteHeader.tsx`, `MegaMenu.tsx`, `MobileNav.tsx`, `src/constants/navigation.ts`, `src/lib/nav.ts`
- **IA 4컬럼 재편(M1)**: 교회안내·예배·설교·교육부서·소식. `navigation.ts`가 단일 출처라 헤더·모바일·푸터가 함께 재편된다. 라우트 없는 목업 항목(부서별 등)은 발명하지 않고 T9/T13 이후로 미룬다.
- **풀폭 메가메뉴(M2/M3)**: 1뎁스 호버/포커스 시 헤더 아래 흰 패널에 4컬럼 전체를 펼친다. 아이콘은 단일 틴트 플레이트(`rounded-full` 40px, `bg-primary-soft` + `text-primary`)로 재해석(목업의 4색 → 단일 액센트).
- **인증 단일 링크(M4)**: `useAuthStore`의 member 스냅샷 유무로 마이페이지/로그인 중 하나만 노출(헤더·모바일 공통). 기존 NAV_AUTH 배열을 NAV_LOGIN·NAV_MYPAGE로 분리.
- **투명 헤더 충돌 방지(M5)**: 메인의 투명 on-dark 헤더에서 패널이 열리면 라이트 스킨으로 강제(`solid ∨ megaOpen`) — 흰 패널 위 흰 글씨 방지.
- **접근성**: 패널에 `aria-hidden`, 링크에 `tabIndex`(닫힘 시 -1), 헤더 `onKeyDown` Esc 닫힘, `onBlur`로 포커스가 헤더 밖으로 나가면 닫힘(WCAG 1.4.13), mouseleave 닫힘.

## 3. 설계 결정

- **date-fns 미사용 (Intl 채택)**: `date-fns format()`은 런타임 로컬 TZ 기준이라 비KST 서버에서 어긋난다. 메인의 날짜 표기는 `Intl.DateTimeFormat` + `Asia/Seoul` 고정으로 처리하고, date-fns 직접 구현은 캘린더(T12)로 미뤘다.
- **`connection()` vs `force-dynamic`**: CI가 백엔드 없이 빌드하므로 prerender를 피해야 하지만, `force-dynamic`은 fetch 데이터 캐시(revalidate 60)까지 죽인다. `await connection()`은 prerender만 요청 시점으로 미루고 캐시는 살린다 — 백엔드 부하를 60s 캐시가 흡수.
- **단일 액센트 토큰 원칙**: 연혁·사역·메가메뉴 모두 참조 사이트의 다색 팔레트를 그대로 옮기지 않고 기존 surface/primary-soft 토큰 교차로 재해석. hex 추가 0.
- **콘텐츠 상수 주입**: 모든 사용자 노출 텍스트는 `content.ts`·`church.ts`·`navigation.ts` 상수 또는 API에서 온다. 배포 시 이 상수 파일만 교체.
- **직렬화 가능한 아이콘 키**: 상수에는 lucide 컴포넌트가 아니라 문자열 키만 두고, 클라이언트 컴포넌트(MegaMenu·MinistryCards)가 lucide로 매핑한다(서버→클라 경계 안전).
