# 액션 버튼(추가·수정·삭제·저장·취소) 형식 일관성 통일

### 📌 작업 개요
코드베이스 전반에서 같은 의미의 액션 버튼이 화면마다 색·강조·라벨·아이콘이 제각각이던 문제를 해소. 의미(semantics)를 기준으로 버튼 형식을 고정하는 단일 규칙표를 만들고, 어드민·콘텐츠·갤러리·마이페이지·인증 전 영역에 일괄 적용.

**변경 규모**: 43개 파일, +301 / −97

### 🎯 구현 목표
- 의미가 같은 액션은 항상 같은 형식(색·강조·라벨·아이콘·위치)으로 렌더
- 디자인 시스템 `Button`을 우회하던 맨버튼/맨링크를 흡수
- 라벨 흔들림(역할 추가/새 X, 저장/변경) 정리
- 다이얼로그 하단 버튼 순서·취소 버튼 통일
- 의미가 다른 액션(연결 해제 vs 영구 삭제)은 구분 유지

### 🧭 적용 규칙 (의미 → 형식)
| 액션 | 강조(variant) | 라벨 | 아이콘 |
|---|---|---|---|
| 생성/추가 | primary | `새 {엔티티}` | Plus |
| 수정 | tertiary | `수정` | Pencil |
| 삭제(트리거) | tertiary | `삭제` | Trash2 |
| 삭제(확인) | destructive | `삭제` | 없음 |
| 저장/제출 | primary | `저장` | 없음 |
| 취소 | tertiary | `취소` | 없음 |

- 강조 구분 규칙: 행 안 액션 = tertiary, 행 밖 보조 = secondary, 주 CTA(생성/저장) = primary(화면당 하나)
- 파괴적 액션의 빨강(destructive)은 확인 모달에서만 사용. 목록 행의 삭제 트리거는 중립색 유지
- 연결 해제(역할 회수·사진 제거)는 영구 삭제가 아니므로 `회수`/`사진 제거` 라벨과 X 아이콘을 유지(Trash2/삭제와 구분)

### ✅ 구현 내용

#### 기반 — 공용 컴포넌트·상수
- **파일**: `src/components/ui/Button.tsx`, `src/constants/actionButton.ts`
- **변경 내용**: `Button`에 아이콘 단독(정사각) 형태를 위한 `iconOnly` prop 추가. 액션 라벨·아이콘을 `ACTION`(수정·삭제·저장·취소)·`CREATE_ICON`·`createLabel` 상수로 중앙화
- **이유**: 라벨·아이콘을 한 곳에서 관리해 도메인별 흔들림을 원천 차단(상수가 일관성을 강제)

#### 파괴 확인 다이얼로그 — 취소 버튼 표준화
- **파일**: `src/components/admin/DeleteConfirmDialog.tsx`
- **변경 내용**: 하단 버튼을 `[취소(tertiary), 삭제(destructive)]`로 통일. 취소는 `DialogClose`로 닫기 처리
- **이유**: 이 컴포넌트 하나를 12개 도메인(주보·설교·공지·일정·역할·부서·태그·직분·미디어·갤러리·약관 등)이 공유하므로, 한 번의 수정으로 모든 파괴 확인에 취소 버튼이 적용됨

#### 어드민 관리 — 역할·직분·태그
- **파일**: `roles/RoleManager.tsx`, `positions/PositionManager.tsx`, `tags/TagManager.tsx` 및 각 폼 다이얼로그
- **변경 내용**: 생성 버튼에 Plus 아이콘(`역할 추가`→`새 역할`로 정리), 목록 행 수정/삭제를 아이콘+텍스트로, 폼 다이얼로그 `[취소, 저장]`. 권한 게이팅(비활성·툴팁)은 그대로 보존
- **특이사항**: `RoleManager`에만 있던 일회성 커스텀 스타일(`actionCls`) 제거, 행 권한 버튼에 KeyRound 아이콘 부여

#### 어드민 — 부서·회원·미디어
- **파일**: `departments/DepartmentTree.tsx`, `members/*`, `media/MediaLibrary.tsx`
- **변경 내용**: 트리 펼치기 화살표·역할 회수 버튼을 아이콘 단독 버튼으로 흡수, 미디어 열기 링크를 버튼 스타일로, 삭제를 아이콘+텍스트로
- **특이사항**: 역할 회수는 연결 해제이므로 `회수` 라벨·X 아이콘 유지. 오버레이 버튼은 위치·배경 스타일을 보존

#### 콘텐츠 — 주보·설교·공지·일정
- **파일**: `bulletins/*`, `sermons/*`, `notices/*`, `events/*`
- **변경 내용**: 4개 도메인의 생성 버튼에 Plus, 목록 행 수정/삭제를 회색(secondary)에서 텍스트형(tertiary)+아이콘으로 정렬, 폼은 다이얼로그/인라인 모두 `[취소, 저장]` 순서로 통일

#### 갤러리
- **파일**: `gallery/GalleryAdminActions.tsx`, `GalleryPhotoManager.tsx`, `PhotoLightbox.tsx`, `AlbumFormDialog.tsx`
- **변경 내용**: 새 앨범 생성 아이콘, 사진 제거 오버레이·라이트박스 이전/다음 버튼을 아이콘 단독 버튼으로 흡수, 앨범 폼 `[취소, 저장]`
- **특이사항**: 사진 제거는 앨범에서의 연결 해제(원본 보존)이므로 `사진 제거` 라벨·X 아이콘 유지

#### 마이페이지·인증
- **파일**: `mypage/*`, `auth/SignupForm.tsx`
- **변경 내용**: 로그아웃을 빨강(destructive)에서 보조색(secondary)으로(로그아웃은 파괴 액션 아님), 비밀번호 변경 제출을 `변경`→`저장`으로, 프로필 수정에 아이콘, 폼 순서 `[취소, 저장]`. 가입 위저드는 이미 규칙과 일치해 변경 없음

#### 부서 폼 다이얼로그 — 누락 보강
- **파일**: `departments/DepartmentFormDialog.tsx`
- **변경 내용**: 다른 폼 다이얼로그와 동일하게 `[취소(tertiary, DialogClose), 저장]` + 상수 라벨로 통일
- **이유**: 종합 점검 단계에서 이 폼만 옛 패턴(취소 회색·하드코딩 라벨)이 남아 있던 것을 발견해 보강. 이로써 6개 폼 다이얼로그가 모두 동일 패턴

- 프로덕션 빌드 정상
- 잔존 맨버튼은 전부 비대상(디자인 시스템 내부·네비·검색·콘텐츠 클릭 영역·캘린더)임을 확인 — 누락된 액션 버튼 없음

### 📌 참고사항
- **인라인 텍스트 링크는 비대상**: 재동의하기·약관 전문 보기·회원 탈퇴 트리거는 의도된 인라인 텍스트 링크라 버튼으로 흡수하지 않음
- **시각 스폿체크 권장**: 모바일(좁은 화면)에서 목록 행 버튼이 아이콘만 노출되는 동작, 역할 회수 칩 버튼 크기는 실제 화면에서 한 번 확인 권장
- 교회별 브랜드 색 교체 시에도 버튼 규칙은 그대로 유지됨(색은 토큰 참조)
