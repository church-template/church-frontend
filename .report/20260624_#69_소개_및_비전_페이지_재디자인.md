# 소개 및 비전 페이지 빅타이포 에디토리얼 재디자인 (#69)

## 📌 작업 개요
`/about` 소개 및 비전 페이지를 **빅타이포 에디토리얼** 스타일로 재디자인. 단일 페이지를 5개 밴드 컴포넌트로 분리하고, 흰→회색→흰→회색→흰 밴드 리듬으로 페이지 호흡을 구성. 콘텐츠는 모두 `content.ts` 상수에서 주입(상수 구동 서버 컴포넌트). 미사용 고아 페이지 `/about/vision` 제거.

## 🎯 구현 목표
- 소개 페이지를 디자인 시스템 토큰 기반 빅타이포 레이아웃으로 재구성
- 밴드별 컴포넌트 분리로 응집도 향상
- 로고 4색 신학 상징을 에디토리얼 넘버드 리스트로 표현
- 비전 6항목을 아이콘 카드 그리드로 표현

## ✅ 구현 내용

### 페이지 오케스트레이터 단순화
- **파일**: `src/app/(site)/about/page.tsx`
- **변경 내용**: 인라인 마크업을 제거하고 5개 밴드 컴포넌트(`VisionHero`·`SymbolismList`·`HopeStatement`·`VisionGoals`·`ChurchStory`)를 순서대로 조합
- **이유**: 밴드별 책임 분리로 가독성·유지보수성 확보. 페이지는 조립만 담당

### 밴드1 — VisionHero (흰 캔버스)
- **파일**: `src/components/about/VisionHero.tsx`
- **변경 내용**: 좌측 빅타이포(라벨·대표문구·인트로) + 우측 로고(`/onlyLogo.png`) 2열 레이아웃. 대표문구에서 강조어(`축복`)만 `text-primary`로 표시하는 `renderStatement` 헬퍼 포함
- **이유**: 단일 액센트 원칙(강조 단어 하나만 primary). 모바일은 로고를 위로 올려 가운데 정렬

### 밴드2 — SymbolismList (회색)
- **파일**: `src/components/about/SymbolismList.tsx`
- **변경 내용**: 로고 4색을 `01–04` 넘버드 에디토리얼 리스트로 표현. 색은 작은 점(`size-3`)과 헤어라인만 사용
- **이유**: 큰 색면 없이 점·가는 룰로만 색을 노출해 단일 액센트 원칙 유지

### 밴드3 — HopeStatement (흰 캔버스)
- **파일**: `src/components/about/HopeStatement.tsx`
- **변경 내용**: 좌측 소제목 / 우측 큰 본문(`body-lg` lead) 2열 구성

### 밴드4 — VisionGoals (회색)
- **파일**: `src/components/about/VisionGoals.tsx`
- **변경 내용**: 비전 6항목을 lucide 아이콘 카드 그리드로(데스크톱 3-up·태블릿 2-up·모바일 1-up)
- **이유**: `VISION.points` 순서에 대응하는 아이콘 매핑(표현용)

### 밴드5 — ChurchStory (흰 캔버스)
- **파일**: `src/components/about/ChurchStory.tsx`
- **변경 내용**: 좌측 소제목 / 우측 이야기 본문(`body-lg` lead) 2열 구성

### 디자인 토큰 추가
- **파일**: `src/app/globals.css`, `src/constants/typography.ts`, `src/lib/utils.ts`
- **변경 내용**:
  - 로고 신학 상징색 4종 추가(`--color-symbol-blue/green/red/orange`)
  - 읽는 본문 강조 토큰 `--text-body-lg`(24px·행간 1.7·400) 추가 → `typo.bodyLg`로 노출
  - `extendTailwindMerge` 목록에 `body-lg` 등록(twMerge 토큰 오인 방지)
- **이유**: hex·px 인라인 금지 규칙 준수. 상징색은 브랜드 팔레트가 아닌 콘텐츠 색이므로 별도 토큰으로 분리

### 콘텐츠 상수 추가
- **파일**: `src/constants/content.ts`
- **변경 내용**: `statementHighlight`(강조 단어), `symbolismHeading`, `symbolismLead` 추가
- **이유**: 사용자 노출 텍스트 하드코딩 금지 — 모든 콘텐츠를 상수에서 주입

### 고아 비전 페이지 제거
- **파일**: `src/app/(site)/about/vision/page.tsx`, `vision/page.test.tsx` (삭제)
- **이유**: 재디자인으로 비전 내용이 `/about`에 통합되어 미사용 라우트가 됨

## 🔧 주요 변경사항 상세

### renderStatement 헬퍼 (VisionHero)
대표문구 문자열에서 강조어 위치를 찾아 해당 부분만 `<span className="text-primary">`로 감싼다. 강조어가 없으면 평문 그대로 반환하여 예외 방어.

**특이사항**:
- 상징색·아이콘은 콘텐츠 배열 순서에 인덱스로 매핑(표현용). 데이터에 색/아이콘을 박지 않고 컴포넌트에서 매핑
- 모든 밴드에 `Reveal` 스태거 애니메이션(`delay` 점증) 적용

## 🧪 테스트 및 검증
- 각 밴드 컴포넌트별 테스트 추가: `VisionHero`·`SymbolismList`·`HopeStatement`·`VisionGoals`·`ChurchStory`
- `about/page.test.tsx` 갱신, `utils.test.ts`에 `body-lg` 토큰 병합 케이스 추가
- 장식 이미지/요소는 `aria-hidden` 처리(상징 점·아이콘)

## 📌 참고사항
- 페이지는 상수 구동 서버 컴포넌트 — 백엔드 연동 불필요(정적 생성)
- 상징색 토큰은 단일 액센트 원칙상 큰 색면·버튼에 쓰지 않고 점·가는 룰·인라인 강조에만 사용
