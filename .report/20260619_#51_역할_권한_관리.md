# 역할·권한 관리 (07A) — 구현 리포트

이슈: #51

---

## 📌 작업 개요

`/mypage/manage/roles` 어드민 화면. `ROLE_MANAGE` 권한 게이트 안에서 역할 CRUD와 역할별 권한 일괄 설정을 제공한다. 태그·직분(#40)과 동형 구조이나, **위계 가드**(시스템 역할·상위 등급 보호)와 **권한 다이얼로그**(카탈로그 체크박스 + PUT 전체 교체)가 추가된 구조다.

---

## 🎯 구현 목표

- 역할 목록 조회 + 등록·수정·삭제
- 역할별 권한 일괄 설정 (권한 이름 배열 PUT — 전체 교체)
- 위계 가드: 시스템 역할·내 등급 초과 역할은 수정·권한·삭제 비활성
- me 미로딩 전 과도 비활성 방지
- 07B(회원 관리) 재사용 인터페이스 설계

## ✅ 구현 내용

### 데이터 레이어

**`src/lib/api/permissions.admin.ts`**
- `getPermissions()` — `GET /api/admin/permissions`, 권한 카탈로그 평배열 반환
- `PermissionResponse` 인터페이스 도메인-로컬(공개 GET 없음, `media.admin.ts` 선례)

**`src/lib/api/roles.admin.ts`**
- `RoleResponse` — `version` 없음. 역할은 낙관락·단건 GET 미존재(태그·직분과 동형). 수정·권한 시드는 목록 행 값 직접 사용
- CRUD: `getRoles` / `createRole` (POST) / `patchRole` (PATCH) / `deleteRole` (DELETE)
- 권한 교체: `setRolePermissions(id, names[])` — `PUT /api/admin/roles/{id}/permissions`, `{ permissions: names }` 전송. 권한 **이름** 배열(id 아님), 빈 배열 = 전 권한 회수

**`src/lib/admin/roleGuards.ts`**
- `canManageRole(role, maxPriority)` — `!role.isSystem && role.priority <= maxPriority`
  - 시스템 역할: 항상 false
  - 내 등급 초과: false
  - 같은 등급(`<=`): true (동급 허용)
  - 서버 `validateMutable`과 동일 조건 — 화면은 UX 선제 비활성, 서버가 최종 403 방어
  - **07B 재사용 금지** 주석 명시: 회원 역할 부여/회수는 `isSystem`을 막으면 MEMBER(시스템 역할) 승인이 차단되므로 별도 `canAssignRole = priority < maxPriority` (strict 부등호, 동급 차단) 필요

## 🔧 주요 변경사항 상세

### 위계 가드 설계

```
편집/삭제/권한: !isSystem && priority <= maxPriority  (동급 허용, UX 가드)
역할 부여/회수: priority < maxPriority               (동급 차단, 07B 전용, 별도 구현 필요)
```

서버가 최종 403을 방어하므로 화면 가드는 UX 선제 비활성 역할에 집중한다.

### me 로딩 전 과도 비활성 제거

기존 `0` 폴백은 me 미정 시 전 역할을 비활성했다. `undefined` 유지 + 조건 분기로 시스템 역할만 비활성하고 나머지는 서버에 위임한다.

### 권한 다이얼로그 시드 패턴

`key={role.id}` keyed 마운트 + `useState(() => new Set(role.permissions.map(...)))` 초기화로 시드. 형제 Dialog(edit 시드)와 달리 `open` prop이 아닌 컴포넌트 생애주기 자체로 초기화하므로 effect 완전 불필요.

### 무효화 규칙 정리

| 작업 | `adminKeys.list("roles")` | `["me"]` |
|---|---|---|
| createRole | ✅ | ✗ |
| patchRole | ✅ | ✅ |
| deleteRole | ✅ | ✗ |
| setRolePermissions | ✅ | ✅ |
