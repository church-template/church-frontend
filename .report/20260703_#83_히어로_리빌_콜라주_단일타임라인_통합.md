### 📌 작업 개요

메인 홈 데스크톱 스크롤 연출의 이음새·사진 중복 노출 문제 해결. 기존에 분리돼 있던 `CrossHero`(십자가 리빌)와 `MediaCollage`(카드 축소 콜라주) 두 sticky 섹션을 **단일 sticky·단일 영상 요소**로 통합한 `HeroReveal` 신규 컴포넌트로 대체. 십자가로 드러난 그 영상 한 장이 그대로 축소되도록 하나의 스크롤 타임라인으로 이었다.

### 🔍 문제 분석

- **이음새 중복**: `CrossHero`(320vh)와 `MediaCollage`(220vh)가 문서 흐름상 순차 배치라, 히어로 마지막 풀스크린 프레임이 밀려나는 동안 아래에서 콜라주의 **동일 영상** 풀스크린 프레임이 올라오는 약 100vh 구간이 발생. 사용자 눈에 "같은 사진이 한 번 더" 나오는 중복으로 보임.
- **프레임 점프 위험**: 두 섹션이 각각 별개 `<video>`라 재생 시점 어긋남 발생 가능.

### 🎯 구현 목표

- 데스크톱에서 단일 sticky·단일 `<video>`로 "십자가 리빌 → 홀드 → 카드 축소 → 타일 진입" 연속 타임라인 구현
- 4-타일 콜라주 유지
- 헤더 투명↔솔리드 전환을 축소 시작 시점(흰 캔버스 노출)에 정확히 맞춤
- 십자가·콜라주 확정 수치(비율·시작크기·clip 목표)는 재설계 없이 그대로 계승

### ✅ 구현 내용

#### 신규 통합 컴포넌트 `HeroReveal`
- **파일**: `src/hero/HeroReveal.tsx`, `src/hero/HeroReveal.module.css`
- **변경 내용**: 두 원본의 sticky 레이어를 하나로 병합. 내부 `RevealMedia` 헬퍼로 `onError→poster` 폴백을 중앙 카드·타일 공용 1벌로 통합. 데스크톱은 스크롤 진행도 비례 단일 `requestAnimationFrame` 스크럽 하나가 십자가 리빌 + 카피 인/홀드/아웃 + 카드 축소 + 타일 스태거를 모두 구동.
- **이유**: 영상 요소를 1개로 합치면 이음새·중복·프레임 점프가 구조적으로 사라짐.

#### 헤더 동기화 배선 변경
- **파일**: `src/components/main/HeroHeaderSync.tsx`
- **변경 내용**: `CrossHero`→`HeroReveal` 교체. 기존 IntersectionObserver(400vh 이탈 시 발화 = 타이밍 반대) 제거하고 `onSolid(boolean)` 콜백 방식 채택. `tiles` prop 추가, `useMediaFlag(MOBILE_MQ)` 추가, `solid` 상태 도입.
- **이유**: 드라이버가 매 프레임 진행도 `p`를 알아 `p>=0.5`(축소 시작 직전)에서 정확히 헤더 solid 통지 가능. 스크롤 업 시 임계 재교차로 투명 양방향 복귀.

#### 페이지 조립 변경
- **파일**: `src/app/page.tsx`
- **변경 내용**: `MediaCollage` import·자식 렌더 삭제. `COLLAGE_TILES`를 `HeroHeaderSync`의 `tiles` prop으로 전달. 나머지 섹션(HistoryBand 등)은 children 유지.

#### 설계 스펙 문서
- **파일**: `docs/superpowers/specs/2026-07-03-hero-reveal-merge-design.md`
- **변경 내용**: 통합 배경·DOM 트리·통합 타임라인 상수 `P`·모바일/reduced 처리(Option A)·회귀 주의 항목을 담은 신규 설계 문서.

### 🔧 주요 변경사항 상세

#### 통합 타임라인 상수 `P` (튜닝 가능)
400vh 섹션 → 300vh 스크롤을 단일 진행도 `p`(0~1)로 정규화하고 페이즈별 구간창으로 분배:

| 구간 | 값 | 처리 |
|---|---|---|
| crossIn | [0.0, 0.34] | easeIn 십자가 열쇠구멍 확대 |
| capIn | [0.36, 0.46] | 카피 페이드/슬라이드 인 |
| (홀드) | 0.46–0.54 | 선명한 풀스크린 영상 + 카피 |
| solid | 0.50 | `onSolid(true)` 임계 — 흰 캔버스 노출 직전 |
| shrink | [0.54, 0.84] | easeOut clip-path 카드 축소 |
| capOut | [0.54, 0.62] | 카피 페이드 아웃 |
| 타일 | 0.62~0.96 | T1~T4 스태거 슬라이드 인 |

각 페이즈 수학은 기존 `CrossHero`·`MediaCollage`에서 그대로 재사용하고 구간창만 통합 타임라인으로 이동. 카피 퇴장은 `opacity = capIn * (1 - capOut)` 곱셈 방식.

#### 모바일 / reduced-motion 처리 (Option A)
영상을 1개로 합치면 "모바일에서 영상 두 번 노출"이 구조적으로 불가능해짐. 채택안:
- **모바일(<640px)**: 십자가 인트로 생략, 영상을 카드로 바로 표시 + 타일 4장 IntersectionObserver 1회 슬라이드 인(스크럽 없음). 카피는 스택 상단 정적 제목(`--color-ink`)으로 이동(`order:-1`).
- **reduced-motion 데스크톱**: 풀스크린 인트로 생략, 완성된 콜라주(중앙 카드 clip-path 고정 + 타일 정적) 표시. 카피는 카드 위 중앙 유지.
- JS는 reduced·mobile 분기에서 조기 return하고 CSS 미디어쿼리가 정적 상태를 표시.

**특이사항**:
- **cleanup 시 인라인 스타일 클리어 필수**: 브레이크포인트 전환 시 데스크톱 스크럽이 남긴 인라인 스타일(clip-path·opacity·transform)이 모바일/reduced CSS를 덮지 않도록 언마운트에서 `""`로 초기화.
- **라디우스 토큰 1회 읽기**: `--radius-xl`을 `getComputedStyle`로 1회만 읽어 clip-path 합성. hex/px 인라인 금지 준수(십자가 SVG 좌표·rgba 커버·`START_PCT` 등은 사전 승인된 검증 수치 예외).
- 애니메이션은 transform/clip-path/opacity만 사용(reflow 금지), `will-change`는 원본 수준 유지.
