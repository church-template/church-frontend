# 마크다운 에디터 서식 툴바·도움말 추가 (#106)

### 📌 작업 개요

공지·설교·일정 등 본문 작성이 마크다운 문법 기반이라, 마크다운을 모르는 관리자는 서식(제목·굵게·목록 등)을 적용하지 못하고 이미지도 `media:{id}` 코드를 손으로 입력해야 했다. 어드민 마크다운 에디터에 서식 버튼 툴바·삽입 다이얼로그·사용법 도움말을 추가해, 마크다운을 몰라도 클릭만으로 서식을 적용하도록 개선했다. 사용처 6개 폼(공지·설교·일정·앨범·챌린지·부서)은 무변경이며, 모든 확장은 `MarkdownEditor` 내부에서 끝난다.

### 🎯 구현 목표

- 항상 보이는 서식 버튼 툴바로 발견성 확보(대상이 마크다운을 모르는 관리자)
- 링크·유튜브·표·이미지는 입력 다이얼로그로 받아 완성형 삽입 — 문법·코드 몰라도 됨
- 화면 안에서 바로 보는 사용법 도움말(치트시트)
- WYSIWYG 라이브러리 도입 없이(허용 스택 유지) textarea + 마크다운 저장 파이프라인 그대로 유지

### 🔧 주요 변경사항 상세

#### 삽입 로직 규칙 (markdownEditing.ts)
- 인라인: 선택 영역을 마커로 감싸고 이미 감싸져 있으면 해제(토글). 선택 없으면 자리표시 문구(굵은 글씨 등) 삽입 후 문구만 선택
- 줄 단위: 선택이 걸친 모든 줄 접두 토글, 제목끼리는 단계 교체(`##`에서 H3 누르면 `###`)
- 블록(구분선·표·유튜브·이미지): 앞뒤 빈 줄을 보장한 뒤 단독 문단으로 삽입 — 유튜브 임베드·문단 분할이 빈 줄 2개에 의존

**특이사항**:
- 입력창에는 `**` 같은 마크다운 기호가 그대로 보인다(WYSIWYG 아님) — 결과 확인은 미리보기 탭·치트시트가 담당(승인된 트레이드오프)
- 문서가 개행으로 시작할 때 첫 위치에서 줄 서식을 누르면 본문이 바뀌던 오프바이원 버그를 수정(`text.slice(0, selStart).lastIndexOf("\n")`)

#### Dialog 삽입 경로 (실사용 버그 수정)
직접 툴바 버튼은 실행취소(Ctrl+Z) 보존을 위해 네이티브 삽입(`execCommand`) 우선, 미지원·실패 시 상태 교체로 폴백한다. 반면 **다이얼로그 삽입은 상태 교체로 직행**한다.

**특이사항**:
- 다이얼로그가 열린 채 삽입이 실행되면 다이얼로그의 포커스 트랩이 `focus()`를 되가져가 `execCommand`가 다이얼로그 입력에 꽂힌 채 성공(true)을 반환, 폴백까지 막혀 본문이 갱신되지 않던 버그가 있었다. 실사용 리포트로 발견 → 다이얼로그 경로는 네이티브를 건너뛰도록 수정하고 회귀 테스트 2건 추가

#### 표 크기 지정
고정 2×2 템플릿 대신 열(1~8)·행(1~20)을 숫자로 입력받아 `buildTable`이 자리표시 헤더(제목1·제목2…)+구분행+빈 본문 행을 생성한다. 세 번째 다이얼로그 등장을 계기로 링크·유튜브·표가 `InsertDialogShell` 공용 뼈대를 공유하도록 정리했다.
