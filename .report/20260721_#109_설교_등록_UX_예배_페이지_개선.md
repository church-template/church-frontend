# 설교 등록 UX 개선 및 예배 안내 페이지 보강 (#109)

### 📌 작업 개요

설교 등록 폼에서 매번 손으로 채우던 설교자·설교일을 기본값으로 자동 채움 처리하고, 설교일 연도 입력을 4자리로 제한. 예배 시간 안내 카드의 시간 표기를 primary 색 강조로 상향하고, 예배 페이지에서 설교 목록으로 가는 진입 버튼 추가.

**보고서 파일**: `.report/20260721_#109_설교_등록_UX_예배_페이지_개선.md`
**PR**: #110 (`20260720_#109_설교_등록_UX_예배_페이지_개선` → `main`)

### 🎯 구현 목표

- 설교자 칸에 교회별 기본 설교자 자동 입력 (템플릿이므로 상수 주입, 하드코딩 금지)
- 설교일 연도 6자리 입력 문제 해결 — 4자리 입력 시 월로 자동 이동
- 설교일 기본값으로 오늘 날짜(한국 시간 자정 기준 전환) 자동 입력
- 예배 카드에서 핵심 정보인 시간이 예배명에 묻히는 문제 해결
- 예배 안내 → 설교 목록 진입 동선 신설

### ✅ 구현 내용

#### KST 오늘 날짜 헬퍼 추가
- **파일**: `src/lib/date.ts`, `src/lib/date.test.ts`
- **변경 내용**: `todayKstDate()` 추가 — `Intl.DateTimeFormat("en-CA", { timeZone: "Asia/Seoul" })`로 `YYYY-MM-DD` 반환
- **이유**: 실행 환경(서버·브라우저)의 로컬 타임존과 무관하게 한국 시간 자정 기준으로 날짜가 바뀌어야 함. en-CA 로케일은 YYYY-MM-DD를 그대로 내서 문자열 조립 불필요

#### 설교 등록 폼 기본값·연도 제한
- **파일**: `src/constants/church.ts`, `src/components/sermons/SermonForm.tsx`, `src/components/sermons/SermonForm.test.tsx`
- **변경 내용**: `SERMON_DEFAULT_PREACHER` 상수 신설, 등록 모드 defaultValues에 설교자 상수·`todayKstDate()` 프리필, 설교일 입력에 `max="9999-12-31"` 부여
- **이유**: 교회별 값은 church.ts가 주입 지점(교회 교체 시 상수 한 줄만 변경). `max`가 크롬 date 입력의 연도 세그먼트를 4자리로 제한해 4자리 입력 시 월로 자동 이동. 수정 모드는 `initial?.x ?? 기본값` 구조로 기존 값 유지

#### datetime-lg 타이포 위계 신설
- **파일**: `.claude/rules/DESIGN.md`, `src/app/globals.css`, `src/constants/typography.ts`, `src/lib/utils.ts`, `src/lib/utils.test.ts`
- **변경 내용**: 22px/600/tnum 위계를 디자인 시스템 절차대로 4단계 등록 (DESIGN.md 표 → `@theme --text-datetime-lg` → `typo.datetimeLg` → twMerge font-size 목록)
- **이유**: 강조용 날짜·시간 위계가 없었음. twMerge 목록 누락 시 `cn()`이 크기 클래스를 색상으로 오인해 제거하는 기존 함정이 있어 회귀 테스트까지 함께 추가

#### 예배 카드 시간 강조 + 설교 진입 CTA
- **파일**: `src/components/worship/WorshipRegular.tsx`, `src/components/worship/WorshipRegular.test.tsx`, `src/constants/content.ts`
- **변경 내용**: 카드 대표시간을 `typo.datetimeLg` + `text-primary`로 상향(기존 18px 회색), 카드 그리드 아래 "설교 말씀 보기" primary 버튼(`/sermons` 링크) 추가. 라벨은 `WORSHIP.sermonCta` 상수
- **이유**: 예배 안내 페이지의 핵심 정보는 시간인데 예배명(22px 굵게)보다 묻혀 있었음. 시간은 곧 행동 정보라는 판단으로 "단일 액센트" 원칙의 페이지 한정 예외를 확정(실화면 확인 후 결정, 코드 주석·스펙 결정 표에 근거 기록)

### 🔧 주요 변경사항 상세

#### 연도 4자리 제한의 동작 원리
크롬 date 입력은 연도가 최대 6자리(275760년)까지 유효해서 4자리를 채워도 다음 세그먼트로 넘어가지 않고 대기. `max="9999-12-31"`을 주면 연도 상한이 4자리가 되어 4자리 입력 즉시 월로 자동 이동. 실브라우저(Chromium)에서 "20260721" 연속 타이핑 → `2026-07-21` 입력 완성 확인.

**특이사항**:
- 백스페이스 연속 삭제(세그먼트 넘어 삭제)는 브라우저 네이티브 동작 한계로 이번 범위에서 제외. 오늘 날짜 기본값 + 세그먼트 덮어쓰기 입력으로 "지울 일" 자체가 거의 사라지는 방식으로 해결. 사용 후에도 불편하면 커스텀 3분할 입력을 별도 이슈로 검토

#### dev 서버 재시작 필요 (로컬 확인 시 주의)
`globals.css`의 `@theme` 토큰 추가는 실행 중인 dev 서버에 반영되지 않음. 재시작 전에는 `text-datetime-lg` 클래스가 빈 토큰을 참조해 16px로 폴백(실측 확인). 로컬에서 변경이 안 보이면 dev 서버 재시작 먼저 확인.

#### 예배별 설교 필터는 보류
"새벽예배 클릭 → 새벽예배 설교만" 아이디어는 설교 데이터에 예배 구분 축이 없어 분류 운영(태그 관례 또는 백엔드 필드)이 전제됨. 이번에는 진입 CTA만 넣고, 분류 데이터가 생기면 재논의하기로 확정. 설교 목록이 회원전용이라 비회원은 클릭 시 기존 로그인·승인 안내를 만나는 동작 유지.
