# 학생부 부서 상세 페이지(`/departments/student`) UI/UX 개선 #78

### 📌 작업 개요
소개 문단 하나만 있던 부서 상세 페이지(`/departments/[slug]`)에, 학생부(`student`)를 시작으로 소개·알림사항·주요활동·특별프로그램·활동사진·초대 6개 섹션을 보강.
데이터는 전부 옵션 필드로 설계해 학생부 외 부서(청년부·예배부·남선교회·여선교회)는 필드를 채우지 않으면 기존 화면 그대로 유지된다.

### 🎯 구현 목표
- 빈 화면에 가까웠던 학생부 상세를 소개·정보·활동·프로그램·사진·초대로 채워 실제 콘텐츠 페이지로 완성
- 부서 데이터 모델을 확장하되 기존 필드·기존 부서는 건드리지 않아 회귀 0
- 신규 섹션은 기존 컴포넌트 패턴(아이콘 맵·카드·라이트박스·Reveal 등장)을 재사용해 디자인 톤 통일
- 콘텐츠는 전부 상수 주입 유지, 정적 생성(SSG) 유지

### ✅ 구현 내용

#### 부서 상세 데이터 모델 확장
- **파일**: `src/constants/departments.ts`, `src/constants/departments.test.ts`
- **변경 내용**: `Department`에 `intro`·`features`·`info`·`activities`·`programs`·`gallery`·`invite` 옵션 필드 추가. 각각의 타입(`DeptFeature`·`DeptInfoItem`·`DeptProgram`·`DeptPhoto`)과 섹션 헤딩 상수 `DEPT_SECTIONS`(알림 사항·주요 활동·특별 프로그램·활동 사진) 신규 정의. `student` 항목만 전체 필드를 채우고 나머지 4개 부서는 기존 필드(`slug`·`name`·`description`·`media`·`caption`)만 유지.
- **이유**: 옵션 필드로 설계해 미기입 부서는 타입·렌더 모두 기존과 동일하게 남도록 하기 위함. 신규 데이터 검증 테스트 2건 추가.

#### 부서 상세 보강 섹션 컴포넌트 6종 신규
- **파일**: `src/components/departments/DeptFeatures.tsx`, `DeptInfo.tsx`, `DeptActivities.tsx`, `DeptPrograms.tsx`, `DeptGallery.tsx`, `DeptInvite.tsx` (+ 각 `.test.tsx`)
- **변경 내용**: 소개(3-up 아이콘 카드) · 알림 사항(2-up 라벨/값 카드) · 주요 활동(체크 목록) · 특별 프로그램(2-up 카드) · 활동 사진(그리드 + 확대 모달) · 초대(닫는 멘트 밴드)를 각각 독립 컴포넌트로 구현. 컴포넌트 테스트 9건 신규 작성.
- **이유**: 섹션별로 재사용 가능한 단위로 분리해 다른 부서가 필드를 채웠을 때도 그대로 재사용 가능하도록 함.

#### 학생부 활동 사진 placeholder 추가
- **파일**: `public/dept/student/1.jpg` ~ `4.jpg`
- **변경 내용**: 활동 사진 섹션이 참조하는 placeholder 이미지 4장 추가.
- **이유**: 실제 사진 확보 전까지 갤러리 섹션이 깨지지 않고 동작하도록 함(배포 전 실사진 교체 전제).

#### 상세 페이지 조립 + 테스트 갱신
- **파일**: `src/app/departments/[slug]/page.tsx`, `src/app/departments/[slug]/page.test.tsx`
- **변경 내용**: 6개 섹션을 삼항 조건부(`?  : null`)로 조립. `dept.intro && dept.features?.length`, `dept.info?.length` 등 데이터 존재 여부로 각 섹션 노출을 결정해 미기입 부서는 렌더 결과가 이전과 동일. 기존 테스트에 보강 섹션 노출 검증 어서션 추가.
- **이유**: 부서별 데이터 유무에 따라 자동으로 섹션이 나타나거나 생략되도록 해, 향후 다른 부서도 상수만 채우면 동일하게 동작.

### 🔧 주요 변경사항 상세

#### 기존 컴포넌트 패턴 재사용
- `DeptFeatures`의 아이콘 매핑(`ICONS: Record<string, LucideIcon>`)은 메인 `MinistryCards`와 동일한 방식 — 상수는 직렬화 가능한 문자열 키(`"book"`·`"users"`·`"sparkles"`)만 갖고, 컴포넌트에서 lucide 아이콘으로 변환.
- `DeptInfo`·`DeptPrograms`는 `Card` 컴포넌트 위에 흰 카드를 얹는 `schedule-card` 계열 패턴, 밴드는 `bg-surface-soft`로 회색 교차.
- `DeptGallery`는 `about/ChurchPhotos`의 그리드+확대 모달(`Dialog`) 라이트박스 구조를 그대로 따르되 카테고리 탭은 제거하고 좌우 이동(``ChevronLeft``/``ChevronRight``, 키보드 방향키)만 남김.
- 전 섹션이 `Reveal`로 뷰포트 진입 시 1회 등장(스태거 120ms)해 기존 히스토리/사역 밴드와 등장 리듬을 맞춤.

#### 밴드 리듬
소개(흰) → 알림 사항(회색) → 주요 활동(흰) → 특별 프로그램(회색) → 활동 사진(흰) → 초대(`bg-primary-soft`)로 흰↔회색 교차 뒤 마지막에 프라이머리 틴트 밴드로 마무리. 다크 밴드는 페이지 최하단 `CtaBand`가 담당해 중복하지 않음.

#### 재사용성(회귀 없음)
청년부·예배부·남선교회·여선교회는 `intro`·`features`·`info`·`activities`·`programs`·`gallery`·`invite`가 전부 `undefined`라 `page.tsx`의 조건부가 전부 `null`을 반환 — 소개 문단까지의 기존 화면이 그대로 유지된다. 필드를 채우면 별도 코드 수정 없이 동일한 섹션이 노출되는 구조.

**특이사항**:
- 신규 6개 섹션 컴포넌트는 `.claude/rules/DESIGN.md`의 `## 컴포넌트` 블록에는 아직 등록되지 않은 상태. 문서 규칙(신규 컴포넌트는 블록에 항목 추가 후 구현)과 어긋나는 부분이라 후속으로 문서 갱신이 필요.
