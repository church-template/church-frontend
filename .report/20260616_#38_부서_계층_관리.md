# 부서 계층 관리 (어드민 04) — #38

## 1. 개요

`/mypage/manage/departments` 경로에서 운영자가 부서 계층(루트·하위)을 생성·수정·삭제할 수 있는 어드민 화면이다. 구현 방향이 두 차례 전환되었다.

**진화 경로**

| 단계 | 방식 | 폐기 이유 |
|---|---|---|
| 1차 | 평면 테이블 | 계층 관계가 보이지 않아 스캔 어려움 |
| 2차 | 마스터–디테일(2단) | 루트 클릭 후에야 하위가 표시 — 선택 간접성 높음. 행 `justify-between`으로 이름·액션 사이가 텅 비어 레이아웃 갭 발생 |
| **최종** | **접이식 단일 트리** | 전체 계층을 한 화면에 노출, 자식 있는 노드만 chevron으로 접기/펼치기, 캡 컬럼(`max-w-[var(--container-modal)]`)으로 갭 제거 |

공개 '교육·부서 소개' 페이지는 프론트 상수(`DEPARTMENTS`) 구동으로 별개이며, 이 화면에서의 변경이 공개 소개 페이지에 자동 반영되지 않는다. 화면 상단에 격리 안내 배너를 고정 노출한다.

## 2. 주요 변경

### 2-1. 데이터·유틸·스키마 (재사용, 무변경)

**`src/lib/api/departments.admin.ts`**
- 읽기: 어드민 전용 GET 없이 공개 엔드포인트를 `cache: "no-store"`로 호출(read-your-writes).
- 쓰기: `POST /api/admin/departments`(생성), `PUT /api/admin/departments/{id}`(수정, 낙관락 `version`), `DELETE /api/admin/departments/{id}`(삭제).
- `apiMutate` 래퍼로 200/201/204 자동 처리, 비2xx는 `ApiError` 상향.
- 클라이언트 컴포넌트 전용 경계 — 서버 번들 포함 시 빌드 오류가 나므로 파일 상단에 명시.

**`src/components/admin/departments/schema.ts`**
- zod 스키마: `name`(1–100자 필수), `description`(≤50000), `leader`(≤100), `parentId`(number|null), `sortOrder`(nonnegative int|null).
- `optional().default()` 미사용 — zodResolver 입출력 타입 불일치 회피. 기본값은 `useForm defaultValues`에서 주입.

**`src/components/admin/departments/treeUtils.ts`**

기존 3개 함수는 유지하고 접이식 트리용 2개를 추가했다.

| 함수 | 역할 |
|---|---|
| `flattenTree` | preorder DFS로 `DepartmentNode[]`를 `{ node, depth }[]`로 평탄화 |
| `collectDescendantIds` | 대상 노드의 모든 하위 id를 `Set<number>`로 반환(자기 제외). 상위 select 순환 방지 |
| `findNode` | id로 트리 내 임의 깊이의 노드를 탐색 |
| **`flattenVisible`** (신규) | preorder DFS; `collapsed`에 포함된 노드의 자식은 건너뛰어 가시 행(`VisibleRow`)만 반환. `hasChildren` 플래그로 chevron/스페이서 분기 |
| **`collectCollapsibleIds`** (신규) | 자식 있는 모든 노드 id 집합 — "전체 접기" 시 collapsed에 투입 |

### 2-2. 접이식 트리 UI

**`DepartmentTree.tsx`** (신규 — 마스터·디테일의 ClusterList·ClusterDetail 대체)

- 표현 전담 컴포넌트. `flattenVisible(roots, collapsed)`로 가시 행만 계산해 `<ul>` 렌더.
- 자식 있는 노드: `ChevronRight`(접힘)·`ChevronDown`(펼침) 버튼. `aria-expanded`·`aria-label` 제공.
- 잎 노드: chevron 자리에 동일 폭(`w-6`) 빈 스페이서 → 이름 시작점 정렬(들쭉날쭉 방지).
- depth별 들여쓰기: spacing 토큰 배열 `["pl-0", "pl-lg", "pl-xxl"]`, 3단 이상 캡.
- `NodeActions`: 각 노드에 `＋하위`·`수정`·`삭제` 버튼. 아이콘은 항상 노출, 텍스트 라벨은 `hidden lg:inline`(모바일 아이콘-only). `aria-label`로 접근성 보완.
- 빈 상태: "등록된 부서가 없습니다." 안내.

**`DepartmentManager.tsx`** (재작성)

- 오케스트레이터. 마스터–디테일의 선택 상태(`selectedId`, `drilledIn`)가 모두 제거되어 단순화됨.
- `collapsed: Set<number>` — 기본 `new Set()`(전체 펼침). 불변 갱신.
- **전체 펼치기**: `setCollapsed(new Set())`.
- **전체 접기**: `setCollapsed(collectCollapsibleIds(roots))`.
- 캡 컬럼: 툴바+트리를 `max-w-[var(--container-modal)]` div로 묶어 가운데 빈 갭 제거(R1). Tailwind t-shirt 폭 토큰 충돌 회피를 위해 CSS 변수 arbitrary value 사용.
- `effect` 미사용 — 접힘 토글과 다이얼로그 열림은 이벤트 핸들러에서 직접 처리.
- 공개 격리 안내 배너: `bg-surface-soft` + `Info` 아이콘 + 안내 문구 고정 노출.
- 다이얼로그 배선: create(`createParentId`) 와 edit(`editId`) 각각 `DepartmentFormDialog` 인스턴스로 분리.

**`DepartmentFormDialog.tsx`** (무변경, 재사용)

- 생성/수정 공용. 열릴 때 `useQuery`로 상세를 `cache: "no-store"` 조회 → `version`·값 시드.
- `seeding(isFetching)` 동안 저장 버튼 비활성화 → stale version 409 방지.
- 시드 조회 실패 시 즉시 에러 토스트(제출 버튼이 말없이 멈추는 것 방지). `retry: false` — 재시도하면 ~30초간 침묵 후 실패.
- 409 충돌 → `detail.refetch()`로 최신본 갱신 후 seeding 게이트가 재차 차단.
- 항상 PUT: 루트화(`parentId null`)·필드 교체를 단일 경로로 처리.
- **순환 방지**: 수정 시 자기 자신 + 모든 하위(`collectDescendantIds`)를 상위 select 옵션에서 제외.
- **`defaultParentId`**: 노드의 `＋하위` 클릭 시 해당 노드 id를 create 다이얼로그에 프리셋.

**`src/app/(site)/mypage/manage/departments/page.tsx`** (라우트)

- `RequirePermission permission="DEPT_WRITE"` 게이트: 제목·컨테이너는 항상 렌더, 권한 없으면 `EditAccessDenied` 폴백.
- `useSearchParams` 미사용이라 Suspense 경계 불필요.

## 3. 요구사항 충족

| 요구사항 | 처리 방식 |
|---|---|
| 부서 계층 시각화 | `flattenVisible` + depth별 들여쓰기로 단일 트리 표시 |
| 노드 CRUD | 툴바 `＋새 부서`(루트), 노드 `＋하위`·`수정`·`삭제` 버튼 |
| 상위 select 순환 방지 | 수정 시 자신 + `collectDescendantIds` 결과 옵션 제외 |
| 하위 있으면 삭제 차단 | 서버가 409 `DEPARTMENT_HAS_CHILDREN` 반환 → `adminOnError` 토스트. 삭제 경고문 UI에도 명시 |
| 루트화 | 수정 다이얼로그에서 상위를 "(없음 — 최상위 부서)"로 변경 후 PUT(parentId null) |
| 동시 수정 충돌 | 낙관락 `version` 필드 — 열 때마다 상세 fresh 조회로 최신 version 확보, 409 시 refetch |
| 공개 부서 페이지 격리 | 화면 상단 안내 배너 + onSuccess 후 ISR revalidate 없음(어드민 캐시만 무효화) |
| 반응형 | 단일 컬럼 풀폭(드릴인·2단 없음). 모바일은 액션 아이콘-only(`hidden lg:inline`) |

## 5. 비고

캡 컬럼 레이아웃(`max-w-[var(--container-modal)]`)과 모바일 텍스트 라벨 토글(`hidden lg:inline`)은 jsdom 환경에서 CSS 미적용 특성상 자동 검증이 불가하다. 실제 동작 확인은 dev 서버 기동 후 DEPT_WRITE 권한 계정 로그인 상태에서 수동으로 진행해야 한다. 확인할 항목: 캡 컬럼이 512px로 제한되어 액션이 이름 바로 옆에 붙는지, 모바일 뷰포트에서 아이콘만 표시되는지.
