# 목회자 인사말 페이지 UI/UX 재디자인 (#67)

### 📌 작업 개요
목회자 인사말 페이지(`/about/pastor`)를 단일 `Container` 안의 단순 텍스트 나열에서, DESIGN 밴드 리듬(흰 캔버스 → 다크 인용 → 회색 약력/철학)을 따르는 4개 컴포넌트 조합으로 재디자인. 페이지는 상수 구동 서버 컴포넌트 구조를 유지하고, 표현 로직을 컴포넌트로 분리.

### 🎯 구현 목표
- 텍스트 일변도 레이아웃을 시각적 위계가 있는 밴드 교차 레이아웃으로 개선
- 초상 사진 영역(자산 미준비 대응 폴백 포함) 도입
- 핵심 인용 1문장을 다크 밴드 elevated 카드로 강조
- 약력은 헤어라인 행, 목회 철학은 아이콘 3-up 그리드로 재구성
- 콘텐츠는 전부 상수(`PASTOR`)에서 주입, 컴포넌트 하드코딩 금지 유지

### ✅ 구현 내용

#### 페이지 진입점 재구성
- **파일**: `src/app/(site)/about/pastor/page.tsx`
- **변경 내용**: 인라인 마크업 전부 제거 후 `PastorIntro` → `PastorQuote` → `PastorDossier` 3개 섹션 조합으로 교체. `metadata.title` 추가.
- **이유**: 표현을 컴포넌트로 분리해 밴드별 배경·리듬을 독립 관리. 페이지는 조립만 담당.

#### PastorIntro — 밴드1 (흰 캔버스 비대칭 스플릿)
- **파일**: `src/components/about/PastorIntro.tsx`
- **변경 내용**: `lg:grid-cols-[5fr_7fr]` 비대칭 그리드로 좌측 초상(`PastorPortrait`) / 우측 인사말 본문 배치. 직분 캡션 + 이름(displayMd) + 직위(titleMd muted) + 학위(datetime) + intro/greeting 본문. `Reveal`로 등장 애니메이션.
- **이유**: 초상과 인사말을 나란히 배치해 인물 페이지의 시각 위계 확보.

#### PastorPortrait — 3:4 초상 액자 + 폴백
- **파일**: `src/components/about/PastorPortrait.tsx`, `src/components/about/PastorPortrait.module.css`
- **변경 내용**: `image` prop이 있으면 `<img>`, `null`이면 `surface-soft` 배경 + lucide `UserRound` 아이콘 플레이스홀더 렌더. 두 분기 모두 동일 3:4 프레임으로 박스 예약.
- **이유**: 포트레이트 자산 미준비 상태에서도 레이아웃이 무너지지 않고, 자산 도착 시 CLS 0 보장. 가짜 사진을 끼우지 않는 정책.

#### PastorQuote — 밴드2 (다크 밴드 elevated 인용 카드)
- **파일**: `src/components/about/PastorQuote.tsx`
- **변경 내용**: `surface-dark` 밴드 위에 `surface-dark-elevated` 카드(`figure`/`blockquote`/`figcaption`). lucide `Quote` 아이콘 + displayLg 인용문 + `Badge(variant="primary")`로 발화자 표기.
- **이유**: 핵심 인용 1문장을 다크 밴드 깊이로 강조(그림자 단계 추가 없이 elevated 패턴). primary 액센트는 이 페이지에서 Badge 1회만 등장(희소성 원칙).

#### PastorDossier — 밴드3 (회색 약력 행 + 철학 아이콘 그리드)
- **파일**: `src/components/about/PastorDossier.tsx`
- **변경 내용**: `surface-soft` 밴드. 약력은 1px 헤어라인 구분 세로 행 목록, 목회 철학은 `sm:2-up`/`lg:3-up` 그리드 카드(각 카드 lucide 아이콘 32px + 텍스트). 철학 키→아이콘 매핑을 컴포넌트 내 `PHILOSOPHY_ICONS`로 관리.
- **이유**: 약력은 읽기 행, 철학은 카드로 위계 분리. 아이콘 색은 `currentColor`(ink 상속)로 단일 액센트 유지(MinistryCards 선례).

#### 콘텐츠 상수 확장
- **파일**: `src/constants/content.ts`
- **변경 내용**: `PASTOR` 객체에 `image`(`{ src, alt } | null`, 기본 null), `pullQuote`(다크 밴드 인용 1문장) 필드 추가. `philosophy.items`를 문자열 배열에서 `{ key, text }` 객체 배열로 변경하고 `PastorPhilosophyKey` 유니온 타입 신설(`worship`·`bible`·`fellowship`·`community`·`nextgen`·`mission`). `satisfies`로 키-타입 정합 보장.
- **이유**: 아이콘 매핑용 직렬화 키를 상수에 부여하되 색/아이콘 컴포넌트는 매핑 책임을 컴포넌트가 가지게 분리. 인용문은 컴포넌트 발췌가 아닌 상수 단일 진실로 관리(하드코딩 방지).

### 🔧 주요 변경사항 상세

**밴드 리듬**: 페이지 주석대로 흰(Intro) → 다크(Quote) → 회색(Dossier) → (전역 레이아웃의 다크 CTA 밴드)로 이어지는 교차 리듬을 구성. 각 섹션이 자기 배경 토큰(`bg-canvas`/`bg-surface-dark`/`bg-surface-soft`)을 직접 소유.

**토큰 준수**: 색·간격·라운드 전부 토큰(`text-ink`·`py-section`·`rounded-xl`·`gap-xxl` 등) 사용. 3:4 커스텀 비율만 arbitrary value 금지 규칙에 따라 CSS 모듈(`aspect-ratio: 3 / 4`)로 처리.

**특이사항**:
- `PastorPortrait`의 `<img>`에 `@next/next/no-img-element` eslint 비활성 주석 — 코드베이스 콘텐츠 이미지 관례(next/image 전환은 후속).
- 철학 카드 `Reveal`은 `delay={i * 120}` 스태거 + `h-full`로 카드 높이 통일.
