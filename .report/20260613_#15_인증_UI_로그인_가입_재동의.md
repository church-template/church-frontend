# 인증 UI 구현 보고서 (#15)

### 📌 작업 개요

회원 인증 3개 화면(로그인·회원가입·약관 재동의)과 마이페이지 진입점을 구현했다. 풀스크린 스플릿 레이아웃, 회원가입 5단계 위저드, 전화번호 자동 하이픈·비밀번호 표시 토글 등 고령 사용자 가독성을 우선한 폼 UX가 핵심이다. 데이터 검증은 zod, 폼 상태는 react-hook-form으로 처리한다.

### 🎯 구현 목표

- 로그인·회원가입·약관 재동의 페이지를 `(auth)` 라우트 그룹의 공용 스플릿 셸로 구성
- 오픈 리다이렉트를 차단하는 `?next=` 복귀 동선 (3페이지 공용)
- 회원가입을 한 화면 = 한 질문의 위저드로 분해, 서버 필드 에러 시 해당 스텝 복귀
- 가입 후 자동 로그인 → `requiresAgreement` 분기로 재동의 체이닝
- 마이페이지 placeholder + 로그아웃 동선 연결

### 🔧 주요 변경사항 상세

#### 마운트 1회 역가드 패턴 (공통)
로그인·가입·마이페이지 모두 `useEffect(() => {...}, [])`로 마운트 시 1회만 세션을 판정한다. `member`를 구독하면 제출 후 자동 리다이렉트와 경합해 `next` 복귀 경로를 잃기 때문이다(스펙 4.4).

#### 전화번호 처리
입력칸에는 하이픈을 자동 부착해 보여주되(`formatPhone`), 전송은 입력값 그대로 보낸다. 백엔드가 하이픈을 떼고 정규화하므로 무방하다(가이드 11장). zod는 숫자만 세어 9~11자리를 검증한다.

**특이사항**:
- 회원가입 약관 동의는 필수 2종(이용약관·개인정보처리방침) 모두 `true` 강제
- 이메일은 선택 — 빈 문자열 허용, 제출 시 `undefined`로 변환. "건너뛰기"로 잘못 입력하다 만 값과 형식 에러를 함께 정리
