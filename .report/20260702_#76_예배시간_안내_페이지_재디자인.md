# 예배 시간 안내 페이지(`/worship`) UI/UX 재디자인 #76

### 📌 작업 개요
제목 + 예배 카드 4개만 있던 최소 `/worship` 페이지를, 정기 예배 · 특별 예배 · 장소·참석 안내 3밴드 구조로 재디자인.
최근 재디자인 페이지(오시는 길 #74)의 흰↔회색↔다크 밴드 리듬·`Reveal` 등장·디자인 토큰 규칙에 맞춰 톤 통일.
지도는 중복 없이 기존 `/about/location`에 링크로 위임.

### 🎯 구현 목표
- 정기 예배: 이름 + 대표 시간 + 찬양 시간 + 설명 3줄을 담은 2×2 카드
- 특별 예배: 연중 6종을 날짜 배지 카드(기존 `EventCard` 재사용) 3-up
- 장소·참석 안내: 다크 밴드 좌우 분할(주소·문의 / 참석 안내) + "오시는 길 자세히" 링크
- 콘텐츠는 전부 상수 주입, 정적 생성 유지(API 호출 0), 메인 페이지 비파괴

### ✅ 구현 내용

#### 콘텐츠 상수 확장 (비파괴)
- **파일**: `src/constants/content.ts`
- **변경 내용**: `WORSHIP` 객체에 섹션 카피 필드 9종 추가(`regularLead`, `special*`, `place*`, `attend*`).
  `WorshipService`에 `praise?`(찬양 시간)·`notes`(설명 3줄) 추가하고 기존 `place` 유지. `SpecialService` 인터페이스 +
  `SPECIAL_SERVICES`(6종) 신규.
- **이유**: 메인 페이지(`WorshipSection`→`ScheduleCard`)가 `WORSHIP.title`·`WORSHIP_SERVICES[].place`를 소비하므로
  기존 필드를 보존하고 추가만 해서 회귀를 막음.

#### 정기 예배 섹션 (흰 캔버스)
- **파일**: `src/components/worship/WorshipRegular.tsx` (신규)
- **변경 내용**: 페이지 h1(`예배 시간 안내`) + 리드 아래 2×2 카드 그리드(`Card surface="soft"`). 카드마다 이름·대표시간·
  찬양시간(주일·수요만, 삼항)·설명 3줄. `Reveal` 스태거 등장.
- **이유**: 4개 예배를 균형 있게 배치하고 예배별 안내를 담기 위해 3-up 대신 2×2 채택.

#### 특별 예배 섹션 (회색 밴드)
- **파일**: `src/components/worship/WorshipSpecial.tsx` (신규)
- **변경 내용**: `bg-surface-soft` 밴드에 h2 + 리드 + `SPECIAL_SERVICES`를 **기존 `EventCard`로 재사용**해 3-up(6종 → 3×2).
  `date`→날짜 배지, `title`→예배명, `time`→시간, `summary`→설명. 비인터랙티브(href 없음).
- **이유**: 날짜 배지 + 제목 + 요약이라는 `EventCard` 구조가 특별 예배와 정확히 일치 — 신규 카드 컴포넌트를 만들지 않음.

#### 장소·참석 안내 섹션 (다크 밴드)
- **파일**: `src/components/worship/WorshipPlace.tsx` (신규)
- **변경 내용**: `bg-surface-dark` 밴드 좌우 2단. 좌 = 장소 안내(주소 + "유리 건물" + 문의 전화 `tel:` 링크) +
  `오시는 길 자세히` 버튼(`/about/location`). 우 = 참석 안내(안내 3줄). 모든 텍스트 on-dark 토큰.
- **이유**: 지도 임베드를 중복하지 않고 풀 지도 페이지(#74)에 위임. 주소·전화는 `church.ts` 상수 재사용.

#### 페이지 조립 + 테스트 갱신
- **파일**: `src/app/(site)/worship/page.tsx`, `src/app/(site)/worship/page.test.tsx`
- **변경 내용**: 페이지를 3밴드 컴포넌트 조립만 하도록 교체(정적 생성 유지). 이전 구조(예배명 h2 4개)를 검증하던
  기존 테스트를 새 구조(h1 + 특별/장소 h2) 스모크 테스트로 교체.
- **이유**: 구조가 바뀌어 기존 단언이 깨지므로 함께 갱신.

### 🔧 주요 변경사항 상세

#### EventCard 재사용
특별 예배는 별도 카드를 만들지 않고 `@/components/cards/EventCard`를 `href` 없이 사용. `EventCard`의 비인터랙티브
분기가 흰 카드(`Card bordered`)로 렌더돼 회색 밴드 위에서 자연스럽게 뜬다.

#### 다크 밴드 토큰 규율
장소 안내는 다크 배경이라 모든 텍스트에 `text-on-dark`·`text-on-dark-soft`만 사용(캔버스 토큰 누수 0). "오시는 길
자세히" 버튼은 `buttonVariants("outlineOnDark")` + `h-14 px-8`으로 기존 프리푸터 CTA 밴드(`CtaBand`)와 동일 크기.

**특이사항**:
- 아이콘(lucide)은 다크 밴드 위 정렬에 스케일 밖 수치가 필요해 토큰 순수성을 위해 생략(디자인 문서상 아이콘은 선택 사항).
- 정기 예배 섹션은 페이지 h1이 곧 섹션 제목을 겸함(별도 "정기 예배" h2 없이 h1+리드) — 설계 단계에서 확정한 구조.
