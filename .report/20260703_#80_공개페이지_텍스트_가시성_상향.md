# 공개 페이지 텍스트 가시성 상향 (#80)

### 📌 작업 개요
`/about`(#69 빅타이포 에디토리얼)에서 검증된 고령 교인 가독성 방식을 나머지 공개 콘텐츠 페이지 전반에 확대 적용. 읽는·보조 텍스트의 `typo.*` 토큰을 요소 역할에 맞춰 **한 단계 상향**해 사이트 전체의 텍스트 가시성을 통일했다. 토큰 기준값(`--text-*`)은 그대로 두고, 컴포넌트가 참조하는 의미 토큰만 교체하는 저위험 변경.

같은 이슈에서 **부서 상세 페이지 섹션 순서를 방문자 서사 퍼널에 맞춰 재배치**(소개 → 활동사진 → 활동·프로그램 → 알림 → 초대)하고, 밴드 교차 리듬을 유지하도록 두 섹션의 밴드 색을 스왑했다.

### 🎯 구현 목표
- 공개 콘텐츠 페이지의 읽는 본문(리드 문단)·보조 텍스트(카드 설명·목록행 부제·안내 3줄·푸터)를 `/about`과 정합하는 스케일로 확대
- DESIGN.md 규칙(단일 액센트·`typo.*` 상수·중첩 라디우스·hex/px 인라인 금지) 전부 유지
- UI 프리미티브·밀도 제약 영역(달력·라이트박스)·날짜시간 메타(`datetime` tnum)·`/about`·어드민은 미변경

### 🔧 상향 규칙 (한 단계 리프트)
설계 스펙(`docs/superpowers/specs/2026-07-03-text-legibility-lift-design.md`) §3 기준.

| 역할 | 변경 전 | 변경 후 | 근거 |
|---|---|---|---|
| 상세/도입 리드 문단(읽는 본문) | `bodyMd` 20 | `bodyLg` 24 | `/about` lead와 동일 스케일 |
| 콘텐츠 보조 텍스트(카드 설명·목록행 부제·안내 3줄·푸터) | `bodySm` 18 | `bodyMd` 20 | 읽는 정보인데 최소치에 몰려 있었음 |
| 카드 subtitle 대비(series·scripture 등) | `caption` 16 muted | `bodySm` + 대비 상향(muted→body) | 설명을 muted 회색으로 흐리지 않는다 |
| 목록행/카드 주제목 · 날짜시간 메타 · 페이지 헤딩 | — | 유지 | 이미 적정/큼 |

원칙: 블라인드 grep 치환이 아니라 **요소 역할을 보고 판단**(그래서 페이지별 방식). 콘텐츠(읽는) 텍스트만 올리고 UI 크롬·밀도 제약·메타는 건드리지 않음.

- **변경**: 부서 상세 섹션 순서를 기존 `소개 → 알림 → 주요활동 → 특별프로그램 → 활동사진 → 초대`에서 `소개 → 활동사진 → 주요활동 → 특별프로그램 → 알림 → 초대`로 재배치. 위치가 바뀐 두 섹션의 밴드 색을 교대 위치에 맞게 스왑 — DeptGallery는 흰 → `bg-surface-soft` 회색 밴드, DeptInfo는 회색 → 흰 밴드 + `Card bordered`.
- **이유**: 방문자가 페이지를 훑는 서사에 맞춰 끌기(소개) → 증명(활동사진, 감정 후크 조기 노출) → 설명(활동·프로그램) → 실용정보(알림: 모임시간·장소) → 전환(초대) 순으로 정보를 흘린다. 알림을 초대 직전에 붙여 "언제/어디로 → 함께해요" 동선을 자연스럽게 연결.

### 🔧 주요 변경사항 상세

#### 마크다운 상세 본문 (`.prose-church`)
- **파일**: `src/app/globals.css`
- 상세 본문(설교·공지·일정·앨범·부서 description)은 `typo.*`가 아니라 `MarkdownContent`의 `.prose-church` CSS로 렌더된다(`dangerouslySetInnerHTML`). 근본 위치인 `.prose-church`의 `font-size`를 `--text-body-md`(20) → `--text-body-lg`(24)로, line-height도 body-lg로 상향.
- 토큰만 참조(px 인라인 아님). 모든 마크다운 상세 본문이 한 번에 `/about` 읽는 본문 스케일로 통일. h1~h3는 명시 크기라 영향 없음.

**특이사항**:
- 토큰 기준값(`--text-*`)은 변경하지 않았다. `/about`이 더 커지지 않고, 입력·뱃지·네비 등 UI 크롬이 부풀지 않도록 페이지별 토큰 스왑 방식을 채택.
- SermonCard subtitle은 크기(caption→bodySm)와 대비(muted→body)를 함께 올렸다 — "설명을 muted 회색으로 흐리지 않는다"는 `/about` 대비 규칙 반영.
- 날짜·시간 메타는 전부 `datetime`(tnum) 유지 — 예배 시간표·목록행에서 자릿수 흔들림 방지.

#### 부서 상세 서사 순서 · 밴드 리듬
- **파일**: `src/app/departments/[slug]/page.tsx`
- 페이지 조립부에서 조건부 렌더 블록의 순서만 재배열해 서사 퍼널(소개 → 활동사진 → 활동·프로그램 → 알림 → 초대)을 구성. 순서를 바꾸면 흰↔회색 교차가 깨지므로 위치가 바뀐 두 컴포넌트의 밴드 색을 스왑:
  - **DeptGallery**: 소개 직후로 올라오며 흰 → `bg-surface-soft` 회색 밴드. `Container`는 max-width라 풀블리드가 안 되어 `<section className="bg-surface-soft py-section">`으로 감싸고 안에 `Container`를 둠.
  - **DeptInfo**: 특별프로그램(회색) 다음·초대(primary-soft) 직전으로 내려오며 회색 → 흰 밴드 + `Card bordered`(schedule-card 변형)로 흰 캔버스 위 카드 대비 확보.
- 결과 밴드 시퀀스 `흰(소개) / 회색(활동사진) / 흰(주요활동) / 회색(특별프로그램) / 흰(알림) / soft(초대)` — 재배치 후에도 흰↔회색↔soft 교차 리듬 유지.

**특이사항**:
- 조립부는 모든 부서가 공유 → 순서 재배치는 섹션이 채워진 부서(학생부 등) 전체에 일관 적용. 데이터가 없는 섹션은 기존대로 조건부 렌더로 자연 생략되어 일부 섹션만 있는 부서도 문제없음.
- 카드 내부 톤 교차(DeptFeatures의 primary-soft/surface-soft/surface-dark)는 밴드와 별개라 미변경.

### 🧪 테스트 및 검증
- 테스트 코드 미작성(사용자 지시 — 텍스트 크기 조정이므로). 기존 테스트는 `text-*` 클래스를 검증하지 않아 토큰 스왑으로 깨지지 않음.
- 정적 게이트: `pnpm lint` + `npx tsc --noEmit` + `pnpm build` 통과.
- 육안 검증: 데스크톱·모바일 폭에서 텍스트 확대·레이아웃(카드 그리드·정렬·줄바꿈) 무결 확인.
