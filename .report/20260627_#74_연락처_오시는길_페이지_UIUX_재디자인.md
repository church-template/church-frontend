# 연락처·오시는 길(/about/location) 페이지 UI/UX 재디자인

**이슈**: #74

---

### 📌 작업 개요

`/about/location` 페이지를 두 섹션 구조로 재설계. 흰 캔버스 섹션(연락 정보 + 약도)과 회색 밴드(찾아오는 방법 아이콘 카드)로 나누어 DESIGN.md 섹션 리듬을 준수. 콘텐츠 전체를 상수(`LOCATION`, `church.ts`) 주입으로 처리해 API 호출 없는 정적 생성(○ Static). 카카오·네이버 외부 지도 버튼의 공식 브랜드색을 단일 액센트 원칙의 의도적 예외로 토큰화하고 DESIGN.md에 명문화.

---

### 🎯 구현 목표

- 연락처(주소·전화·이메일) + 약도 2단 레이아웃 구성
- 지도 임베드(`MAP_EMBED_SRC`) 유무에 따른 폴백 분기 보존
- 교통 수단별 찾아오는 방법을 아이콘 카드로 시각화
- 카카오맵·네이버지도 외부 연결 버튼 브랜드색 처리
- 토큰·`typo.*` 준수, Reveal 등장, 반응형 그리드

---

### ✅ 구현 내용

#### 페이지 조립 — page.tsx
- **파일**: `src/app/(site)/about/location/page.tsx`
- **변경 내용**: `<LocationContact />`와 `<LocationDirections />` 두 컴포넌트만 합성. `export default` 함수, API 호출 없음 → 빌드 타임 정적 생성(○ Static) 확정.
- **이유**: 페이지 단은 조립만 담당, 렌더 로직은 각 컴포넌트에 격리.

#### 흰 섹션 — LocationContact.tsx (신규)
- **파일**: `src/components/about/LocationContact.tsx`
- **변경 내용**: 비대칭 5/7 그리드(`lg:grid-cols-[5fr_7fr]`)로 좌측 연락 정보, 우측 약도 배치.
  - 좌측 `<dl>`: 주소·전화·이메일을 헤어라인 구분행으로 나열. 전화는 `tel:` 링크(탭 발신), 이메일은 `mailto:` 링크.
  - 우측: `MAP_EMBED_SRC`가 있으면 `<iframe loading="lazy" className="aspect-video ...">` 렌더, 없으면 `<img>` 약도 + 외부 지도 버튼 2개(카카오·네이버) 2열 그리드로 폴백.
- **이유**: 지도 임베드 URL이 미설정인 초기 배포에서도 약도 이미지와 외부 링크로 동등한 경험 제공.

#### 회색 밴드 — LocationDirections.tsx (신규)
- **파일**: `src/components/about/LocationDirections.tsx`
- **변경 내용**: `bg-surface-soft` 배경 `<section>` 안에 `LOCATION.directions` 배열을 아이콘 카드(`rounded-xl border border-hairline bg-canvas p-xl`) 그리드로 렌더. 아이콘은 `DIRECTION_ICONS` 레코드(`car→Car`, `transit→Bus`) 매핑, 미정의 key는 `Car` 폴백.
  - 카드 높이 동일화: `<li>`에 `h-full`이 없어도 `<Reveal className="h-full">`과 내부 div `h-full flex-col`로 grid stretch가 작동.
  - 스태거드 Reveal: 카드 인덱스 × 120ms delay.
- **이유**: 교통 수단 종류가 늘어나도 상수 배열만 추가하면 자동 렌더.

#### 상수 재구성 — content.ts
- **파일**: `src/constants/content.ts`
- **변경 내용**: `LOCATION` 객체를 `title`, `lead`, `map`(`{ src, alt }`), `directionsHeading`, `directions[]`(`key/title/lines[]`) 구조로 정비.
  - `map.src`는 `"/location/map-placeholder.svg"` — 교회가 약도 이미지를 준비하면 경로만 교체(단일 교체점).
  - `directions`는 `"car"`·`"transit"` 두 항목 초기값. 대중교통 실제 노선은 교회 입력 대기.
- **이유**: 모든 사용자 노출 문구를 상수에서 주입해 하드코딩 금지 원칙 준수.

#### 상수 추가 — church.ts
- **파일**: `src/constants/church.ts`
- **변경 내용**: `naverMapSearchUrl(address: string)` 함수 추가(`https://map.naver.com/p/search/{인코딩주소}`). 기존 `mapSearchUrl`(카카오)과 쌍으로 폴백 버튼 생성에 사용.
- **이유**: 외부 지도 URL 조합 로직을 컴포넌트에 인라인하지 않고 단일 진실로 노출.

#### 약도 플레이스홀더 — map-placeholder.svg
- **파일**: `public/location/map-placeholder.svg`
- **변경 내용**: 회색 배경 + 라운드 점선 경계 + "약도 자리표시자" 안내 텍스트 SVG. `aria-label="약도 자리표시자"` 포함. 교회가 실제 약도 이미지를 준비하면 같은 경로에 덮어쓰거나 `LOCATION.map.src` 값만 변경.

### 🔧 상세 변경사항

#### 카카오·네이버 브랜드 버튼 토큰화

**단일 액센트 원칙의 의도적 예외**로 처리. 제3자 서비스의 공식 채움색을 사용하는 버튼이라 교회 팔레트 토큰과 구분.

**`src/app/globals.css`** — `@theme` 블록에 4개 토큰 추가:
```css
--color-kakao: #fee500;
--color-kakao-ink: #000000;   /* 카카오 노랑 위 라벨(공식 검정) */
--color-naver: #03c75a;
--color-naver-on: #ffffff;    /* 네이버 초록 위 라벨(흰색) */
```
파일 내 주석으로 "button-kakao/naver 외엔 쓰지 않는다"를 명시해 범용 사용 차단.

**`src/components/ui/Button.tsx`** — `ButtonVariant` 타입에 `"kakao"·"naver"` 추가. `variantClass` 레코드에 variant 정의:
- `kakao`: `bg-kakao text-kakao-ink`, hover·press는 `brightness-95/90`(색상 토큰 추가 없이 절제)
- `naver`: `bg-naver text-naver-on`, 동일 brightness 방식
- 두 variant 모두 `focus-visible:ring` 접근성 처리 포함

**`.claude/rules/DESIGN.md`**:
- 버튼 컴포넌트 목록에 `button-kakao`/`button-naver` 항목 추가(용도·색·제한 명시)
- Don't 목록 첫 항목에 예외 주석 추가: "단, 외부 지도 서비스 공식 버튼은 예외"

#### 지도 버튼 라벨 정확화

버튼 라벨을 "카카오맵에서 보기" / "네이버지도에서 보기"로 확정. "열기"나 "길찾기" 라벨은 부정확(URL이 주소 검색이지 길찾기가 아님). `mapSearchUrl`·`naverMapSearchUrl` 모두 `q={주소}` 검색 URL을 반환해 place ID 의존 없이 동작.

#### 카드 높이 동일화

`LocationDirections`의 `<li>` 자체를 grid item으로 두고, `<Reveal className="h-full">`로 Reveal 래퍼가 full height를 차지하게 한 뒤, 내부 `<div className="flex h-full flex-col ...">` 패턴으로 grid stretch 복구. 명시적 `min-h-*`나 `aspect-*` 없이 처리.

### 📌 참고사항

- **약도 이미지 교체**: `public/location/map-placeholder.svg`를 실제 약도 이미지로 덮어쓰거나 `LOCATION.map.src` 값을 변경(단일 교체점). `alt`는 `LOCATION.map.alt`에서 함께 수정.
- **지도 임베드 설정**: 카카오·네이버 임베드 URL 확보 시 `church.ts`의 `MAP_EMBED_SRC`에 입력하면 자동으로 iframe 분기 활성화.
- **대중교통 안내**: `LOCATION.directions[1].lines`(transit)는 현재 "버스 노선 정보 준비 중입니다." placeholder. 교회가 실제 노선 정보를 채우면 된다.
- **카카오·네이버 토큰 용도 제한**: `--color-kakao`·`--color-naver` 계열 4개 토큰은 `button-kakao`/`button-naver` 외 다른 컴포넌트에 적용 금지(globals.css 주석·DESIGN.md에 명시).
