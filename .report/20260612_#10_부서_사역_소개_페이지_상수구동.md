# 부서(사역) 소개 페이지 — 프론트 상수 구동 전환

## 작업 개요

부서(사역) 소개 페이지를 **프론트 상수 기반 자립 구조**로 구현한다. 초기 설계는 백엔드 `GET /api/departments` fetch 방식이었으나, 공개 인트로 페이지는 상수로 충분하고 백엔드 `department` 엔드포인트는 교인 정보관리(어드민) 전용이라는 결정에 따라 전환했다. 메인 페이지(`constants/church.ts`)와 동일한 패턴을 적용한다.

## 주요 변경 상세

### 1. 상수 구동 결정 근거

백엔드 `department` 엔드포인트는 교인 정보(직분·소속·권한)를 관리하는 어드민 전용 엔티티다. 공개 사역 인트로에 필요한 데이터(이름·인도자·설명·미디어)는 자주 바뀌지 않고, 교회 운영자가 관리하는 방식도 코드 배포를 통해 충분히 커버된다. 따라서 메인 페이지(`church.ts`)와 동일하게 **프론트 상수를 단일 출처**로 삼고, 백엔드 API 레이어는 삭제하지 않고 어드민용으로 그대로 보존한다.

### 2. DeptHero — 14B 카드 확장 스크롤

- `height: 300vh` sticky 섹션 안에서 scroll progress(`p`)를 0~1로 정규화.
- `p 0→0.55` 구간: `clip-path: inset(...)` 보간으로 카드 상태(컨테이너 내 라운드 카드) → 풀스크린 확장.
- `p 0→0.35` 구간: 타이틀 translateY(-80px) + opacity 0으로 페이드아웃.
- `p 0.6→0.9` 구간: caption opacity 0→1 + translateY 30px→0 페이드인.
- HeroMedia 타입(image/video)은 CrossHero와 공유(video는 `onError` 폴백 → img).
- `prefers-reduced-motion`: `height: auto`, sticky 해제, clip-path 제거, 정적 이미지(70vh) 표시.

### 3. 라우트 렌더링 전략

- `/departments`: 데이터 fetch 없음 → **Static** 자동 분류.
- `/departments/[slug]`: `generateStaticParams`가 `allDepartmentSlugs()`(7개: student·middle·high·youth·praise·men·women)를 반환 → **SSG**. 상수에 없는 slug는 `notFound()`.
- 상세 페이지는 `SiteShell` 대신 `SiteHeader(variant="transparent" solid)` + `main` + `CtaBand` + `SiteFooter`를 직접 합성한다(DeptHero 위에 투명 헤더 고정 필요).

### 4. nav IA 개편

```
변경 전                   변경 후
교육부서 (6개 링크)  →   사역 (5개 링크: student·youth·praise·men·women slug)
소식                →   교회소식
```

`navigation.test.ts`가 사역 하위 링크의 slug를 `allDepartmentSlugs()`와 대조해 드리프트를 자동 감시한다. DEPARTMENTS에 부서를 추가하거나 slug를 변경하면 이 테스트가 실패해 nav도 함께 업데이트하도록 강제한다.

### 5. API 레이어 보존

`src/lib/api/departments.ts`는 공개 페이지에서 호출하지 않으나 삭제하지 않는다.
- `getDepartments()`·`getDepartment(id)`: ISR 60초 캐시로 어드민이 소비 가능.
- `buildDepartmentTree(list)`: 평배열 → parentId 트리 조립(불변), 고아 노드 루트 승격.
- `bySortOrder`: `sortOrder` 기준 정렬 비교자.
- 연관 타입(`DepartmentCardResponse`·`DepartmentDetailResponse`·`DepartmentNode`)은 `src/lib/api/types.ts`에 추가.

### 6. MegaMenu 아이콘 확장

`NavIconKey` 유니온에 `users·music·heart` 추가. `MegaMenu.tsx`의 `ICONS` 맵에 Lucide `Users·Music·Heart`를 등록해 사역 링크에 아이콘 플레이트를 표시한다(DESIGN.md mega-menu 스펙: 40px 원형, primary-soft 배경, 20px currentColor 아이콘).

## 참고 사항

- **placeholder 교체**: `public/dept/{slug}.jpg`를 실제 사진으로 덮어쓰면 코드 변경 없이 반영된다. 파일명 관례(`slug + .jpg`)를 유지한다.
- **부서 추가**: `DEPARTMENTS`에 항목을 추가하면 `generateStaticParams`가 자동으로 정적 페이지를 생성하고, `navigation.test.ts`가 slug 드리프트를 감시한다. `MINISTRY_LINKS`(navigation.ts)도 함께 업데이트해야 테스트가 통과한다.
- **내비 하위부서 미노출**: 학생부 하위(중등부·고등부)는 메가메뉴에 없고 학생부 상세 페이지의 `SubDepartments` 섹션에서만 노출한다. 메가메뉴 depth를 늘리지 않는 DESIGN.md 원칙을 유지한다.
- **어드민 API 보존 이유**: 향후 어드민 회원관리 화면에서 소속 부서 선택·표시에 `getDepartments()`·`buildDepartmentTree()`를 재사용할 수 있도록 삭제하지 않는다.
