# [어드민][미디어] 갤러리·주보·미디어 라이브러리 관리 (#39)

### 📌 작업 개요

운영자가 미디어 파일을 중앙 라이브러리에서 관리하고, 주보·갤러리를 등록·수정·삭제하는 어드민 연동을 구현했다. 세 영역(미디어 라이브러리·주보·갤러리)을 한 스펙으로 묶어 3단계로 구현했고, 06·07 도메인이 재사용할 공용 컴포넌트(`DataTable`·`MediaUploader`·`MediaPicker`)를 단독 생산했다.

기능별 5개 커밋으로 정리: 공용 인프라 / 미디어 라이브러리 / 주보 / 갤러리 / 문서.

### 🎯 구현 목표

- 미디어 라이브러리 전용 화면 — 업로드·목록·삭제, 사용 중이면 사용처 표시 후 삭제 차단
- 갤러리 앨범 생성·수정·삭제, 사진 다중 추가(신규 업로드 + 기존 파일 재사용)·제거
- 주보 등록·수정·삭제(PDF 업로드 또는 기존 파일 재사용)
- 동시 수정 충돌(낙관락) 안내, 허용 형식·용량 초과 안내

### ✅ 구현 내용

#### 1) 어드민 공용 인프라 (단독 생산 — 06·07 소비)

- **`src/lib/admin/apiUpload.ts`** — 비-JSON 어드민 쓰기 헬퍼(멀티파트 업로드 + 쿼리파라미터 전용 쓰기). `authFetch`(401 refresh·큐잉) 재사용, Content-Type 미설정으로 브라우저가 multipart boundary 설정.
- **`src/components/admin/DataTable.tsx`** — lean 컬럼 구동 공용 표(`Column<T>` + rows + actions). 정렬·선택은 미내장, 페이지네이션은 `common/Pagination` 조합. 모바일 가로 스크롤(`overflow-x-auto`).
- **`src/components/admin/MediaUploader.tsx`** — 네이티브 파일 선택 + 클라 사전검증(형식 5종·10MB) + 라이브러리 업로드. `accept`(image/pdf/all)·`multiple`.
- **`src/components/admin/MediaPicker.tsx`** — 라이브러리/새 업로드 허브(Dialog + Tabs). 기존 미디어 선택 + 신규 업로드를 모두 `mediaId`로 통일해 반환. 갤러리(image·다중)·주보(pdf·단일) 공용.

#### 2) 미디어 라이브러리 (`/mypage/manage/media`)

- **`src/lib/api/media.admin.ts`** — 목록(type·날짜 필터·페이지)·업로드·참조 조회·삭제 API.
- **`src/components/admin/media/MediaLibrary.tsx`** — 필터바 + 업로더 + `DataTable`(파일명·썸네일·유형·크기·업로드일) + 페이지네이션. 행별 "열기" 링크(새 탭 — 이미지 미리보기·PDF 다운로드)·삭제.
- **`src/components/admin/media/MediaReferencesDialog.tsx`** — 차단형 삭제 안내. 사용 중인 콘텐츠(유형·제목) 목록 표시, 삭제 버튼 없이 편집 유도.
- **`src/app/(site)/mypage/manage/media/page.tsx`** — `MEDIA_MANAGE` 권한 게이트 + Suspense 경계.

#### 3) 주보 인라인 어드민 (공개 목록)

- **`src/lib/api/bulletins.admin.ts`** — 등록·수정·삭제(쿼리파라미터 쓰기, 수정은 낙관락 `version`).
- **`src/lib/api/bulletins.ts`/`types.ts`/`revalidate.ts`** — 상세 타입(`BulletinDetailResponse`, version 포함)·캐시 태그·`revalidateBulletins` 추가.
- **`BulletinFormDialog`·`BulletinAdminActions`** — 등록/수정 다이얼로그(MediaPicker로 PDF 선택), 공개 목록에 "새 주보" 버튼 + 행별 수정/삭제.

#### 4) 갤러리 인라인 어드민 (회원 목록·상세)

- **`src/lib/api/gallery.admin.ts`** — 앨범 생성/수정/삭제(JSON, 수정은 `version`)·사진 추가(`mediaIds`)·제거.
- **`AlbumFormDialog`·`GalleryAdminActions`·`GalleryPhotoManager`** — 앨범 등록/수정 다이얼로그, 목록 "새 앨범"·상세 수정/삭제, 사진 추가(MediaPicker)·사진별 제거.

### 🔧 주요 변경사항 상세

#### 통합 업로드 모델

모든 신규 파일은 항상 `POST /api/admin/media`로 라이브러리에 먼저 올려 `mediaId`를 받고, 갤러리·주보는 `mediaId`로만 참조한다. 결과적으로 멀티파트 업로드는 미디어 라이브러리 한 곳에만 존재하고, 갤러리 사진추가·주보 등록/수정은 쿼리파라미터 쓰기로 단순화됐다. 이슈 의도("업로드 파일은 라이브러리에 모여 여러 콘텐츠가 재사용")와 일치한다.

#### 차단형 삭제

미디어 삭제는 먼저 `GET /api/admin/media/{id}/references`로 사용처를 사전 조회한다. 참조가 있으면 안내 다이얼로그를 띄우고 삭제를 막는다(편집으로 유도). 동시성 대비로 실제 삭제 시 `409 MEDIA_IN_USE`가 와도 같은 안내로 전환하는 백스톱을 둔다. 앨범/주보 삭제·사진 제거는 연결 해제일 뿐 원본은 라이브러리에 보존된다.

#### 낙관락 `version`

수정 요청에 `version`을 동봉해 동시 수정 충돌을 감지한다. **갤러리 앨범은 body, 주보는 query**로 위치가 다르다(백엔드 계약 차이). 주보 수정은 목록 카드에 version이 없어, 다이얼로그 열 때 `getBulletin`(no-store)으로 최신 version을 시드한다(캐시 시 stale version → 즉시 충돌 방지). 충돌 시 최신본 재조회 후 재편집 안내.

#### 캐시 무효화 (영역별 경계)

- 주보(공개 RSC/ISR): `revalidateBulletins`(태그 무효화) + `router.refresh()`
- 갤러리(회원 클라이언트): `queryClient.invalidateQueries(["albums"]/["album", id])`
- 미디어 라이브러리(어드민 클라이언트): 접두 무효화 `["admin","media","list"]`(필터·페이지·MediaPicker 쿼리 일괄)

#### 반응형

미디어 목록 표는 모바일에서 가로 스크롤(`overflow-x-auto`)하고, 부차 컬럼(유형·크기)은 작은 화면에서 숨긴다(`hidden sm:table-cell`). 필터바는 모바일 세로 스택, MediaPicker 그리드는 `grid-cols-2 sm:grid-cols-3`.

**특이사항**:
- 권한 게이트(`RequirePermission`)는 UX 최적화일 뿐 보안 경계가 아니다 — 서버 `/api/admin/**`가 2단 방어한다.
- 클라 파일 검증(형식·용량)은 즉시 피드백용이며, 백엔드 매직바이트 검증이 실제 경계다.
- 어드민 쓰기 함수·요청 타입은 공개 GET 모듈과 분리(`*.admin.ts`)했다 — `authFetch`/`authStore` 체인이 서버 번들에 끌려가 빌드 실패하는 것을 방지.

### 📌 참고사항

- **구현 착수 전 실행 백엔드로 검증할 항목**: ① 갤러리 사진추가가 `mediaIds`만으로 200인지(OpenAPI는 `files` required 표기) ② 미디어 업로드 multipart 인코딩 수용 여부(머신 스펙은 application/json+binary로 렌더되는 springdoc 쿼크) ③ 업로드 직전 토큰 만료 시 FormData 재시도 본문 보존.
- `ContentRef.type` 값 casing이 OpenAPI에 미정의 → 참조 안내는 텍스트 표시만, 콘텐츠 딥링크는 실제 응답값 확인 후 추가.
- `DataTable`·`MediaUploader`는 06(분류)·07(거버넌스)이 재사용하는 공용 컴포넌트(크로스도메인 공유). 나머지(MediaPicker·폼 다이얼로그 등)는 05 전용.
