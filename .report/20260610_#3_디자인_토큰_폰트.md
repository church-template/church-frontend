# 디자인 토큰·폰트 구축 (#3 / T2)

### 📌 작업 개요

DESIGN.md를 단일 진실로 삼아 색·간격·라운드·타이포 토큰을 Tailwind v4 `@theme`로 노출하고, 텍스트 위계를 `typo.*` 의미 상수로 매핑했다. Pretendard Variable을 self-host로 적용하고 자동 다크모드를 제거했다. 빌드·린트 검증을 통과했다.

### 🎯 구현 목표

- DESIGN.md 토큰을 `globals.css @theme`에 1회만 정의 → Tailwind 유틸 자동 생성 (`bg-primary`, `p-section`, `text-display-mega` 등)
- 1토큰 = 1유틸 원칙으로 hex·px 인라인 사용 차단
- 디스플레이 타이포를 `clamp()`로 모바일→데스크톱 유동 스케일
- Pretendard Variable self-host (CDN 미사용), `lang=ko`, title=은샘교회
- 텍스트는 `typo.*` 상수로만 적용하도록 규칙 명문화

### ✅ 구현 내용

#### 디자인 토큰 정의 — globals.css

- **파일**: `src/app/globals.css`
- **변경 내용**: Tailwind v4 `@theme` 블록에 DESIGN.md의 전 토큰을 단일 정의. 색(primary 계열·ink/body/muted·hairline·surface 계열·on-dark·success/error), 라운드(xs~pill/full, 카드 표준 xl=24px), 의미 간격(xxs~section, `p-section`=96px), 레이아웃(container-max 1200px·container-padding 24px·nav 64px), 타이포(size+line-height+weight+letter-spacing 수식키 묶음)를 노출
- **이유**: 값을 한 곳에만 두고 Tailwind 유틸을 자동 생성해, hex·px 인라인을 구조적으로 막고 교회별 primary 4토큰 교체만으로 리브랜딩 가능하게 한다

#### cover-dark 토큰 추가

- **파일**: `src/app/globals.css`, `.claude/rules/DESIGN.md`
- **변경 내용**: 메인 십자가 히어로(가이드 14A CrossHero) 덮개색 `--color-cover-dark: #0a0f1f`를 토큰으로 추가. DESIGN.md `colors:` 블록과 프로젝트 규칙 §3 히어로 연동 항목에도 반영
- **이유**: 히어로 덮개색을 인라인 hex가 아닌 토큰(`bg-cover-dark`)으로 참조하기 위함

#### 자동 다크모드 제거

- **파일**: `src/app/globals.css`
- **변경 내용**: `prefers-color-scheme` 기반 자동 다크모드를 걷어내고 body를 흰 캔버스(`--color-canvas`)·ink 텍스트로 고정
- **이유**: DESIGN.md Known Gaps — 다크 모드는 v1 범위 밖. 자동 반전 시 디자인 시스템 톤이 깨진다

#### 타이포 의미 상수 — typography.ts

- **파일**: `src/constants/typography.ts`
- **변경 내용**: `typo` 객체로 `displayMega`~`navLink` 16개 위계를 Tailwind 유틸 별칭으로 매핑. `datetime`은 `text-datetime tabular-nums`로 자릿수 고정(tnum) 부여. `TypoToken` 타입 노출
- **이유**: 컴포넌트가 size/weight를 직접 쓰지 않고 의미 이름(`typo.displayMega`)으로만 적용하도록 강제. 실제 값은 globals.css에 단일 정의되고 여기선 이름만 매핑

#### 폰트 self-host — layout.tsx + 폰트 파일

- **파일**: `src/app/layout.tsx`, `src/app/fonts/PretendardVariable.woff2`(신규)
- **변경 내용**: `next/font/local`로 Pretendard Variable(가변, weight `100 900`, `display: swap`)을 self-host. CSS 변수 `--font-pretendard`를 `<html>`에 주입하고 globals.css `@theme inline`의 `--font-sans`가 이를 참조. `lang="ko"`, `metadata.title`=은샘교회(`CHURCH_NAME` 상수)
- **이유**: CDN 의존 제거로 로딩 안정성·개인정보 측면 강화. 가변 폰트 하나로 100~900 전 굵기 커버

#### 규칙 명문화 — CLAUDE.md

- **파일**: `CLAUDE.md`
- **변경 내용**: "텍스트 스타일은 `typo.*` 상수 사용" 금지 규칙 추가. 새 위계는 DESIGN.md → globals.css `@theme --text-*` → `typo` 순으로 추가 후 사용하도록 흐름 명시
- **이유**: 토큰 단일 진실 흐름을 어기는 인라인 스타일을 차단

### 🔧 주요 변경사항 상세

#### 반응형 디스플레이 (clamp 유동)

`display-mega`~`display-sm`은 `clamp(min, vw, max)`로 정의(예: `--text-display-mega: clamp(36px, 6vw, 72px)`). 별도 미디어쿼리 없이 뷰포트 폭에 따라 72→36px로 유동 스케일된다. 본문·타이틀은 고정 크기를 유지한다.

#### 1토큰 = 1유틸 (수식키 묶음)

각 타이포 토큰은 size와 함께 `--text-*--line-height`, `--text-*--font-weight`, `--text-*--letter-spacing` 수식키를 같이 정의한다. 그 결과 `text-display-mega` 유틸 하나에 크기·행간·굵기·자간이 모두 묶여 적용되어, 한 토큰을 한 유틸로 완결한다.

**특이사항**:
- 폰트는 런타임에 next/font가 주입하는 변수라 `@theme inline`으로 매핑(빌드 타임 고정값이 아님)
- `datetime`의 tnum은 globals.css가 아니라 `typo.datetime` 상수가 `tabular-nums` 유틸로 부여
