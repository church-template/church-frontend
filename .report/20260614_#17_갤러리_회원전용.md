### 📌 작업 개요

회원 전용 갤러리 기능(T16)을 조회 전용으로 구현했다. `GALLERY_VIEW` 권한 게이팅을 최우선으로 적용해, 권한이 없는 사용자는 API 호출 없이 안내 화면을 받는다. 앨범 목록(`/gallery`), 앨범 상세(`/gallery/albums/[id]`), 사진 라이트박스를 포함하며, 데이터 패칭은 TanStack Query + `authFetch` 경유 클라이언트 컴포넌트로 처리한다.

---

### 🎯 구현 목표

- **게이팅**: `GALLERY_VIEW` 미보유 시 API 호출 없이 안내(비로그인·USER 모두)
- **목록**: 앨범 카드(썸네일·photoCount·태그·author), `createdAt,desc` 정렬·태그 필터, Pagination
- **상세**: description 마크다운 + 사진 그리드 + 라이트박스(←/→·ESC·caption)
- **범위 밖(YAGNI)**: 앨범·사진 생성/수정/삭제(어드민 `GALLERY_WRITE`), 무한스크롤, 다운로드, EXIF

#### ② GalleryGate — 권한 게이트

- **파일**: `src/components/gallery/GalleryGate.tsx` (신규)
- **변경 내용**: `useMe()` + `usePermission("GALLERY_VIEW")`로 분기. 권한 없으면 `children`을 마운트하지 않아 하위 API 호출을 구조적으로 차단한다("권한 전 호출 0회").

  분기 순서는 **비로그인 → 로딩 → 에러 → 권한없음 → 통과** 순으로 반드시 지켜야 한다. `useMe`는 `enabled: !!accessToken`이라 **비로그인 시 isPending이 영구 true**이므로, `!accessToken`을 `isPending`보다 먼저 평가하지 않으면 비로그인 사용자에게 Skeleton이 영구 노출된다.

  | 순위 | 상태 | 조건 | 표시 |
  |---|---|---|---|
  | 1 | 비로그인 | `!accessToken` (useMe 미발화) | "로그인 후 이용 가능" + 로그인 CTA |
  | 2 | 하이드레이션 전 | `!hydrated` | GallerySkeleton(깜빡임 방지) |
  | 3 | 로딩 | `isPending` | GallerySkeleton |
  | 4 | 에러 | `isError` | "다시 시도" 버튼 |
  | 5 | 권한 없음(USER) | `GALLERY_VIEW` 미보유 | "교인 승인 후 이용 가능" |
  | 6 | 통과 | `GALLERY_VIEW` 보유 | `children` 렌더 |

  방어선: 앨범 쿼리가 403 `ACCESS_DENIED`로 실패하는 예외 상황(세션 중 권한 회수)은 일반 에러 안내로 처리.

- **이유**: 게이트가 마운트를 통제해야 하위 쿼리 발화 자체를 막을 수 있다. 쿼리 레벨 `enabled` 조합만으로는 GalleryGate가 없는 경우 직접 import 시 호출을 보장하기 어렵다. 로그인 CTA 경로는 기존 패턴(`/login?next=${encodeURIComponent(sanitizeNext(pathname))}`) 재사용.

#### ③ useHasHydrated — 하이드레이션 가드

- **파일**: `src/lib/auth/authStore.ts` (수정)
- **변경 내용**: `useHasHydrated()` 추가. `useSyncExternalStore`로 Zustand persist 복원 완료 여부를 구독형으로 읽는다.
  ```ts
  export function useHasHydrated(): boolean {
    return useSyncExternalStore(
      (onChange) => useAuthStore.persist.onFinishHydration(onChange),
      () => useAuthStore.persist.hasHydrated(),
      () => false,
    );
  }
  ```
- **이유**: `useEffect + setState` 패턴은 "set-state-in-render-effect" 경고 및 SSR 스냅샷 불일치를 유발한다. `useSyncExternalStore`는 서버 스냅샷(항상 `false`)·클라이언트 스냅샷을 분리해 안전하게 처리한다. 기존 `hasHydrated()`(동기 게터)는 미사용 보존.

### 🔧 주요 결정·특이사항

#### 왜 클라이언트 컴포넌트인가

설교·공지 같은 공개 페이지는 서버 컴포넌트 + ISR이지만, 갤러리는 `GALLERY_VIEW` 회원전용이라 쿠키 없는 서버 fetch가 불가하다. 가이드 15.1 데이터 패칭 경계에 따라 **클라이언트 컴포넌트 + TanStack Query + `authFetch`**로 구현. (마이페이지와 동일 경계.)

#### 게이트가 자식 마운트를 통제하는 이유

`GalleryGate`가 `children`을 마운트하지 않으면 하위 쿼리 훅 자체가 실행되지 않는다 → API 호출 완전 차단. 쿼리 훅의 `enabled: false` 조합만으로는 훅이 정의된 컴포넌트가 마운트되는 것 자체를 막을 수 없어 구조가 더 취약하다.

#### `useHasHydrated` — useSyncExternalStore 선택 이유

persist 복원 완료를 `useEffect + setState`로 읽으면 리액트가 "render 중 set-state" 경고를 낼 수 있고, SSR 스냅샷(서버 렌더)과 클라이언트 첫 페인트가 일치하지 않는다. `useSyncExternalStore`는 서버 스냅샷 `() => false`를 명시해 이 문제를 회피한다.

#### `useGalleryTags` — getTags 재사용(공개 fetch)

`/api/tags`는 OpenAPI 전역 security 블록에 포함되지만 가이드 2.3·6.1에 따라 공개다. `authFetch`로 토큰을 부착하지 않고 기존 `getTags`(plain fetch)를 그대로 재사용해, 비로그인 상태에서도 TagFilter 렌더가 가능하다.

#### 기존 `hasHydrated()` 미삭제

`hasHydrated()`(동기 게터)는 T15 이전에 선언된 함수로, 현재 직접 참조하는 코드가 없다. 삭제 기준(내가 만든 변경으로 미사용이 된 것)에 해당하지 않아 확인 없이 삭제하지 않았다.

### 📌 참고·알려진 한계

- **403 세션 중 권한 회수**: 정상 흐름에서 403은 게이트가 선차단해 발생하지 않는다. 드물게 권한이 세션 중 회수된 경우 쿼리 실패 시 일반 에러 안내("다시 시도")로 처리하며, 별도 세분화 폴백은 미구현.
- **`img` 태그 사용**: `AlbumCard`·`PhotoGrid`·`PhotoLightbox` 모두 `<img>`(eslint 예외 주석)를 사용한다. 이 프로젝트에서 `next/image`로의 교체는 후속 과제로 남긴다.
- **어드민 GALLERY_WRITE**: 앨범·사진 생성·수정·삭제는 이슈 §4 명시에 따라 이번 배치 제외(조회만).
