# 공개 페이지 이동 로딩 스켈레톤 추가 (#95)

### 📌 작업 개요

공개 페이지(공지·주보·일정·설교)는 링크 클릭 후 서버 응답이 올 때까지 이전 화면이 그대로 멈춰 있어 클릭 접수 신호가 없었다. 특히 주 사용자층인 고령 교인은 반응이 없으면 같은 링크를 다시 누른다. Next.js App Router의 `loading.tsx`(Suspense 경계)로 클릭 즉시 콘텐츠 영역에 스켈레톤 자리표시가 뜨도록 추가했다.

### 🎯 구현 목표

- 각 공개 세그먼트에 로딩 스켈레톤 배치 — 클릭 즉시 피드백, 내비게이션·뒤로가기는 계속 사용 가능
- 스켈레톤이 실물 페이지와 동일한 반응형 구조를 미러링해 실제 콘텐츠 교체 시 레이아웃 점프 방지
- 화면 전체를 막는 전역 로딩 모달은 미채택 (짧은 이동에도 조작을 차단해 오히려 경험 저하)

### ✅ 구현 내용

#### 공용 스켈레톤 컴포넌트 3종 추가
- **파일**: `src/components/common/ListRowsSkeleton.tsx`, `CardGridSkeleton.tsx`, `DetailSkeleton.tsx`
- **변경 내용**: 기존 `Skeleton` 프리미티브(펄스 애니메이션)를 조합해 콘텐츠 유형별 자리표시 3종 구현
  - `ListRowsSkeleton` — 목록 행(공지·주보) 미러: 제목+날짜 가로 행 + hairline 구분, `rows` prop(기본 10)
  - `CardGridSkeleton` — 카드 그리드(설교) 미러: 16:9 썸네일 + 제목/메타 2줄, `count` prop(기본 6)
  - `DetailSkeleton` — 상세 본문(공지·설교·일정) 공통: 뒤로가기 줄 + 제목 + 메타 + 구분선 + 본문 문단
- **이유**: 도메인마다 스켈레톤을 중복 작성하지 않고 콘텐츠 형태(목록/그리드/상세) 단위로 재사용

#### 공개 세그먼트에 loading.tsx 배치 (7개)
- **목록**: `notices/loading.tsx`, `bulletins/loading.tsx`, `events/loading.tsx`, `sermons/loading.tsx`
- **상세**: `notices/[id]/loading.tsx`, `events/[id]/loading.tsx`, `sermons/[id]/loading.tsx`
- **변경 내용**: 각 세그먼트에 `loading.tsx`를 두어 라우트 전환 시 자동 표시. 실물 페이지와 동일 골격(제목 즉시 표시 → 검색/필터 자리 → 목록/그리드/캘린더)으로 구성
- **이유**: App Router는 `loading.tsx`가 있으면 해당 세그먼트 로딩 중 자동으로 렌더 — 별도 상태 관리 없이 서버 응답 대기 피드백 확보

#### 단위 테스트 3종
- **파일**: `ListRowsSkeleton.test.tsx`, `CardGridSkeleton.test.tsx`, `DetailSkeleton.test.tsx`
- **변경 내용**: 각 스켈레톤의 렌더 개수(rows/count) 및 구조 검증

### 🔧 주요 변경사항 상세

#### 반응형 미러링
`CardGridSkeleton`은 설교 목록과 동일한 그리드 클래스(`grid gap-base sm:grid-cols-2 lg:grid-cols-3`)를 그대로 사용한다. 모바일 1열 / 태블릿 2열 / 데스크톱 3열이 자동으로 일치하므로 스켈레톤 → 실제 카드 교체 시 열 수·간격이 흔들리지 않는다.

`DetailSkeleton`은 제목 자리를 `h-9`로 두어 실제 제목(`titleLg` 30px)의 줄높이를 근사, 교체 시 세로 점프를 최소화한다.

#### 접근성
- 로딩 안내는 `<p role="status" className="sr-only">페이지를 불러오는 중입니다</p>`로 스크린리더에만 전달
- 시각적 자리표시(스켈레톤 블록)는 모두 `aria-hidden` — 보조기기가 의미 없는 반복 요소를 읽지 않게 함
- 목록 loading은 페이지 제목(`<h1>`)을 즉시 노출해 어느 페이지로 이동 중인지 바로 알 수 있게 함

#### 서버 컴포넌트 / 토큰 준수
- 스켈레톤·loading 모두 순수 표시용 서버 컴포넌트(클라이언트 상태·JS 0줄) — 별도 번들·하이드레이션 부담 없음
- 색·간격·라운드는 전부 디자인 토큰 사용(`bg-surface-strong`, `border-hairline`, `gap-base`, `py-section` 등) — hex·px 인라인 없음, 타이포는 `typo.*` 상수

**특이사항**:
- **주보 상세(`bulletins/[id]`)는 loading 미배치** — 주보 행은 새 탭 PDF 링크(`GET /api/media/{id}`)라 상세 페이지 자체가 없어 로딩 경계가 불필요
- 일정(`events`) 목록은 캘린더 UI라 행/그리드 스켈레톤 대신 캘린더 블록을 근사한 단일 자리표시(`h-96` 라운드 박스) 사용
