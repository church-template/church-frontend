# T07 — 앱 셸 (레이아웃 · 네비 · 푸터 · 정적 페이지 · 404/error)

### 📌 작업 개요
모든 페이지가 얹히는 전역 셸을 구축. 공유 컨테이너(1200/24), 네비게이션 2모드(라이트/투명 + 모바일 Sheet), 프리푸터 CTA 밴드 + 푸터, API에 없는 정적 콘텐츠 페이지(소개·연혁·비전·오시는 길·예배 시간), 라우트 레벨 에러 페이지(404·런타임 바운더리)를 구현. 도메인 페이지(T8~T16)가 이 셸을 소비한다.

T07은 셸/프레젠테이션 레이어로, 도메인 데이터 패칭이나 히어로(14A/14B) 실제 구현은 포함하지 않는다(투명 네비 variant 자리만 제공).

### 🎯 구현 목표
- 표준 서브페이지·메인/부서·404·런타임 에러 4종 진입 경로를 단일 셸 컴포넌트로 합성
- IA(정보구조) 단일 출처 확립 — 헤더·모바일·푸터가 동일 데이터 소비
- 교회 고유값·콘텐츠 하드코딩 0 (상수 주입)
- TDD로 80%+ 커버리지 확보

### ✅ 구현 내용

#### 셸 아키텍처 — route group + 재사용 컴포넌트 합성
- **파일**: `src/app/(site)/layout.tsx`, `src/components/shell/SiteShell.tsx`
- 표준 서브페이지는 route group `(site)/layout.tsx`가 `<SiteShell>`을 주입(URL 세그먼트 추가 없음 — `/about` 그대로).
- 메인(T8)·부서(T9)의 **투명 네비**가 서브페이지 라이트 네비와 충돌하므로, 루트 레이아웃에 네비를 박지 않고 그룹 레이아웃이 라이트 셸을 주입하는 구조 채택.
- **404/error는 루트 파일이라 그룹 레이아웃을 상속하지 못하므로** `<SiteShell>`을 직접 합성(레이아웃 상속 대신 컴포넌트 합성).
- `<SiteShell>`은 순수 합성 컴포넌트라 서버 트리(`not-found.tsx`)·클라이언트 트리(`error.tsx`) 양쪽에서 렌더. 내부 `<SiteHeader>`만 `'use client'`이고 client인 `error.tsx`에 임포트돼도 문제없다.
- 루트 `app/layout.tsx`는 변경 없음(html/body/폰트/Providers/Toaster 최소 유지).

#### 단일 컨테이너 — Container
- **파일**: `src/components/shell/Container.tsx`
- `mx-auto w-full max-w-[var(--container-max)] px-[var(--container-padding)]` — 모든 섹션·히어로가 이 컴포넌트를 거쳐 폭(1200px)/패딩(24px)을 공유.
- props `{ as?, className?, children }`. `as`로 시맨틱 태그(`section` 등) 지정, `className`은 `cn()` 병합.
- t-shirt 토큰 충돌(`max-w-md` 류) 회피 위해 CSS 변수 직접 참조.

#### 네비게이션 2모드 — SiteHeader
- **파일**: `src/components/shell/SiteHeader.tsx` (`'use client'`)
- **light**: `border-b border-hairline bg-canvas text-ink`, 64px, 일반 흐름(콘텐츠를 밀어냄). 서브페이지 기본.
- **transparent**: `fixed inset-x-0 top-0 z-nav bg-transparent text-on-dark`. 메인(14A) 어두운 덮개 위 가독성용.
- 구성: 로고(좌, `CHURCH_NAME` 워드마크 — 별도 이미지 에셋 불필요) / 데스크톱 메뉴(`NAV_PRIMARY` 소비) / 우측 인증 CTA(로그인·마이페이지) / 768px↓ 햄버거(`lucide-react` Menu) → `<MobileNav>`.
- 자식 있는 항목(교회소개·소식)은 T4 `DropdownMenu` 재스킨(포털 z-popover), 단일 항목은 `<Link>`.
- 활성 표시: `usePathname()` + `isActiveItem`/`isActivePath`로 현재 경로 prefix 매칭 → 밑줄 강조.
- 햄버거 버튼은 아이콘 24 + 패딩 8×2 = 40px 터치 타깃, `aria-expanded`·focus-ring 부여.

#### 모바일 Sheet — MobileNav
- **파일**: `src/components/shell/MobileNav.tsx` (`'use client'`)
- T4 `Sheet`(side="right") 재스킨. SiteHeader가 `open` 상태를 소유하는 제어형.
- 동일 IA 트리를 펼친 섹션으로 표시 + 핵심 CTA(로그인/마이페이지) 유지.
- **닫힘 메커니즘**: 각 nav `<Link>`를 `SheetClose asChild`로 래핑 → 클릭 즉시 닫힘. 보강으로 `usePathname()` 변경 감지 effect가 `open=false`로 동기화(프로그램적 라우트 전환·뒤로가기 대비, 최초 렌더는 `useRef`로 스킵).
- `aria-describedby={undefined}`로 Radix 설명 경고 억제(네비 시트는 별도 설명 텍스트 불필요).

#### 프리푸터 CTA 밴드 + 푸터
- **파일**: `src/components/shell/CtaBand.tsx`, `src/components/shell/SiteFooter.tsx` (둘 다 순수)
- `CtaBand`: `bg-surface-dark py-section`(96px). 헤드라인 "처음 오셨나요?"(`typo.displayLg text-on-dark`) + CTA 2개 — 새가족 안내(`buttonVariants("pillCta")` → `/about`) · 오시는 길(`outlineOnDark`, 56px → `/about/location`). 카피는 `content.ts`.
- `SiteFooter`: `footer-light`(교회 정보 + `FOOTER_COLUMNS` 링크열, 4열 그리드) + `legal-band`(`© {연도} {CHURCH_NAME}`). 연도는 렌더 시점 `getFullYear()`.

#### 정적 콘텐츠 페이지 5종
- **파일**: `src/app/(site)/about/{page,history,vision,location}`, `src/app/(site)/worship/page.tsx`
- 라우트 + DESIGN 토큰 단순 섹션, 콘텐츠는 `content.ts` 상수 소비(패칭 없음, 과설계 금지).
- 오시는 길: `MAP_EMBED_SRC` 상수가 있으면 `<iframe>` 임베드, 없으면 주소 + "지도에서 보기" 외부 링크 폴백(지도 SDK 라이브러리 미사용).
- 예배 시간: `WORSHIP_SERVICES`(T8 메인 섹션과 공유) 렌더.

#### 라우트 레벨 에러 페이지
- **파일**: `src/app/not-found.tsx`(서버), `src/app/error.tsx`(`'use client'`), `src/app/global-error.tsx`(`'use client'`)
- `not-found`: `<SiteShell showCtaBand={false}>` + "페이지를 찾을 수 없습니다" + 홈 복귀. 미매칭 URL과 공개 서버 페이지의 `notFound()`를 **둘 다** 전역 catch.
- `error`: `unstable_retry`(Next 16.2, `reset` 대체)로 재시도 + 홈. `useEffect`로 `console.error`(콘솔만), 사용자에겐 일반 문구만 — **스택·원문 메시지 비노출**.
- `global-error`: 루트 레이아웃 붕괴 최후 바운더리. 자체 `<html lang="ko"><body>` 포함, globals.css가 보장되지 않으므로 예외적으로 최소 인라인 스타일만 사용(토큰 값을 주석으로 병기).
