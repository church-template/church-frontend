# 배포 전 종합 점검 후속 개선 6건 (#61)

### 📌 작업 개요

배포 전 12차원 종합 점검(적대적 검증 포함)에서 확정된 should_fix 개선 6건을 반영했다.
모두 **동작 변경 없는 품질·견고성 개선**으로, 기존 컨벤션을 따르는 범위 안에서 최소 수정으로 처리했다.
점검에서 도출된 잠재 부담(refetch storm, 상태 미구분 UI, 토큰 미준수, 키 하드코딩, 미검증 분기, 운영 환경변수 안내 부족)을 코드/문서 레벨에서 정리했다.

### 🎯 개선 목표

- 회원 영역 TanStack Query의 재요청 부담을 기본 옵션으로 통제한다.
- 태그 선택 컴포넌트에서 "없음"과 "실패"를 시각적으로 구분한다.
- 히어로 인라인 색상값을 디자인 토큰 참조로 통일한다.
- 어드민 목록 무효화 쿼리키를 중앙 규약으로 일원화한다(하드코딩 제거).
- 핵심 에러 처리기의 미검증 방어 분기를 테스트로 보강한다(브랜치 100%).
- 운영 환경변수의 주입 시점·실패 위험을 문서로 명확히 한다.

### ✅ 구현 내용

#### 1. QueryClient 기본 옵션 추가 — refetch storm 방지

- **파일**: `src/app/providers.tsx`
- **변경 내용**: 회원 영역 전용 `QueryClient`의 `defaultOptions.queries`에 `staleTime: 60_000`, `refetchOnWindowFocus: false`, `retry: false`를 추가.
- **전후 차이**: 기존에는 기본값(staleTime 0 · 탭 복귀 시 자동 재요청 · 실패 시 자동 재시도)이 그대로 적용되어, 사용자가 다른 탭을 다녀오면 회원 영역 쿼리가 일괄 재요청되는 부담이 있었다. 변경 후 60초 동안은 캐시를 신선하게 보고 탭 복귀 재요청을 끄며, 인증 흐름에 무의미한 재시도도 제거된다.
- **무효화와의 관계**: 짧은 캐시를 두되, 변경(mutation) 후에는 각 컴포넌트의 `onSuccess`에서 `invalidateQueries`로 즉시 갱신하므로 데이터 신선도는 유지된다(주석에 명시).

#### 2. TagMultiSelect 로딩·오류 상태 분리

- **파일**: `src/components/admin/TagMultiSelect.tsx`
- **변경 내용**: `useQuery`에서 `isPending`/`isError`를 함께 받아 팝오버 내부 표시를 3-way로 분기 — 오류(`isError`) → "태그를 불러오지 못했습니다."(`text-error`), 로딩(`isPending`) → "불러오는 중…", 그 외 목록 있음/없음.
- **전후 차이**: 기존에는 `data = []` 폴백만 사용해 **로딩 중·불러오기 실패·실제로 태그 없음**이 모두 "등록된 태그가 없습니다."로 동일하게 보였다. 변경 후 운영자가 일시적 실패와 빈 목록을 구분할 수 있다.
- **참고**: 분기는 프로젝트 규칙대로 삼항으로 작성, 메시지 색은 semantic 토큰(`text-error`/`text-muted`)으로 처리.

#### 3. 히어로 인라인 #fff → 디자인 토큰

- **파일**: `src/hero/CrossHero.module.css`, `src/hero/DeptHero.module.css`
- **변경 내용**: 캡션 텍스트 색상의 인라인 `#fff`를 `var(--color-on-dark)`로 교체.
- **전후 차이**: 다크 히어로 위 텍스트가 하드코딩 색상이 아니라 디자인 토큰을 참조하게 되어, "다크 밴드·히어로 덮개 안 텍스트는 on-dark 계열 토큰 사용" 규칙(DESIGN.md 구현 노트 3)에 부합한다.
- **예외 유지**: 검증 로직 성격의 내부 수치(`text-shadow`의 `rgba(...)` 등)는 히어로 토큰 인라인 금지의 예외라 그대로 둔다.

#### 4. 어드민 무효화 전용 쿼리키 헬퍼 `listAll` 추가 및 하드코딩 치환

- **파일**: `src/lib/admin/queryKeys.ts`, `src/lib/admin/queryKeys.test.ts`
- **소비 측 치환**: `src/components/admin/members/MemberProfileForm.tsx`, `MemberRolesSection.tsx`, `MemberPositionSection.tsx`, `src/components/admin/media/MediaLibrary.tsx`
- **변경 내용**: 무효화 전용 prefix 헬퍼 `listAll(domain) => ["admin", domain, "list"]`(3요소)를 추가하고, 위 4개 컴포넌트에서 하드코딩된 무효화 키를 `adminKeys.listAll("members")` / `adminKeys.listAll("media")`로 치환.
- **전후 차이**: 무효화 키가 컴포넌트마다 하드코딩되어 중앙 규약과 분리돼 있던 것을, 단일 헬퍼 참조로 일원화했다.

##### ⚠️ 함정 회피 포인트 (핵심)

- 기존 `list(domain)`은 `["admin", domain, "list", params]` 형태로 **항상 4번째 요소가 붙는다**. params 없이 무효화 목적으로 `list(domain)`을 호출하면 `["admin", domain, "list", undefined]`가 되어 **trailing `undefined`** 가 끼어든다.
- TanStack Query의 퍼지(prefix) 매칭은 `["admin", domain, "list", undefined]`를 `["admin", domain, "list", { page: 0 }]` 같은 **params 있는 실제 쿼리와 매칭하지 못한다** → 목록 캐시가 무효화되지 않는 정적 위험.
- 그래서 무효화는 4요소 `list()`가 아니라 **3요소 `listAll()` prefix**(trailing `undefined` 없음)를 써서 params 유무와 무관하게 모든 목록 쿼리를 퍼지 매칭으로 잡도록 했다. (헬퍼 주석에도 이 의도를 명시)
- 테스트(`queryKeys.test.ts`)에 `listAll`이 3요소 prefix임을 검증하는 케이스(`["admin","members","list"]`)와, `list`가 params 없이도 trailing `undefined`를 동반함을 보이는 케이스를 함께 둬 회귀를 막는다.

#### 5. handleApiError 방어 분기 테스트 보강 (브랜치 81% → 100%)

- **파일**: `src/lib/auth/handleApiError.test.ts`
- **변경 내용**: 각 `errorCode` case의 폴백 체인(`error.detail ?? error.title ?? 기본 메시지`)과 핸들러 부재 경로를 모두 덮는 케이스를 보강. 예: `ACCESS_DENIED`의 detail/title/기본 3단 폴백, `DUPLICATE_RESOURCE`·`MEDIA_IN_USE`·`INVALID_INPUT_VALUE`의 핸들러 없음·title 폴백·기본 메시지, `OPTIMISTIC_LOCK_CONFLICT`의 `onReedit` 부재 경로 등.
- **전후 차이**: 점검 시 `??` 체인의 우측 분기(detail 없음 → title, title 없음 → 기본)와 콜백 미제공 폴백 일부가 테스트로 검증되지 않아 브랜치 커버리지가 81% 수준이었다. 보강 후 동일 함수에서 브랜치 100%를 달성했다.

#### 6. .env.example 운영 환경변수 안내 보강

- **파일**: `.env.example`
- **변경 내용**: `NEXT_PUBLIC_API_BASE`가 **빌드 타임에 인라인**되므로 빌드 시점에 반드시 설정돼 있어야 한다는 점(없으면 빈 문자열로 인라인되어 전 API 호출 실패)과, 운영(Vercel) 환경에서는 대시보드 Environment Variables에 등록해야 빌드가 주입한다는 안내를 보강.
- **전후 차이**: 운영 환경변수의 필요성·주입 시점이 문서로 명확해져, 배포 시 빈 값 인라인으로 인한 전면 API 실패를 사전에 방지한다.

### 📌 영향 범위 / 리스크

- 6건 모두 **동작을 바꾸지 않는 품질·견고성 개선**이다.
  - QueryClient 옵션: 재요청 빈도만 조정(데이터 신선도는 mutation 후 invalidate로 유지).
  - TagMultiSelect: 표시 메시지 분리(기능 동일).
  - 히어로 색상: 토큰 참조로 교체(렌더 색상 동일).
  - `listAll` 치환: 오히려 **무효화 누락 위험을 제거**(trailing undefined 함정 회피)하는 방향이라 견고성 강화.
  - 테스트/문서: 런타임 코드 동작 무변경.
- 무효화 키 치환은 함정 회피가 핵심이라, 회귀 방지용 단위 테스트가 함께 들어가 안전망이 확보돼 있다.
