# 버튼 커서 복원 · 교인 목록 페이지 크기 고정

### 📌 작업 개요
서로 독립적인 두 건의 변경.

1. Tailwind v4가 제거한 버튼 손가락 커서(`cursor: pointer`)를 base 레이어에서 전역 복원해 활성 버튼의 클릭 어포던스 회복.
2. 어드민 교인 목록(`/mypage/manage/members`)이 페이지당 10명 단위로 끊겨 보이도록 목록 쿼리에 페이지 크기를 고정.

---

## 1. 버튼 커서(`cursor: pointer`) 전역 복원

### 🔍 문제 분석
Tailwind v4의 preflight가 v3에 있던 `button { cursor: pointer }` 리셋을 제거했다. 결과적으로 활성화된 버튼에 마우스를 올려도 손가락 커서가 아니라 브라우저 기본 화살표 커서가 떠서, "클릭 가능한 요소"라는 시각적 신호(인터랙션 어포던스)가 사라졌다. UX 저하 이슈.

### ✅ 구현 내용

#### base 레이어에 버튼 커서 규칙 추가
- **파일**: `src/app/globals.css`
- **변경 내용**: `@layer base` 안에 `button:not(:disabled)`·`[role="button"]:not(:disabled)`에 `cursor: pointer` 규칙 추가
- **이유**: v3 동작을 전역으로 복원하되, 유틸리티로 개별 오버라이드가 가능하도록 우선순위가 낮은 base 레이어에 배치

```css
@layer base {
  button:not(:disabled),
  [role="button"]:not(:disabled) {
    cursor: pointer;
  }
}
```

### 🔧 주요 변경사항 상세
- **전역 적용**: 커스텀 `Button` 컴포넌트뿐 아니라 shadcn 재스킨 동작 컴포넌트(Dialog·Select·Sheet 등)의 버튼까지 한 번에 커버한다.
- **비활성 제외(`:not(:disabled)`)**: 비활성 버튼은 손대지 않는다. `Button`이 이미 `disabled:cursor-not-allowed`로 별도 처리하므로 충돌·중복을 피한다.
- **`[role="button"]` 포함**: `<button>` 외에 ARIA role로 버튼처럼 동작하는 요소도 동일하게 커버.

**특이사항**:
- base 레이어에 둔 것이 핵심이다. utilities 레이어 클래스가 항상 이 규칙을 이길 수 있어, 특정 컴포넌트에서 커서를 바꿔야 할 때 유틸로 덮어쓰면 된다.

### 🧪 테스트 및 검증
- `/showcase` 페이지에서 브라우저 자동화로 확인: 활성 버튼 20개 전부 커서 `pointer`, 비활성 버튼 5개 전부 `not-allowed`로 의도대로 분리됨.

## 2. 교인 목록 페이지 크기 고정 (10명/페이지)

### 🎯 구현 목표
어드민 교인 관리 화면(`/mypage/manage/members`)의 목록이 페이지당 일정 인원(10명) 단위로 페이지네이션되도록 한다.

### ✅ 구현 내용

#### 목록 쿼리에 페이지 크기 파라미터 전달
- **파일**: `src/components/admin/members/MemberManager.tsx`
- **변경 내용**: 상수 `PAGE_SIZE = 10` 추가, 회원 목록 쿼리 파라미터(`MemberListParams`)의 `size`에 주입
- **이유**: 목록 응답을 10명 단위로 끊어 한 화면 밀도를 일정하게 유지하고, 하단 `Pagination`이 페이지를 나눠 보여줄 수 있게 한다

```tsx
// 교인 목록 페이지 크기 — 10명 단위로 본다.
const PAGE_SIZE = 10;

const params: MemberListParams = {
  q: searchParams.get("q") || undefined,
  page: Number(searchParams.get("page") ?? "0") || 0,
  size: PAGE_SIZE,
};
```

### 🔧 주요 변경사항 상세
- 페이지 번호(`page`)는 기존대로 URL 쿼리스트링으로 구동되며, 검색·필터 변경 시 `page`를 리셋한다. 이번 변경은 `size`만 고정값으로 추가한 것이다.
- 하단 `Pagination`은 `members.data.page.totalPages > 1`일 때만 노출되므로, 결과가 10명 이하면 페이지네이션은 그대로 숨는다.

**특이사항**:
- `PAGE_SIZE`를 상수로 분리해 하드코딩 매직 넘버를 제거했다. 향후 페이지 크기 조정 시 한 곳만 바꾸면 된다.

### 📌 참고사항
- 두 변경은 서로 의존성이 없다. 커서 복원은 디자인 시스템 전역(globals.css)에, 페이지 크기는 어드민 회원 도메인 컴포넌트에 국한된다.
