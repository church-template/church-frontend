### 📌 작업 개요

어드민 트랙 06 — 운영자가 마이페이지 관리 허브에서 전역 태그와 직분(표시용 레이블)을 추가·수정·삭제할 수 있는 관리 화면 2종을 구현했다. 태그는 공개 ISR 소비자가 실재해 `revalidateTags()`로 Next.js 캐시를 무효화하고, 직분은 공개 소비자가 없어 클라이언트 쿼리 무효화만 수행한다.

---

### 🎯 구현 목표

- 태그 CRUD + 중복 이름 인라인 에러 + 삭제 시 연결 콘텐츠 경고
- 직분 CRUD + 정렬순서 입력 + "권한 무관 표시용" 안내 배너
- 권한(TAG_MANAGE / POSITION_MANAGE) 독립 게이트 적용
- TDD RED→GREEN, 전체 테스트 692건 통과

### 📌 비고

- `positions.ts`의 `getPositions`은 RSC import 금지 주석 명시 — 서버 번들 경계 위반 시 빌드 오류.
- POST 성공 코드는 OpenAPI 응답맵상 `200`(설명문만 "201 Created") → `apiMutate`가 2xx 일괄 처리하므로 status 코드 하드코딩 없음.
- 관리 허브(`manage-hub`) 카드 자동 노출은 `manageDomains.ts`·`permissions.ts`에 이미 등록된 내용 재활용 — 이번 구현에서 추가 변경 없음.
- 콘텐츠(설교·공지·일정 등)에 임베드된 태그 이름 반영은 지연(각 도메인 ISR 캐시 만료 시점) — 에픽 00 "공개 페이지 반영 지연 안내" 정책에 따라 cascade revalidate 미수행.
