# 회원 직분 부여/변경/해제 기능

### 📌 작업 개요
어드민 회원 상세 화면에서 회원에게 직분(목사·장로 등)을 부여·변경·해제할 수 있는 기능 추가. 기존에 역할(role)·권한(permission)은 부여 가능했으나 직분은 조회만 가능했던 문제 해결. 백엔드 직분 부여 엔드포인트 신설에 맞춰 프론트 연동.

### 🎯 구현 목표
- 회원 상세 다이얼로그에 전용 '직분' 섹션 추가 (역할 섹션과 동형)
- 직분 카탈로그에서 선택해 부여/변경, '(없음)' 선택으로 해제
- `MEMBER_MANAGE` 권한 보유자만 변경, 미보유 시 현재 직분 읽기 전용 표시

### ✅ 구현 내용

#### 직분 부여 API 함수 추가
- **파일**: `src/lib/api/members.admin.ts`
- **변경 내용**: `changePosition(uuid, positionId)` 함수 추가. `PUT /api/admin/members/{uuid}/position` 호출, body `{ positionId }`
- **이유**: 부여·변경·해제를 단일 엔드포인트로 처리. `positionId: null` 전송 시 직분 해제

#### 직분 섹션 컴포넌트 신설
- **파일**: `src/components/admin/members/MemberPositionSection.tsx` (신규)
- **변경 내용**: 직분 카탈로그 select + 변경 버튼. 권한 게이트(`MEMBER_MANAGE`)에 따라 편집/읽기전용 분기
- **이유**: 역할 섹션과 일관된 UX로 직분을 관리. 직분은 권한과 무관해 위계 검증 없이 단순 select 구조

#### 회원 상세 다이얼로그 조립
- **파일**: `src/components/admin/members/MemberDetailDialog.tsx`
- **변경 내용**: 역할 섹션과 약관 동의 섹션 사이에 헤어라인 구분선 + 직분 섹션 삽입
- **이유**: 역할 다음에 직분을 배치해 회원 권한·소속 정보를 한 흐름으로 노출

### 🔧 주요 변경사항 상세

#### MemberPositionSection
직분 카탈로그(`["positions"]` 쿼리)는 `PositionManager`와 키를 공유해 재사용. 회원의 현재 직분은 **이름 문자열**로 내려오므로, 카탈로그에서 이름→id를 역매핑해 select 초기값을 시드한다.

변경 성공 시 `setQueryData`로 회원 상세 캐시를 즉시 갱신하고 회원 목록 쿼리를 무효화해 화면을 동기화한다.

**특이사항**:
- **현재값 시드는 keyed 마운트 패턴 사용**: select 초기값을 effect 안 setState로 채우면 lint 에러(set-state-in-effect)가 나므로, 내부 편집 컴포넌트의 `key`를 현재 직분 id로 묶어 remount 방식으로 초기화. 변경 성공 후 현재값이 바뀌면 자동 remount되어 select가 새 값으로 갱신된다.
- **삭제된 직분 폴백**: 회원의 현재 직분 이름이 카탈로그에 없으면(직분 삭제 등) select는 '(없음)'으로 폴백.
- **해제 처리**: '(없음)' 선택 후 변경 = `positionId: null` 전송.
- **자기수정 허용**: 직분은 권한과 무관·위계 검증이 없어, 역할 섹션과 달리 자기 자신의 직분도 변경 가능.

#### API 경로 비대칭
회원 직분 조회 값은 이름 문자열인데 부여 API는 숫자 id를 받는다. 이 매핑을 컴포넌트에서 카탈로그 기준으로 처리한다.

### 🧪 테스트 및 검증
- API 단위 테스트: `changePosition`이 올바른 경로·body(`positionId` 부여/`null` 해제)로 호출되는지 검증
- 컴포넌트 테스트 5종: 현재 직분 시드 · 변경 호출 · 해제(null) · 권한 미보유 읽기전용 · 카탈로그 미존재 직분 '(없음)' 폴백
- 전체 테스트 스위트 777개 전부 통과, 타입체크·린트 통과

### 📌 참고사항
- 직분은 권한 체계와 완전히 분리된 표시용 정보 (목사·장로 등 호칭)
- 직분 카탈로그 자체(이름·정렬순서)는 별도 직분 관리 화면(`PositionManager`)에서 CRUD
- 관련 백엔드 엔드포인트: `PUT /api/admin/members/{uuid}/position` (`MEMBER_MANAGE` 권한)
