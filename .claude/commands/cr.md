# CR — Commit + Report + 보고서 커밋

`/commit`과 `/report`를 실행하고, 마지막에 **보고서 파일까지 커밋**하는 통합 커맨드.

**최적화 원리 (순서):** 소스 커밋(A)과 보고서 생성(B)은 서로 독립적이라 **병렬**로 돌린다.
- B(보고서)는 **소스 파일을 직접 읽어** 분석하고 결과를 `.report/`에만 쓴다 → **git 명령을 쓰지 않는다** → A의 git 인덱스 작업과 파일이 겹치지 않아 락 경합/충돌 없음.
- 보고서는 커밋 해시에 의존하지 않으므로(구현 내용 보고서), A보다 먼저/나중에 끝나도 무방 → 병렬로 시간을 아낀다.
- **보고서 커밋(3단계)은 A·B가 모두 끝난 뒤** 실행한다 → 소스 커밋과 보고서 커밋이 뒤섞이지 않고, 보고서 커밋이 항상 마지막에 온다.

> 참고: 이 저장소에서 `.report/`는 **추적 대상**이다(보고서도 커밋한다). 소스 커밋(A)은 `.report/`를 건드리지 않고, 보고서 커밋(3단계)만 `.report/`의 새 파일을 명시적으로 add 한다.

## 실행 절차

### 1단계: 변경 스냅샷 1회 캡처
`git status`를 **한 번만** 실행해 변경 파일 목록·현재 브랜치·이슈 번호(브랜치명에서 `#숫자` 추출)를 확보한다. 이 목록을 두 작업의 공통 입력으로 쓴다(이후 두 작업은 파일을 직접 읽으므로 `git diff`에 의존하지 않는다). 변경사항이 없으면 "커밋/보고할 변경사항이 없어요"로 종료.

### 2단계: 두 서브에이전트를 한 메시지에서 동시 dispatch (병렬)

- **에이전트 A — 소스 커밋**: `.claude/commands/commit.md`(Smart Commit) 규칙을 그대로 따른다.
  - 기능 단위 분리 커밋, 한글 메시지, `<type> : <설명> #이슈`
  - **push 금지 · Co-Authored-By 금지 · GPG 서명 금지 · 시크릿 파일 제외 · `git add .`/`-A` 금지(파일 경로 명시 지정)**
  - **`.report/`는 커밋하지 않는다**(3단계 소관).
- **에이전트 B — 보고서 생성**: `.claude/commands/report.md`(Report Mode) 규칙을 그대로 따른다.
  - `.report/[YYYYMMDD]_[#이슈]_[설명].md` 생성(파일 경로를 3단계에 넘길 것)
  - **작성자/작성일/AI 언급 금지 · 민감정보 마스킹**
  - 변경 파일을 직접 읽어 분석하고, **git 명령은 쓰지 않는다**(A와 병렬 실행 중이므로 인덱스 락 회피).

### 3단계: 보고서 커밋 (A·B 완료 후)
A와 B가 **둘 다 끝난 것을 확인한 뒤**, B가 만든 보고서 파일**만** 명시적으로 커밋한다.
- `git add ".report/<B가 생성한 파일명>"` (정확한 경로만 — `git add .`/`-A` 금지)
- `git commit -m "docs : 리포트 문서 작성 #이슈"`
- push 금지 · Co-Authored-By 금지 · GPG 서명 금지.

### 4단계: 통합 보고
결과를 합쳐 출력한다:
- 커밋 표 (해시 · 메시지 · 파일 수) — 소스 커밋들 + 보고서 커밋 포함
- 보고서 저장 경로

## 주의
- 확인 없이 즉시 실행, push 안 함.
- 변경사항이 없으면 "커밋/보고할 변경사항이 없어요"로 종료.
- 보고서 커밋은 반드시 마지막(모든 소스 커밋 뒤).
